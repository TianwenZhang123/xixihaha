## 三、各指标详细说明

### 3.1 FVD (Fréchet Video Distance) ↓

**脚本**: `evaluation/run_fvd_eval.py`

**原理**: 衡量两个视频集合在特征空间中的分布距离，是视频生成领域的标准指标。

**计算方法**:
1. 对每个视频均匀采样 16 帧
2. 用 I3D (Kinetics-400) 提取 400-dim 特征
3. 分别计算两组特征的均值 μ 和协方差 Σ
4. `FVD = ||μ_r - μ_g||² + Tr(Σ_r + Σ_g - 2·(Σ_r·Σ_g)^{1/2})`

**输出**: 单个 FVD 数值（越低越好）

**CLI 用法**:
```bash
python evaluation/run_fvd_eval.py \
    --orig-dir data/videos \
    --gen-dir outputs/pflow_200cases \
    --output-dir outputs/pflow_200cases/eval_fvd \
    --num-frames 16 \
    --device cuda
```

**参数**:
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--orig-dir` | `data/videos` | 原始视频目录 |
| `--gen-dir` | `outputs/baseline_batch` | 生成视频目录 |
| `--output-dir` | `outputs/eval_results/fvd` | 输出目录 |
| `--num-frames` | 16 | 每个视频采样帧数 |
| `--i3d-model` | `""` | I3D 权重路径（空=自动下载） |
| `--batch-size` | 8 | 批量推理大小 |
| `--device` | auto | 推理设备 |
| `--no-cache` | false | 禁用特征缓存 |
| `--limit` | 0 | 限制视频数量（0=全部） |

**输出文件**:
- `fvd_results.json` — FVD 数值 + 统计信息
- `eval_summary.md` — Markdown 报告
- `features/real/*.npy` — 原始视频 I3D 特征（缓存）
- `features/gen/*.npy` — 生成视频 I3D 特征（缓存）

**注意事项**:
- FVD 是集合级指标，需要 ≥100 个视频才能稳定（200 样本刚好满足）
- 如果 `pytorch-fvd` 不可用，会自动 fallback 到 ResNet3D-18（特征维度 512）
- 特征缓存为 `.npy` 文件，避免重复推理

### 3.2 DINO-Score (主体身份一致性) ↑

**脚本**: `evaluation/run_dino_eval.py`

**原理**: 用 DINOv2 的自监督特征衡量帧间主体一致性。DINOv2 对物体外观/纹理/形状极其敏感，能检测"同一辆车是否始终是同一辆车"。

**计算方法**:
1. 逐帧提取 DINOv2 ViT-B/14 的 [CLS] token 特征
2. **dino_temporal**: 生成视频相邻帧余弦相似度均值 → 帧间一致性
3. **dino_orig_gen**: 原始帧 vs 生成帧的逐帧余弦相似度均值 → 细粒度保真度

**输出**: 每个视频对 2 个数值（均越高越好）

**CLI 用法**:
```bash
python evaluation/run_dino_eval.py \
    --orig-dir data/videos \
    --gen-dir outputs/pflow_200cases \
    --output-dir outputs/pflow_200cases/eval_dino \
    --dinov2-model facebook/dinov2-vitb14 \
    --sample-frames 8 \
    --device cuda
```

**参数**:
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--orig-dir` | `data/videos` | 原始视频目录 |
| `--gen-dir` | `outputs/baseline_batch` | 生成视频目录 |
| `--output-dir` | `outputs/eval_results/dino` | 输出目录 |
| `--dinov2-model` | `facebook/dinov2-vitb14` | DINOv2 模型名称或本地路径 |
| `--sample-frames` | 8 | 每个视频采样帧数 |
| `--device` | auto | 推理设备 |
| `--limit` | 0 | 限制样本数量 |

**输出文件**:
- `dino_results.csv` — 逐样本结果
- `dino_results.json` — 汇总统计
- `eval_summary.md` — Markdown 报告（含逐样本表格）

**对 P-Flow 的意义**: L3 Feature Injection 的核心目标是注入参考视频的外观细节，DINO-Score 直接衡量这一效果。

### 3.3 Flow EPE (光流一致性) ↓

**脚本**: `evaluation/run_flow_eval.py`

**原理**: 用 RAFT 分别估算原始和生成视频的光流场，然后逐帧对计算端点误差。**这是验证 SVD Noise Prior (L2) 最直接的指标。**

**计算方法**:
1. 用 RAFT 估算原始/生成视频的相邻帧光流场 F_orig, F_gen
2. **flow_epe**: `mean_p(||F_orig(p) - F_gen(p)||₂)` 均值（越低越好）
3. **flow_cosine_sim**: 光流场展平后的余弦相似度（越高越好）
4. **dynamic_degree**: 生成视频光流幅度均值（越高越好）

**输出**: 每个视频对 3 个数值

**CLI 用法**:
```bash
python evaluation/run_flow_eval.py \
    --orig-dir data/videos \
    --gen-dir outputs/pflow_200cases \
    --output-dir outputs/pflow_200cases/eval_flow \
    --sample-frames 16 \
    --resize-width 640 --resize-height 360 \
    --device cuda
```

**参数**:
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--orig-dir` | `data/videos` | 原始视频目录 |
| `--gen-dir` | `outputs/baseline_batch` | 生成视频目录 |
| `--output-dir` | `outputs/eval_results/flow` | 输出目录 |
| `--sample-frames` | 16 | 采样帧数（建议 ≥16 以获得稳定光流） |
| `--resize-width` | 640 | 缩放宽度（降低 RAFT 计算量） |
| `--resize-height` | 360 | 缩放高度 |
| `--device` | auto | 推理设备 |
| `--limit` | 0 | 限制样本数量 |

**输出文件**:
- `flow_results.csv` — 逐样本结果（flow_epe, flow_cosine_sim, dynamic_degree）
- `flow_results.json` — 汇总统计
- `eval_summary.md` — Markdown 报告

**注意事项**:
- RAFT 推理较慢（每对视频约 5-10 秒），200 样本约 20-30 分钟
- 默认缩放到 640x360 以平衡精度和速度
- RAFT 输出为 1/4 分辨率光流，计算 EPE 时自动对齐

### 3.4 LPIPS (感知距离) ↓

**脚本**: `evaluation/run_lpips_eval.py`

**原理**: 用 LPIPS VGG backbone 计算逐帧感知距离，比 SSIM/PSNR 更符合人类感知。能检测局部纹理失真（CLIP 对此不敏感）。

**计算方法**:
1. 逐帧对齐（原始帧 vs 生成帧，同一首帧生成，天然对齐）
2. 逐帧计算 LPIPS 距离
3. 输出均值、最大值、最小值

**输出**: 每个视频对 3 个数值（均越低越好）

**CLI 用法**:
```bash
python evaluation/run_lpips_eval.py \
    --orig-dir data/videos \
    --gen-dir outputs/pflow_200cases \
    --output-dir outputs/pflow_200cases/eval_lpips \
    --sample-frames 8 \
    --device cuda
```

**参数**:
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--orig-dir` | `data/videos` | 原始视频目录 |
| `--gen-dir` | `outputs/baseline_batch` | 生成视频目录 |
| `--output-dir` | `outputs/eval_results/lpips` | 输出目录 |
| `--sample-frames` | 8 | 采样帧数 |
| `--resize-width` | 0 | 缩放宽度（0=保持原始） |
| `--resize-height` | 0 | 缩放高度 |
| `--batch-size` | 8 | 批量推理大小 |
| `--device` | auto | 推理设备 |
| `--limit` | 0 | 限制样本数量 |

**输出文件**:
- `lpips_results.csv` — 逐样本结果（lpips_avg, lpips_max, lpips_min）
- `lpips_results.json` — 汇总统计
- `eval_summary.md` — Markdown 报告

### 3.5 Temporal Flickering (时序闪烁) ↓

**脚本**: `evaluation/run_temporal_eval.py`

**原理**: 检测相邻帧之间的高频闪烁（纹理跳动、光照突变）。仅需生成视频，不需要原始视频。

**计算方法**:
1. **temporal_flicker**: 相邻帧逐像素 MAE 均值（越低越好）
2. **dynamic_degree**: 像素差均值 / RAFT 光流幅度均值（越高越好，可选 RAFT）

**CLI 用法**:
```bash
# 快速模式（仅像素级，不需要 RAFT）
python evaluation/run_temporal_eval.py \
    --gen-dir outputs/pflow_200cases \
    --output-dir outputs/pflow_200cases/eval_temporal

# RAFT 模式（更精确的 dynamic degree）
python evaluation/run_temporal_eval.py \
    --gen-dir outputs/pflow_200cases \
    --output-dir outputs/pflow_200cases/eval_temporal \
    --use-raft
```

**参数**:
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--gen-dir` | `outputs/baseline_batch` | 生成视频目录（不需要原始视频） |
| `--output-dir` | `outputs/eval_results/temporal` | 输出目录 |
| `--sample-frames` | 16 | 采样帧数 |
| `--resize-width` | 640 | 缩放宽度 |
| `--resize-height` | 360 | 缩放高度 |
| `--use-raft` | false | 使用 RAFT 计算 dynamic degree |
| `--device` | auto | 推理设备 |
| `--limit` | 0 | 限制样本数量 |

**输出文件**:
- `temporal_results.csv` — 逐样本结果（temporal_flicker, dynamic_degree）
- `temporal_results.json` — 汇总统计
- `eval_summary.md` — Markdown 报告

## 四、一键运行

**脚本**: `evaluation/run_all_metrics.py`

按顺序运行全部 5 个指标并汇总结果。

```bash
# 完整运行（200 样本，所有指标）
python evaluation/run_all_metrics.py \
    --orig-dir data/videos \
    --gen-dir outputs/pflow_200cases \
    --output-dir outputs/pflow_200cases/eval_all

# 跳过慢指标（RAFT），快速出结果
python evaluation/run_all_metrics.py \
    --orig-dir data/videos \
    --gen-dir outputs/pflow_200cases \
    --output-dir outputs/pflow_200cases/eval_all \
    --skip-flow --skip-temporal

# 仅 5 个样本快速测试
python evaluation/run_all_metrics.py \
    --orig-dir data/videos \
    --gen-dir outputs/pflow_200cases \
    --output-dir outputs/pflow_200cases/eval_all \
    --limit 5
```

**聚合输出**:
- `eval_all/all_results.json` — 所有指标汇总
- `eval_all/eval_summary.md` — 统一报告（含总览表格 + 各子报告链接）
