# P-Flow L1 & L2 实验版本对比

> **实验环境**：AutoDL A800 80GB，`/root/xixihaha/P-Flow`
>
> **生成模型**：Wan2.1-T2V-1.3B（UMT5 text encoder）
>
> **生成参数**：steps=30, guidance=5.0, seed=42
>
> **样本集**：10 个固定样本（IDs: 7, 17, 21, 31, 32, 33, 34, 43, 46, 47）
>
> **评测指标**：`orig_gen_clip`（CLIP 帧级相似度）、`orig_gen_xclip`（X-CLIP 时序语义相似度）

## 一、P-Flow 架构总览

P-Flow 采用分层条件注入框架（Hierarchical Conditioning Inversion），由三层组成：

- **L1（Prompt Rewrite）**：通过 LLM 对 VLM caption 进行结构性改写，优化文本条件
- **L2（SVD Noise Prior）**：对 DDIM 反演噪声进行 SVD 滤波，提取运动先验注入初始噪声
- **L3（Velocity Field Matching）**：速度场匹配，进一步约束生成轨迹（本文档不涉及）

理想目标是各层正交增益叠加：L1+L2 > max(L1, L2)。

## 二、SVD v1 方案详细流程（当前最优 L2 方案）

SVD v1 是当前唯一验证有效的 L2 方案（CLIP +2.4%, XCLIP +5.1% vs baseline）。以下详细描述其完整流程。

### 2.1 整体 Pipeline

```
参考视频 V_ref
    │
    ▼
[VAE Encode] ─── V_ref → z_0 (latent representation)
    │
    ▼
[Flow Matching Inversion] ─── z_0 → η_inv (反演噪声, 从 t=0 积分到 t=1)
    │                          输入: z_0, prompt_embedding
    │                          方法: Euler ODE solver, 30 steps
    │                          输出: η_inv ∈ R^(1, C, F, H, W), 包含视频的全部信息
    ▼
[SVD Two-Stage Filtering] ─── η_inv → η_temporal (运动先验)
    │                          Stage 1: 空间去内容 → 去除外观/静态信息
    │                          Stage 2: 时间保运动 → 只保留时间维度主成分
    │                          输出: η_temporal, std ≈ 0.28~0.41 (远小于 1.0)
    ▼
[Noise Blending] ─── η_temporal + η_random → z_T (初始噪声)
    │                 公式: z_T = √α · η_temporal + √(1-α) · η_random
    │                 α = 0.004 (极小剂量)
    │                 η_random ~ N(0, 1) (纯随机高斯噪声)
    ▼
[T2V Generation] ─── z_T + prompt → 生成视频
                      Wan2.1 模型 30 步去噪
                      模型完全冻结 (黑盒)
```

### 2.2 Stage 1: Spatial Decontenting（空间去内容）

目的：去除反演噪声中的"看起来像什么"信息，保留"怎么动"的信息。

算法：
1. 将 η_inv 从 (C, F, H, W) reshape 为 (C×F, H×W)，即把时间和通道维度合并为行，空间维度展开为列
2. 对这个 2D 矩阵做 SVD 分解：`M = U·S·Vᵀ`
3. 找到能量阈值 k_s：去除的 top-k_s 个空间主成分占总能量 ≥ (1-ρ_s)，其中 ρ_s=0.1（即去除 90% 的空间能量）
4. 减去 top-k_s 重建：`M_filtered = M - U[:,:k_s] · diag(S[:k_s]) · V[:,:k_s]ᵀ`
5. reshape 回 (C, F, H, W)

直觉：空间维度的前几个主成分编码了"这是什么物体"、"背景长什么样"等内容信息。去除后剩下的是跨帧变化的模式——即运动。

### 2.3 Stage 2: Temporal Retention（时间保运动）

目的：从去内容后的噪声中提取最显著的时间维度运动方向。

算法：
1. 将 Stage 1 输出从 (C, F, H, W) reshape 为 (C×H×W, F)，即空间和通道维度合并为行，时间维度为列
2. 对这个 2D 矩阵做 SVD 分解
3. 找到能量阈值 k_m：保留的 top-k_m 个时间主成分占总能量 ≥ ρ_m=0.9（保留 90% 的时间能量）
4. 只保留 top-k_m 重建：`M_temporal = U[:,:k_m] · diag(S[:k_m]) · V[:,:k_m]ᵀ`
5. reshape 回 (C, F, H, W)，得到 η_temporal

直觉：时间维度只有 F≈21 个分量（对应 81 帧 VAE 压缩后），SVD 保留的 top-k_m 个成分代表了最主要的时间变化模式（如"向左移动"、"逐渐变大"等全局运动方向）。

### 2.4 Stage 3: v1 模式 — 不做后处理

在 v1 模式下，Stage 2 的输出直接作为 η_temporal 使用，不做任何 renormalization。这意味着 η_temporal 的 std 远小于 1.0（约 0.28~0.41），实际注入量为 `√0.004 × 0.3 ≈ 0.019`。

这个"缺陷"反而是优势：极低的注入量使得 z_T 几乎等于纯随机噪声（direction_shift ≈ 0.02），只提供了统计层面的微弱方向偏置，不会破坏 T2V 模型对 N(0,1) 输入的期望。

### 2.5 Noise Blending（噪声混合）

公式：`z_T = √α · η_temporal + √(1-α) · η_random`

其中 α=0.004，代入得 √α ≈ 0.063，√(1-α) ≈ 0.998。考虑到 η_temporal 的 std ≈ 0.3：

- η_temporal 的实际贡献：0.063 × 0.3 ≈ 0.019
- η_random 的实际贡献：0.998 × 1.0 ≈ 0.998

z_T 仍然是一个几乎纯随机的高斯噪声，只带有 ~2% 的方向微偏。这个微偏足以在 30 步去噪过程中引导模型倾向于生成与参考视频类似的运动模式。

### 2.6 为什么 v1 有效

核心发现是：**T2V 模型对 z_T 中的结构化偏离极度敏感**。任何大幅偏离 N(0,1) 的初始噪声都会被 30 步去噪过程逐步放大为 artifact。v1 之所以有效，恰恰因为：

1. η_temporal 的 std 自然衰减到 ~0.3（两次 SVD 截断了大量能量）
2. α=0.004 极小，只贡献 ~2% 的方向偏移
3. 两者叠加使得 z_T 几乎是合法的 N(0,1) 噪声，模型"几乎感觉不到"先验的存在
4. 但这 2% 的统计偏置在 30 步积分中被累积放大，最终体现为生成视频与参考视频在运动模式上的相似性提升

## 三、实验版本对比

### 3.1 总表

| # | 版本 | 配置 | CLIP | X-CLIP | vs Baseline |

- **当前最优 = Pure L2 (Exp 1)**：CLIP 0.8964, XCLIP 0.7874

- **根因**：v7e 精确运动描述与 SVD 方向偏置产生信息冲突

## 四、各版本详解

### 4.1 Exp 0: Baseline — 裸 Caption 直出

做了什么：Qwen2.5-VL-7B 对原始视频生成 caption，不做任何后处理，直接送入 Wan2.1 生成。

结果：CLIP 0.8753, XCLIP 0.7491

存在的问题：VLM 原始 caption 有三个结构性缺陷——(1) 100% 以 "The video depicts/features/captures..." 开头浪费 UMT5 position-0 注意力权重；(2) 72% 尾部是无用的情感总结；(3) 80% 包含 "suggesting/possibly" 等对冲词分散 attention。运动描述保守模糊，只说 "moves forward" 不说具体方向和速度。

### 4.2 Exp 1: Pure L2 — 原始 Caption + SVD v1 ✅ 当前最优

做了什么：保留 VLM 原始 caption 不改动，只在噪声层面注入 SVD v1 运动先验（α=0.004）。

结果：CLIP 0.8964 (+2.4%), XCLIP 0.7874 (+5.1%)

为什么有效：原始 caption 的运动描述模糊（"the boat moves"），T2V 模型不确定"怎么动"。SVD 先验通过 z_T 中 2% 的方向偏置，补充了 prompt 缺失的运动方向信息。两个信号互补而非冲突——prompt 说"什么在动"，SVD 补充"往哪动"。

### 4.3 Exp 2: Pure L1 — v7e Prompt Rewrite

做了什么：用 LLM (v7e 策略) 对 VLM caption 进行结构性改写，不启用任何 SVD 噪声注入。v7e 做 4 处修改：(1) 主体前置到句首；(2) 运动描述加时序连接词 (initially → then → gradually)；(3) 添加一个视觉推断属性；(4) 末尾补镜头语。最重要的是 Motion Fidelity 规则：逐字保留原始运动描述，不推断方向/速度/轨迹。

结果：CLIP 0.8946 (+2.2%), XCLIP 0.7727 (+3.2%)

分析：通过 prompt 结构优化让 UMT5 更好地编码内容（position-0 优势、尾部镜头引导），在两个维度均取得稳定正增益。但 XCLIP (+3.2%) 低于 Pure L2 (+5.1%)，说明 prompt 优化在运动对齐方面不如直接注入运动先验。

### 4.4 Exp 3: L1+L2 — v7e + SVD v1 ❌ 叠加冲突

做了什么：同时使用 v7e 改写 + SVD v1 噪声注入，期望两者正交增益叠加。

结果：CLIP 0.8945 (≈Pure L1), XCLIP 0.7413 (-4.1% vs Pure L1) ❌

为什么失败：v7e 的 Motion Fidelity 规则使 prompt 已精确描述运动方向（如 "drifts forward slowly"），T2V 模型的文本条件通道已编码明确方向。此时 SVD 再注入 ~2% 方向偏置，即使大体一致（cos≈0.98），剩余的 2% 误差也会与文本条件产生对抗。结果是 prompt 说"往北走"，噪声微微偏"北偏东 2°"，两个信号在 30 步去噪中互相拉扯，降低了时序连贯性。

核心教训：**L1 越精确，L2 的容错空间越小**。当 prompt 已经"说清楚了怎么动"，任何非精确匹配的噪声偏置都是干扰。

### 4.5 Exp 4: L1+L2 — v7e + SVD highfreq ❌ 更差

做了什么：假设"全局方向冲突是低频问题"，只注入 SVD 的高频时间成分（局部运动细节），renorm 到 std=1.0。

结果：CLIP 0.8804 (-1.6% vs Pure L1), XCLIP 0.7031 (-9.0% vs Pure L1) ❌❌

为什么更差：高频时间成分的帧间余弦相似度接近随机（~0.05），不携带连贯运动信息，等效于注入额外随机噪声。renorm 将 std 从 ~0.3 强制拉到 1.0，放大了注入量 3 倍（0.019 → 0.063），加剧了干扰。

### 4.6 Exp 5: L1+L2 — v7e + SVD renorm ❌ 同样退化

做了什么：使用完整两阶段 SVD + 强制归一化至 std=1.0（让 η_temporal 成为"正规"的 N(0,1) 噪声）。

结果：XCLIP 约 -4.1% vs Pure L1 ❌

为什么失败：renorm 把注入量从 0.019 放大到 0.063（3x），方向偏移从 2% 增大到 ~6%。T2V 模型对此更敏感，退化程度与 v1 模式相当（方向相同但更"用力"）。

### 4.7 Exp 6: L1-lite+L2 — 简化 Prompt + SVD v1 ⚠️ 部分改善

做了什么：设计 v7e-lite 策略——去掉 temporal chain（不再对运动描述加时序词），保留主体前置、视觉推断、镜头收束。运动描述完全保留 VLM 原文（verbatim copy），不做任何修改。目的是"给 SVD 留出运动信息空间"。

结果：CLIP 0.8842 (-1.2% vs Pure L1), XCLIP 0.7788 (+0.8% vs Pure L1) ⚠️

分析：方向正确但不够——XCLIP 微升说明减少 prompt 的运动精确度确实缓解了冲突，但 CLIP 反降说明 v7e-lite 牺牲了外观对齐能力。综合来看未实现 1+1>2。

## 五、核心发现与结论

### 5.1 SVD 有效的必要条件

SVD 先验起作用需要一个前提：**prompt 的运动信息不足**。

| 场景 | 运动信息来源 | SVD 效果 |
|------|:---:|:---:|
| 原始 VLM caption (模糊) | SVD 补充 | ✅ +5.1% XCLIP |
| V4 prompt (保守改写) | SVD 补充 | ✅ +3.5% XCLIP |
| v7e prompt (精确改写) | prompt 已充足，SVD 冗余 | ❌ -4.1% XCLIP |

结论：SVD 先验本质上是一个"运动信息补充器"——当 prompt 说不清楚怎么动时，它通过 z_T 的微弱方向偏置给模型一个 hint。但当 prompt 已经说清楚了，这个 hint 就变成了噪音。

### 5.2 SVD 模式对比

| SVD 模式 | 对 η_temporal 的处理 | 实际注入量 | 与 v7e 组合效果 |
|----------|------|:---:|:---:|
| v1 (原始) | 不做后处理，std≈0.3 | ~0.019 | -4.1% XCLIP |
| renorm | 强制 std=1.0 | ~0.063 (3x) | -4.1% XCLIP |
| highfreq | 只保留高频 + renorm | ~0.063 | -9.0% XCLIP |
| adaptive | 按 motion_strength 跳过 | 不确定 | -1.5% XCLIP |

所有模式与 v7e 组合均为负向。问题不在 SVD 的具体实现，而在"精确 prompt + 任何噪声偏置"的结构性矛盾。

### 5.3 最终策略

**不再试图让 L1 和 L2 各管运动**。正确的分工是：

- **L1 只做 prompt 结构优化**（主体前置、去废话、补镜头），**不碰运动描述**
- **L2 (SVD v1) 完全负责运动方向引导**

这就是 v8-minimal 策略的设计意图：在 Pure L2 (CLIP 0.8964, XCLIP 0.7874) 的基础上，通过最小化 prompt 微调（不涉及运动）尝试进一步提升 CLIP。

## 六、历史参考数据

V4 是早期较保守的 prompt 策略（3 处修改，运动描述基本照抄原文），验证了 SVD 在保守 prompt 上的正增益：

| 配置 | CLIP | X-CLIP | 说明 |
|------|:----:|:------:|------|
| Baseline | 0.8753 | 0.7491 | 原始 caption |
| V4 L1 only | 0.8794 | 0.7594 | 保守改写 |

| Pure L2 (v1) | 0.8964 | 0.7874 | 当前最优 |

规律：**prompt 对运动的改写越少，SVD 的正向贡献越大**。
