## 一、方法概述

### 1.1 P-Flow 三层架构

P-Flow 采用分层条件注入框架（Hierarchical Conditioning Inversion）：

- **L1（Prompt Rewrite）**：通过 LLM 对 VLM caption 进行结构性改写，优化文本条件以提升 T2V 模型的生成质量
- **L2（SVD Noise Prior）**：对 DDIM 反演噪声进行 SVD 滤波，提取运动先验并注入生成过程的初始噪声
- **L3（Velocity Field Matching）**：速度场匹配，进一步约束生成轨迹（本文档暂不涉及）

理想目标是各层正交增益叠加，即 L1+L2 > max(L1, L2)。

### 1.2 L1：v7e Prompt Rewrite 策略

v7e 是当前最优的 Prompt 改写策略（commit `be25e7c`），核心设计为"4 处手术式修改 + 其余逐字复制"：

**四处修改**：

1. **OPENING**（主体前置）：将动作主体移至句首，利用 UMT5 position-0 注意力权重优势。通过 6 组正反对比示例教会 LLM 区分"动作主体"与"背景/环境"。
2. **TEMPORAL CHAIN**（时序链）：对主体运动描述的 1-2 句添加时序连接词（initially → then → gradually），增强时序连贯性。
3. **ONE VISUAL INFERENCE**（单一视觉推断）：添加一个物理隐含的材质/光照属性（如 "wooden hulls" → "dark brown hulls"），不添加新对象。
4. **CAMERA CLOSING**（镜头收束）：末尾补充镜头描述（如 "The camera remains steady in close-up"），利用 UMT5 尾部 token 权重。

**最高优先级规则 — Motion Fidelity**：
运动描述必须从输入逐字保留，禁止推断运动方向、改变运动动词、改变运动速度、添加未提及的轨迹。这确保 prompt 中的运动信息与原始视频一致。

**约束条件**：output 长度 ≈ input ±15%，不添加原文未提及的信息，5 个带 WHY 标注的 few-shot examples 引导格式。

**脚本路径**：`scripts/rewrite_hybrid.py`（SYSTEM_PROMPT 定义于 lines 64-146）

### 1.3 L2：SVD Noise Prior（运动先验注入）

SVD 滤波器对 DDIM 反演噪声进行处理，提取运动方向信息并注入生成噪声：

**三阶段流水线**：

1. **Stage 1 — Spatial Decontenting**：将噪声 reshape 为 (C×F, H×W)，SVD 分解后去除 top-k_s 空间主成分（保留 ≥10% 能量），消除空间内容信息，保留运动结构。
2. **Stage 2 — Temporal Retention**：将 Stage 1 输出 reshape 为 (C×H×W, F)，SVD 分解后保留 top-k_m 时间主成分（保留 ≥90% 能量），提取时间维度的运动方向。
3. **Stage 3 — Post-processing**：根据模式进行后处理。

**SVD 模式**：

| 模式 | 描述 | 特征 |
|------|------|------|
| v1 | 原始两阶段（spatial + temporal），不做 renorm | η_temporal std ≈ 0.28-0.41，保留低剂量方向偏置 |
| renorm | 两阶段 + 强制归一化至 std=1.0 | 噪声注入量从 ~0.019 放大至 ~0.063 |
| highfreq | 频率分离（高频细节注入）+ renorm | 理论上避免低频全局运动冲突 |
| adaptive | 根据 motion_strength 自动判断是否启用 | motion_strength < 0.15 则 skip |

**噪声混合公式**：`z_init = (1-α) × z_random + α × η_temporal`，其中 α=0.004

**脚本路径**：`src/svd_filter.py`（V2 版本，606 行，4 种模式）

**入口命令**：`scripts/run_hybrid_iter.py --noise_prior --alpha 0.004 --svd_mode <mode>`

## 三、各实验详细记录

### 3.1 Exp 0：Baseline（原始 VLM caption 直出）

**配置**：Qwen2.5-VL-7B 对原始视频生成 caption，不做任何后处理，直接送入 Wan2.1 生成视频。

**结果**：

| 指标 | 均值 |
|------|------|
| orig_gen_clip | **0.8753** |
| orig_gen_xclip | **0.7491** |

**定位**：作为所有实验的参照基准。VLM 原生输出语义完整，但缺少针对 T2V 模型的结构优化。

---

### 3.2 Exp 1：Pure L2（原始 caption + SVD v1）

**配置**：使用原始 VLM caption（不做 LLM 改写），仅启用 SVD v1 模式注入运动先验。

**参数**：`--noise_prior --svd_mode v1 --alpha 0.004 --skip_vlm`（跳过 VLM 重新推理，直接使用已有 caption）

**结果**：

| 指标 | 均值 | vs Baseline |
|------|------|-------------|
| orig_gen_clip | **0.8964** | +0.0211（+2.4%） |
| orig_gen_xclip | **0.7874** | +0.0383（+5.1%） |

**分析**：SVD v1 在保守的原始 caption 上取得显著正向收益。原始 caption 运动描述模糊（如 "the boat moves"），SVD 先验补充了"怎么动"的方向信息，与文本条件形成互补而非冲突。这验证了 SVD 先验的核心价值——为运动信息不足的 prompt 提供方向引导。

---

### 3.3 Exp 2：Pure L1（v7e prompt，无 SVD）

**配置**：使用 v7e LLM 改写策略优化 caption，不启用任何 SVD 噪声注入。

**参数**：不加 `--noise_prior` 标志

**生成目录**：`outputs/hybrid_v7e/generated`

**结果**：

| 指标 | 均值 | vs Baseline |
|------|------|-------------|
| orig_gen_clip | **0.8946** | +0.0193（+2.2%） |
| orig_gen_xclip | **0.7727** | +0.0236（+3.2%） |

**分析**：v7e 改写策略通过 subject-first + temporal chain + motion fidelity 三重约束，在 CLIP 和 X-CLIP 两个维度均取得稳定正向增益。特别是 Motion Fidelity 规则确保了运动描述的准确性，使得 prompt 中包含了精确的运动方向和速度信息。

---

### 3.4 Exp 3：L1+L2（v7e + SVD v1）

**配置**：v7e 改写后的 caption + SVD v1 模式注入。

**参数**：`--noise_prior --svd_mode v1 --alpha 0.004`

**结果**：

| 指标 | 均值 | vs Baseline | vs Pure L1 |
|------|------|-------------|------------|
| orig_gen_clip | **0.8945** | +2.2% | -0.01% |
| orig_gen_xclip | **0.7413** | -1.0% | **-4.1%** |

**分析**：叠加 SVD v1 后，CLIP 基本持平但 X-CLIP 相比 Pure L1 下降 4.1%。这表明 SVD 先验与 v7e 精确运动描述产生了冲突。SVD 的 direction_shift 约 2%，η_temporal std 约 0.28-0.41，虽然注入量不大（α=0.004），但当 prompt 已经精确描述了运动方向时，哪怕微小的方向偏差也会与文本条件产生矛盾信号。

**冲突机制**：v7e 通过 Motion Fidelity 规则保留了如 "drifts forward slowly" 的精确描述，T2V 模型的文本条件通道已经编码了明确的运动方向。SVD 先验通过反演噪声提取的方向可能存在微小误差（~2%），在精确 prompt 场景下，这种误差会与文本条件产生对抗而非协同。

---

### 3.5 Exp 4：L1+L2（v7e + SVD highfreq）

**配置**：v7e 改写 + SVD highfreq 模式（频率分离 + renorm）。理论假设是高频成分只包含运动细节而不含全局方向，可以避免与 prompt 的方向冲突。

**参数**：`--noise_prior --svd_mode highfreq --alpha 0.004`（ρ_s=0.1, ρ_m=0.9）

**结果**：

| 指标 | 均值 | vs Baseline | vs Pure L1 |
|------|------|-------------|------------|
| orig_gen_clip | **0.8804** | +0.6% | **-1.6%** |
| orig_gen_xclip | **0.7031** | -6.1% | **-9.0%** |

**分析**：highfreq 模式表现最差。日志分析显示滤波后帧间余弦相似度接近随机（~0.05），说明高频分量不携带连贯运动方向，注入后等效于向初始噪声中添加了高斯噪声。10 个样本中有 6 个出现 SVD 方向偏好与 prompt 指令的明显冲突。renorm 将 std 强制归一化至 1.0，放大了噪声注入量（从 ~0.019 到 ~0.063），进一步加剧了干扰。

---

### 3.6 Exp 5：L1+L2（v7e + SVD renorm）

**配置**：v7e 改写 + SVD renorm 模式（两阶段 + 强制 std=1.0）。

**参数**：`--noise_prior --svd_mode renorm --alpha 0.004`

**结果**：X-CLIP 相比 Pure L1 下降约 4.1%（具体 CLIP 数值因实验中断未完整记录）。

**分析**：renorm 破坏了时间相关性，将 η_temporal 的 std 从 ~0.3 强制拉伸到 1.0，等效于增大了噪声注入幅度约 3 倍（0.019 → 0.063）。在 v7e 的高质量 prompt 已经将文本-视频对齐推至接近天花板（CLIP ~0.895）的情况下，任何非必要的噪声扰动都会造成退化。

## 四、核心发现

### 4.1 L1 和 L2 独立均有效，但叠加产生冲突

| 对比 | CLIP 增益 | X-CLIP 增益 |
|------|-----------|-------------|
| Pure L2 vs Baseline | +2.4% | +5.1% |
| Pure L1 vs Baseline | +2.2% | +3.2% |
| L1+L2 vs Baseline | +2.2% | -1.0% |

L1 和 L2 单独使用时都能显著超越 baseline，但组合后 X-CLIP 反而低于 baseline。这不是简单的"1+1<2"，而是"1+1<1"——叠加后比单独使用 L1 还差。

### 4.2 冲突根因：信息冗余 + 方向误差

**SVD 有效的前提**是 prompt 的运动信息不足：

- 原始 VLM caption 运动描述保守模糊 → SVD 提供互补信息 → 正向增益（+5.1% X-CLIP）
- v7e prompt 运动描述精确详尽 → SVD 提供冗余/冲突信息 → 负向干扰（-4.1% X-CLIP）

本质上，v7e 的 Motion Fidelity 规则使得 prompt 已经承担了"告诉模型怎么动"的全部职责，SVD 先验试图再次注入运动方向时，即使方向大体一致（~98% cosine similarity），剩余的 2% 误差也会在扩散模型的 ODE 求解过程中被放大，导致生成轨迹偏移。

### 4.3 SVD 模式对比

| SVD 模式 | X-CLIP vs Pure L1 | 问题 |
|----------|:------------------:|------|
| v1（原始） | -4.1% | 低频全局方向与精确 prompt 微冲突 |
| highfreq | -9.0% | 高频无连贯方向，等效随机噪声 |
| renorm | -4.1% | std 放大 3x，破坏时间相关性 |
| adaptive | -1.5% | 大部分样本被 skip，少量启用的样本仍有干扰 |

所有模式在 v7e 场景下均为负向，说明问题不在 SVD 的具体实现方式，而在于"精确 prompt + SVD 先验"这一组合的结构性矛盾。

## 七、结论与下一步方向

当前实验明确揭示了一个核心矛盾：**v7e 的 Motion Fidelity 设计使 prompt 精确到足以独占运动信息通道，SVD 先验因信息冗余反而成为干扰源**。要实现 L1+L2 > L1 的目标，必须打破这一零和博弈，让两层各自负责不重叠的信息维度。

可行的下一步方向见文末"后续实验计划"。
