# 场景感知门控策略设计

> **版本**：v4.0（2026-06-18）
>
> **核心思想**：TSR 单独无法区分"物体运动"和"环境动态"，需要 Prompt 语义修正。通过 **$M_d$（运动明确度）× TSR** 的融合门控，实现场景感知的自适应参数调节。

---

## 一、当前门控系统全景

### 1.1 门控层级总览

```
输入: η_temporal, η_spatial, caption, prompt_embeds
│
├── L2: 噪声空间门控 → 决定 α_eff, β_eff
│   │
│   ├── [已实现] TSR → α_eff（信号侧，已验证 ⚠️ 部分失效）
│   │   α = α_min + TSR × (α_max - α_min)
│   │   问题: 无法区分物体运动 vs 环境动态
│   │
│   ├── [已实现] M_d × TSR 融合门控（Prompt侧 + 信号侧）
│   │   α_eff = max(α_floor × M_d, α_min + f(M_d,TSR) × (α_max - α_min))
│   │   f(M_d,TSR) = M_d × TSR + (1-M_d) × 0.1 × TSR
│   │
│   └── [未实现] SSR → β_eff
│       β = β_max × SSR
│       优先级低: β 效果极度样本相关，暂不门控
│
├── L3: 特征空间门控 → 决定 λ_eff
│   │
│   ├── [已实现] Quality Scale × M_d（样本级）
│   │   QS_eff = QS_signal × M_d  (M_d < 1.0 时压低 QS)
│   │
│   └── [已实现] Adaptive Gate（步级 × 层级）
│       cos(h_gen, h_ref) → gate = 1 - sigmoid(temp*(cos-0.5))
│       特征已对齐→gate小→少注入；偏离→gate大→多注入
│
└── 最终: α_eff, β_eff, λ_eff
    λ_final = λ_schedule[t] × quality_scale_eff × adaptive_gate
```

### 1.2 门控状态汇总

| 门控 | 层级 | 粒度 | 状态 | 输入 | 输出 | 作用 |
|------|:----:|:----:|:----:|------|------|------|
| **TSR → α** | L2 | 样本级 | ✅ 已实现 | η_temporal + S_temporal | TSR ∈ [0,1] | 信号可靠性 → α 大小 |
| **M_d × TSR → α** | L2 | 样本级 | ✅ 已实现 | caption (LLM离线) + TSR | α_eff | 运动语义修正信号可靠性 |
| **α_floor 保底** | L2 | 样本级 | ✅ 已实现 | M_d | α_floor × M_d | M_d确认物体运动时保底注入 |
| **SSR → β** | L2 | 样本级 | ⚪ 未实现 | η_spatial | SSR ∈ [0,1] | 空间信号可靠性 → β |
| **Quality Scale × M_d** | L3 | 样本级 | ✅ 已实现 | η_temporal 帧间 cos + M_d | scale ∈ [0,1.0] | 运动连贯性 + 语义修正 → λ 缩放 |
| **Adaptive Gate** | L3 | 步×层 | ✅ 已实现 | cos(h_gen, h_ref) | gate ∈ [0,1] | 特征对齐度 → λ 调节 |

> **结论**：TSR 单独作为场景感知门控不够，必须引入 Prompt 语义修正。

## 三、M_d（运动明确度）方案

### 3.1 M_d 定义

$M_d$ 回答：**"这个 caption 描述的是物体级运动，还是环境级/场景级动态？"**

$$M_d \in [0, 1]$$

- $M_d = 1$：caption 描述了明确的物体运动 → SVD 运动先验有意义 → α 应大
- $M_d = 0.3$：caption 有相机运动/场景变化 → 谨慎注入
- $M_d = 0$：caption 描述静态场景 → SVD 运动先验可能是噪声 → α 应极小

### 3.2 实现方式

LLM 离线评分 + 运行时查表。脚本 `scripts/compute_md.py`，输出 `data/md_scores.csv`。

### 3.3 LLM 评分 Prompt（v2，精确区分运动来源）

```
Classify the PRIMARY source of motion in this video caption:

1 = No motion: static scene, still life, landscape, architecture — nothing moves.
2 = Camera/scene motion: the VIEWPOINT moves (FPV, drone, tracking shot, zoom,
    flythrough, hyperlapse), or the SCENE transforms (plants growing, objects
    morphing, smoke/steam rising, water flowing, objects deforming). The key signal
    is: no character/subject is locomoting through space under its own power.
3 = Object/subject motion: a person, animal, vehicle, or physical object is actively
    moving through space under its own power (walking, running, flying, swimming,
    chasing, colliding, accelerating).
```

### 3.4 M_d 映射

| LLM 评分 | 含义 | $M_d$ |
|:---------:|------|:-----:|
| 1 | 无运动（静态场景） | 0.0 |
| 2 | 环境/相机动态（FPV、zoom、水流、形变、蒸汽） | 0.3 |
| 3 | 物体运动（人/动物/车辆主动运动） | 1.0 |

### 3.5 M_d 评分质量验证

200 个样本评分分布：md=1(15个), md=2(37个), md=3(148个)

| 判定 | 数量 | 占比 |
|------|:----:|:----:|
| ✅ 正确 | 160 | 80% |
| ⚠️ 可商榷 | 27 | 13.5% |
| ❌ 明确错误 | 13 | 6.5% |

❌ 错误主要集中在：FPV/zoom 类被打 md=1（应为2）、deformation 类被打 md=3（应为2）。这些错误对门控的影响有限——md=1 对 scene 类是保护性的，md=3 对 deformation 类多给了注入但无害。

## 四、TSR × M_d 融合门控

### 4.1 融合公式

$$\alpha_{\text{eff}} = \max\left(\alpha_{\text{floor}} \times M_d, \; \alpha_{\min} + f(M_d, \text{TSR}) \times (\alpha_{\max} - \alpha_{\min})\right)$$

其中 $f(M_d, \text{TSR})$ 是融合函数：

$$f(M_d, \text{TSR}) = M_d \times \text{TSR} + (1 - M_d) \times 0.1 \times \text{TSR}$$

**直觉**：
- $M_d = 1$（物体运动）：$f = \text{TSR}$，完全信任 TSR，且有 α_floor 保底
- $M_d = 0$（静态场景）：$f = 0.1 \times \text{TSR}$，TSR 被大幅压低，α_floor=0 无保底
- $M_d = 0.3$（环境动态）：$f = 0.37 \times \text{TSR}$，谨慎注入

### 4.2 典型案例推演

| ID | M_d | TSR | f(M_d,TSR) | α_fusion | α_floor | **α_eff** | 纯TSR的α |
|:--:|:---:|:---:|:-----------:|:--------:|:-------:|:---------:|:--------:|
| 32 | 1.0 | 0.151 | 0.151 | 0.0006 | 0.0020 | **0.0020** | 0.0006 |
| 33 | 1.0 | 0.337 | 0.337 | 0.0013 | 0.0020 | **0.0020** | 0.0013 |
| 7 | 1.0 | 0.575 | 0.575 | 0.0023 | 0.0020 | **0.0023** | 0.0023 |
| 73 | 0.3 | 0.330 | 0.122 | 0.0005 | 0.0006 | **0.0006** | 0.0013 |
| 80 | 0.0 | 0.570 | 0.057 | 0.0002 | 0.0000 | **0.0002** | 0.0023 |
| 72 | 0.0 | 0.041 | 0.004 | 0.0000 | 0.0000 | **0.0000** | 0.0002 |

**改善**：
- 80号（蒲公英）：α 从 0.0023 → 0.0002 ✅ 环境动态有害注入被消除
- 73号（仓库）：α 从 0.0013 → 0.0006 ✅ 场景变化注入被压低
- 32/33号：α_floor 保底 0.002 ✅ 间歇运动不再被 TSR 低估

## 六、门控策略修正建议

### 6.1 当前融合公式的偏差

| 场景 | M_d | TSR预期 | 当前α_eff | 实验最优α | 问题 |
|------|:---:|:-------:|:---------:|:---------:|------|
| collision | 1.0 | 中→高 | 0.002~0.004 | 0.002~0.004 | ✅ 吻合 |
| deformation | 1.0→**0.3** | 中 | 0.0006~0.001 | 0.001~0.002 | ⚠️ M_d应为0.3 |
| gravity | 1.0 | 中 | 0.002 | 样本相关 | ⚠️ 无法统一 |
| thermo | 0.3 | 低→中 | 0.0002~0.0005 | **0.004** | ❌ 严重偏低！ |
| emotion | 1.0 | 低 | 0.002(保底) | **0.001** | ❌ α_floor太高！ |

### 6.2 发现的关键问题

**问题 1：热力学类 M_d=0.3 导致 α 被过度压低**

热力学（蒸发、溅射）被 LLM 评为 md=2（环境动态），M_d=0.3 → α 被压到 ~0.0002，但实验显示 α=0.004 效果最好。原因：虽然蒸汽不是"物体运动"，但 SVD 能有效传递热力学变化的时序模式，需要强注入。

**问题 2：情感类 M_d=1.0 + α_floor=0.002 导致注入过量**

情感类（面部微表情）被 LLM 评为 md=3（物体运动），M_d=1.0 → α_floor=0.002 保底生效，但实验显示 α=0.001 最好。原因：微表情的"运动"幅度极小，SVD 提取的运动信号实际上是噪声。

### 6.3 修正方案

#### 方案 A：调整 M_d 映射（不动公式，只调查表）

修正 LLM 对特殊场景的误判：

| 场景 | 当前 M_d | 修正 M_d | 理由 |
|------|:--------:|:--------:|------|
| physics-thermodynamics | 0.3 | **0.7** | 虽非物体运动，但 SVD 注入极有效 |
| physics-deformation | 1.0 | **0.3** | 原位形变，α 不宜过大 |
| human-emotion | 1.0 | **0.1** | 微弱运动，应极低注入 |

修正方式：在 `data/md_scores.csv` 中手动覆盖这些样本的 M_d 值。

#### 方案 B：降低 α_floor（最简单）

当前 α_floor=0.002 对 emotion 类太高。改为 α_floor=0.001：

| 场景 | M_d | α_floor×M_d | 修正后α_eff |
|------|:---:|:-----------:|:-----------:|
| emotion | 1.0 | 0.001 | 0.001 ✅ |
| collision | 1.0 | 0.001 | 0.001+TSR → ~0.002~0.004 ✅ |
| deformation | 0.3 | 0.0003 | 0.0003 ✅ |

但 thermo 的 M_d=0.3 × α_floor=0.001 = 0.0003，仍远低于最优 0.004 ❌

#### 方案 C（推荐）：方案 A + 方案 B 组合

1. **α_floor 降为 0.001**
2. **手动修正 M_d**：thermo 从 0.3 → 0.7，deformation 从 1.0 → 0.3，emotion 从 1.0 → 0.1

| 场景 | 修正M_d | α_floor | TSR | α_eff | 实验最优 | 匹配 |
|------|:-------:|:-------:|:---:|:-----:|:-------:|:----:|
| collision | 1.0 | 0.001 | 0.5 | ~0.002 | 0.002~0.004 | ✅ |
| deformation | 0.3 | 0.0003 | 0.3 | ~0.0004 | 0.001~0.002 | ⚠️ |
| gravity | 1.0 | 0.001 | 0.4 | ~0.0016 | 样本相关 | 🟡 |
| thermo | 0.7 | 0.0007 | 0.5 | ~0.002 | 0.004 | ⚠️ |
| emotion | 0.1 | 0.0001 | 0.2 | ~0.0002 | 0.001 | ⚠️ |

> deformation 和 thermo 仍偏低，说明 **α_max=0.004 的上限可能需要提高**，或者这两类场景需要独立的注入参数。

### 6.4 最终推荐参数

```python
# 默认参数
alpha_floor = 0.001       # 从 0.002 降低，保护 emotion 类
alpha_max = 0.006         # 从 0.004 提高，让 thermo/gravity 有空间
alpha_min = 0.0

# M_d 手动修正（在 md_scores.csv 中覆盖）
# thermo:   M_d = 0.7  (SVD 对热力学动态极有效)
# deform:   M_d = 0.3  (原位形变不需要太多运动注入)
# emotion:  M_d = 0.1  (微弱运动应极低注入)
```

推演修正后效果：

| 场景 | M_d | α_floor | f(M_d,TSR) | α_fusion | α_eff(α_max=0.006) | 最优α |
|------|:---:|:-------:|:----------:|:--------:|:------------------:|:-----:|
| collision | 1.0 | 0.001 | 0.5 | 0.003 | **0.003** | 0.002~0.004 |
| deformation | 0.3 | 0.0003 | 0.11 | 0.0007 | **0.0007** | 0.001~0.002 |
| gravity | 1.0 | 0.001 | 0.4 | 0.0024 | **0.0024** | 样本相关 |
| thermo | 0.7 | 0.0007 | 0.37 | 0.0022 | **0.0022** | 0.004 |
| emotion | 0.1 | 0.0001 | 0.02 | 0.0001 | **0.0001** | 0.001 |

> thermo 仍然偏低（0.0022 vs 最优 0.004），emotion 也偏低（0.0001 vs 最优 0.001）。
> 这表明 **融合公式对极端场景的适应性有限**，可能需要场景特定的 α_max 或直接场景查表。

## 七、L3 门控：Quality Scale 与 Adaptive Gate

### 7.1 Quality Scale × M_d（已实现）

```python
quality_scale_eff = quality_scale_signal * M_d  # M_d < 1.0 时压低 QS
```

scene 类样本 QS 被压低 → FI λ 被压低 → 避免有害特征注入。

### 7.2 Adaptive Gate（已实现，保持不变）

```python
gate = 1 - sigmoid(temp * (cos(h_gen, h_ref) - 0.5))  # temp=5.0
λ_final = λ_schedule[t] × quality_scale_eff × gate
```

## 十、关键洞察

1. **"运动明确度"不等于"注入量"**：M_d 度量的是运动语义类型，但某些环境动态（热力学）反而需要强注入。M_d 的真正作用是**区分 SVD 信号是否可信**，而非直接决定注入量。

2. **TSR × M_d 融合对常规场景有效，对极端场景不足**：碰撞/动物/人活动等"物体运动"场景，融合门控工作良好。但热力学和情感类需要**场景特定的参数覆盖**。

3. **最简方案**：先跑 Phase 5 验证实验，如果 10 个已知样本指标提升，说明融合门控有效。极端场景的修正可以在全量验证时通过手动 M_d 覆盖解决。
