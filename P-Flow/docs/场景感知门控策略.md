# 场景感知门控策略设计

> **版本**：v5.0（2026-06-19）
>
> **核心思想**：TSR 单独无法区分"物体运动"和"环境动态"，需要 Prompt 语义修正。通过 **$M_d$（运动明确度）× TSR** 的融合门控，实现场景感知的自适应参数调节。

---

## 一、当前门控系统全景

### 1.1 门控层级总览

```
输入: η_temporal, η_spatial, caption, prompt_embeds
│
├── L2: 噪声空间门控 → 决定 α_eff, β_eff
│   │
│   ├── [已实现] TSR → α_eff（信号侧，已验证 ⚠️ 部分失效）
│   │   α = α_min + TSR × (α_max - α_min)
│   │   问题: 无法区分物体运动 vs 环境动态
│   │
│   ├── [已实现] M_d × TSR 融合门控（Prompt侧 + 信号侧）
│   │   f(M_d, TSR) = M_d × TSR + (1-M_d) × 0.1 × TSR
│   │   α_fusion = α_min + f(M_d,TSR) × (α_max - α_min)
│   │
│   ├── [已实现] α_floor 保底 (含 alpha_md_floor)
│   │   md_for_alpha = max(M_d, alpha_md_floor)  ← 防止M_d=0时α归零
│   │   α_floor_val = alpha_floor × md_for_alpha
│   │   α_eff = max(α_floor_val, α_fusion)
│   │
│   └── [未实现] SSR → β_eff
│       β = β_max × SSR
│       优先级低: β 效果极度样本相关，暂不门控
│
├── L3: 特征空间门控 → 决定 λ_eff
│   │
│   ├── [已实现] Quality Scale × M_d（样本级，含 fi_qs_md_floor 保底）
│   │   md_for_qs = max(M_d, fi_qs_md_floor)  ← 防止M_d=0时FI完全关闭
│   │   QS_eff = QS_signal × md_for_qs
│   │
│   └── [已实现] Adaptive Gate（步级 × 层级）
│       cos(h_gen, h_ref) → gate = 1 - sigmoid(temp*(cos-0.5))
│       特征已对齐→gate小→少注入；偏离→gate大→多注入
│
└── 最终: α_eff, β_eff, λ_eff
    λ_final = λ_schedule[t] × QS_eff × adaptive_gate
```

### 1.2 门控状态汇总

| 门控 | 层级 | 粒度 | 状态 | 输入 | 输出 | 作用 |
|------|:----:|:----:|:----:|------|------|------|
| **TSR → α** | L2 | 样本级 | ✅ 已验证 | η_temporal + S_temporal | TSR ∈ [0,1] | 信号可靠性 → α 大小 |
| **M_d × TSR → α** | L2 | 样本级 | ✅ 已验证 | caption (LLM离线) + TSR | α_fusion | 运动语义修正信号可靠性 |
| **α_floor 保底** | L2 | 样本级 | ✅ 已验证 | M_d + alpha_md_floor | α_floor × max(M_d, 0.3) | M_d确认物体运动时保底注入 |
| **alpha_md_floor** | L2 | 样本级 | ✅ 已验证 | M_d | max(M_d, 0.3) | 防止M_d=0时α完全归零 |
| **SSR → β** | L2 | 样本级 | ⚪ 未实现 | η_spatial | SSR ∈ [0,1] | 空间信号可靠性 → β |
| **Quality Scale × M_d** | L3 | 样本级 | ✅ 已验证 | η_temporal 帧间 cos + M_d + fi_qs_md_floor | scale ∈ [0,1.0] | 运动连贯性 + 语义修正 → λ 缩放 |
| **fi_qs_md_floor** | L3 | 样本级 | ✅ 已验证 | M_d | max(M_d, 0.5) | 防止M_d=0时FI被完全关闭 |
| **Adaptive Gate** | L3 | 步×层 | ✅ 已验证 | cos(h_gen, h_ref) | gate ∈ [0,1] | 特征对齐度 → λ 调节 |

### 1.3 当前默认参数（v3.3）

```python
alpha_floor = 0.003       # M_d=1.0 保底 α=0.003
alpha_max = 0.006          # 高 TSR+M_d=1.0 时 α_fusion 上限
alpha_min = 0.0
alpha_md_floor = 0.3      # α 保底中 M_d 的下限
fi_qs_md_floor = 0.5       # FI QS 中 M_d 的下限
fi_lambda = 0.05           # FI 注入强度
fi_adaptive_temp = 5.0     # Adaptive Gate 温度
```

### 2.2 TSR 失效分析

**TSR 度量的是"SVD 时间信号的统计可靠性"，而不是"视频里有没有物体运动"。**

| 预期 | 实际 | 判定 |
|------|------|:----:|
| animal(32) TSR 应高 | TSR=0.151（低） | ❌ 间歇运动→TAC低→TSR低 |
| animal(43) TSR 应高 | TSR=0.000（极低） | ❌ 同上 |
| scene(72) TSR 应低 | TSR=0.041（确实低） | ✅ |
| scene(73) TSR 应低 | TSR=0.330（中等） | ❌ 有持续微弱动态 |
| scene(80) TSR 应低 | TSR=0.570（最高！） | ❌ 蒲公英飘动→TAC高→TSR高 |

> **结论**：TSR 单独作为场景感知门控不够，必须引入 Prompt 语义修正。

## 三、M_d（运动明确度）方案

### 3.1 M_d 定义

$M_d$ 回答：**"这个 caption 描述的是物体级运动，还是环境级/场景级动态？"**

$$M_d \in [0, 1]$$

- $M_d = 1$：caption 描述了明确的物体运动 → SVD 运动先验有意义 → α 应大
- $M_d = 0.3$：caption 有相机运动/场景变化 → 谨慎注入
- $M_d = 0$：caption 描述静态场景 → SVD 运动先验可能是噪声 → α 应极小

### 3.2 实现方式

LLM 离线评分 + 运行时查表。脚本 `scripts/compute_md.py`，输出 `data/md_scores.csv`。

### 3.3 LLM 评分 Prompt（v2，精确区分运动来源）

```
Classify the PRIMARY source of motion in this video caption:

1 = No motion: static scene, still life, landscape, architecture — nothing moves.
2 = Camera/scene motion: the VIEWPOINT moves (FPV, drone, tracking shot, zoom,
    flythrough, hyperlapse), or the SCENE transforms (plants growing, objects
    morphing, smoke/steam rising, water flowing, objects deforming). The key signal
    is: no character/subject is locomoting through space under its own power.
3 = Object/subject motion: a person, animal, vehicle, or physical object is actively
    moving through space under its own power (walking, running, flying, swimming,
    chasing, colliding, accelerating).
```

### 3.4 M_d 映射

| LLM 评分 | 含义 | $M_d$ |
|:---------:|------|:-----:|
| 1 | 无运动（静态场景） | 0.0 |
| 2 | 环境/相机动态（FPV、zoom、水流、形变、蒸汽） | 0.3 |
| 3 | 物体运动（人/动物/车辆主动运动） | 1.0 |

### 3.5 M_d 评分质量验证

200 个样本评分分布：md=1(15个), md=2(37个), md=3(148个)

| 判定 | 数量 | 占比 |
|------|:----:|:----:|
| ✅ 正确 | 160 | 80% |
| ⚠️ 可商榷 | 27 | 13.5% |
| ❌ 明确错误 | 13 | 6.5% |

❌ 错误主要集中在：FPV/zoom 类被打 md=1（应为2）、deformation 类被打 md=3（应为2）。这些错误对门控的影响有限——md=1 对 scene 类是保护性的，md=3 对 deformation 类多给了注入但无害。

## 四、TSR × M_d 融合门控

### 4.1 融合公式（v3.3 最终版）

$$\alpha_{\text{eff}} = \max\left(\alpha_{\text{floor}} \times \max(M_d, \alpha_{\text{md\_floor}}), \; \alpha_{\min} + f(M_d, \text{TSR}) \times (\alpha_{\max} - \alpha_{\min})\right)$$

其中 $f(M_d, \text{TSR})$ 是融合函数：

$$f(M_d, \text{TSR}) = M_d \times \text{TSR} + (1 - M_d) \times 0.1 \times \text{TSR}$$

**直觉**：
- $M_d = 1$（物体运动）：$f = \text{TSR}$，完全信任 TSR，且有 α_floor 保底
- $M_d = 0$（静态场景）：$f = 0.1 \times \text{TSR}$，TSR 被大幅压低，但 alpha_md_floor 保证微量注入
- $M_d = 0.3$（环境动态）：$f = 0.37 \times \text{TSR}$，谨慎注入

### 4.2 L3 FI 门控公式（v3.3 最终版）

$$\text{QS}_{\text{eff}} = \text{QS}_{\text{signal}} \times \max(M_d, \text{fi\_qs\_md\_floor})$$

$$\lambda_{\text{final}} = \lambda_{\text{schedule}}[t] \times \text{QS}_{\text{eff}} \times \text{gate}$$

### 4.3 保底机制三层设计

| 保底 | 参数 | 默认值 | 作用 |
|------|------|:------:|------|
| **α_floor** | alpha_floor | 0.003 | M_d=1.0 时 α 保底 0.003 |
| **alpha_md_floor** | alpha_md_floor | 0.3 | M_d=0 时 md_for_alpha=0.3，α 保底=0.0009 |
| **fi_qs_md_floor** | fi_qs_md_floor | 0.5 | M_d=0 时 FI QS 不低于 50% |

### 7.1 Quality Scale × M_d（已实现，含保底）

```python
fi_qs_floor = getattr(cfg, 'fi_qs_md_floor', 0.5)  # v3.3 默认0.5
md_for_qs = max(M_d, fi_qs_floor)                    # 防止M_d=0时FI完全关闭
quality_scale = quality_scale * md_for_qs
```

### 7.2 Adaptive Gate（已实现，保持不变）

```python
gate = 1 - sigmoid(temp * (cos(h_gen, h_ref) - 0.5))  # temp=5.0
λ_final = λ_schedule[t] × quality_scale_eff × gate
```

## 十、关键洞察

1. **"运动明确度"不等于"注入量"**：M_d 度量的是运动语义类型，但某些环境动态（热力学）反而需要强注入。M_d 的真正作用是**区分 SVD 时间信号是否可信**，而非直接决定注入量。

2. **TSR × M_d 融合对常规场景有效，对极端场景不足**：碰撞/动物/人活动等"物体运动"场景，融合门控工作良好。但热力学和情感类需要**场景特定的参数覆盖**。

3. **M_d=0 不等于"注入有害"**：72号（静态场景）旧版 α=0.004 有正增益（X-CLIP=+0.120），因为 SVD 空间信息碰巧有用。这不是时间运动信号的贡献，而是空间信息的贡献，属于 β 门控的范畴。

4. **门控核心价值是场景自动适配**：不需要每个 case 都超过旧版。80号（蒲公英）从 -0.101 改善到 -0.073、31号从 -0.053 变为 +0.109，证明门控在关键场景上有保护作用。

5. **三层保底机制缺一不可**：
   - `alpha_floor`：保证物体运动样本最低注入
   - `alpha_md_floor`：防止 M_d=0 时 α 完全归零
   - `fi_qs_md_floor`：防止 M_d=0 时 FI 完全关闭
