# 场景感知门控策略设计

> **版本**：v3.0（2026-06-18）
>
> **核心思想**：TSR 单独无法区分"物体运动"和"环境动态"，需要 Prompt 语义修正。通过 **$M_d$（运动明确度）× TSR** 的融合门控，实现场景感知的自适应参数调节。

---

## 一、当前门控系统全景

### 1.1 门控层级总览

```
输入: η_temporal, η_spatial, caption, prompt_embeds
│
├── L2: 噪声空间门控 → 决定 α_eff, β_eff
│   │
│   ├── [已实现] TSR → α_eff（信号侧，已验证 ⚠️ 部分失效）
│   │   α = α_min + TSR × (α_max - α_min)
│   │   问题: 无法区分物体运动 vs 环境动态
│   │
│   ├── [待实现] M_d → α_eff 修正（Prompt侧）
│   │   LLM 离线评分，运行时查表
│   │   解决: "这个 caption 描述的是物体级运动吗？"
│   │
│   └── [未实现] SSR → β_eff
│       β = β_max × SSR
│       优先级低: β 效果极度样本相关，暂不门控
│
├── L3: 特征空间门控 → 决定 λ_eff
│   │
│   ├── [已实现] Quality Scale（样本级）
│   │   基于 η_temporal 帧间 mean_cos → sigmoid → scale ∈ [0.1, 1.0]
│   │
│   └── [已实现] Adaptive Gate（步级 × 层级）
│       cos(h_gen, h_ref) → gate = 1 - sigmoid(temp*(cos-0.5))
│       特征已对齐→gate小→少注入；偏离→gate大→多注入
│
└── 最终: α_eff, β_eff, λ_eff
    λ_final = λ_schedule[t] × quality_scale × adaptive_gate
```

### 1.2 门控状态汇总

| 门控 | 层级 | 粒度 | 状态 | 输入 | 输出 | 作用 |
|------|:----:|:----:|:----:|------|------|------|
| **TSR → α** | L2 | 样本级 | ✅ 已实现，⚠️ 部分失效 | η_temporal + S_temporal | TSR ∈ [0,1] | 信号可靠性 → α 大小 |
| **M_d → α 修正** | L2 | 样本级 | 🔴 待实现 | caption (LLM 离线评分) | M_d ∈ [0,1] | 运动语义 → α 修正 |
| **SSR → β** | L2 | 样本级 | ⚪ 未实现 | η_spatial | SSR ∈ [0,1] | 空间信号可靠性 → β |
| **Quality Scale** | L3 | 样本级 | ✅ 已实现 | η_temporal 帧间 cos | scale ∈ [0.1,1.0] | 运动连贯性 → λ 缩放 |
| **Adaptive Gate** | L3 | 步×层 | ✅ 已实现 | cos(h_gen, h_ref) | gate ∈ [0,1] | 特征对齐度 → λ 调节 |

### 2.3 TSR 失效分析

**TSR 度量的是"SVD 时间信号的统计可靠性"，而不是"视频里有没有物体运动"。**

| 预期 | 实际 | 判定 |
|------|------|:----:|
| animal(32) TSR 应高 | TSR=0.151（低） | ❌ 间歇运动→TAC低→TSR低 |
| animal(43) TSR 应高 | TSR=0.000（极低） | ❌ 同上 |
| scene(72) TSR 应低 | TSR=0.041（确实低） | ✅ |
| scene(73) TSR 应低 | TSR=0.330（中等） | ❌ 有持续微弱动态 |
| scene(80) TSR 应低 | TSR=0.570（最高！） | ❌ 蒲公英飘动→TAC高→TSR高 |

**根因**：持续性微弱动态（微风、水纹、光线变化）TAC 高 → TSR 高，但这些**不是物体级运动**，SVD 注入反而有害。间歇性物体运动 TAC 低 → TSR 低 → α 反而被压小，但 SVD 其实是有用的。

### 2.4 结论

> **TSR 单独作为场景感知门控是不够的。必须引入 Prompt 语义信息修正。**

## 三、M_d（运动明确度）方案设计

### 3.1 M_d 的定义

$M_d$ 回答的问题：**"这个 caption 描述的是物体级运动，还是环境级/场景级动态？"**

$$M_d \in [0, 1]$$

- $M_d = 1$：caption 描述了明确的物体运动 → SVD 运动先验有意义 → α 应大
- $M_d = 0.5$：caption 有运动但不确定是物体级 → 谨慎注入
- $M_d = 0$：caption 描述静态场景/环境动态 → SVD 运动先验可能是噪声 → α 应极小或 0

### 3.2 实现方式：LLM 离线评分 + 运行时查表

**为什么不用词表/规则？**
- 词表匹配是 heuristic hack，不学术
- 无法处理语义歧义（"flowing" 可以是水流也可以是裙子飘动）

**为什么不用在线 LLM？**
- 需要额外加载模型，浪费显存
- $M_d$ 只依赖 caption 文本，可以预计算

**方案**：独立脚本用 LLM 评分，存 CSV，生成时查表。零运行时开销。

### 3.3 LLM 评分 Prompt

```
Classify the type of motion described in this video caption:

1 = No motion (static scene, landscape, still life, architecture, no movement)
2 = Ambient motion only (wind, water ripples, lighting changes, camera movement, subtle drift)
3 = Object motion present (people, animals, vehicles, or objects actively moving with clear physical displacement)

Caption: {caption}

Answer with ONLY 1, 2, or 3.
```

### 3.4 M_d 映射

| LLM 评分 | 含义 | $M_d$ |
|:---------:|------|:-----:|
| 1 | 无运动（静态场景） | 0.0 |
| 2 | 环境动态（微风、水纹、镜头运动） | 0.3 |
| 3 | 物体运动（人/动物/车辆明显运动） | 1.0 |

| 32 | animal | puppies playing, pop out | 3 | 1.0 | ✅ 正增益 |

## 四、TSR × M_d 融合门控

### 4.1 融合公式

$$\alpha_{\text{eff}} = \alpha_{\min} + f(M_d, \text{TSR}) \times (\alpha_{\max} - \alpha_{\min})$$

其中 $f(M_d, \text{TSR})$ 是融合函数：

$$f(M_d, \text{TSR}) = M_d \times \text{TSR} + (1 - M_d) \times 0.1 \times \text{TSR}$$

**直觉**：
- $M_d = 1$（物体运动）：$f = \text{TSR}$，完全信任 TSR
- $M_d = 0$（静态场景）：$f = 0.1 \times \text{TSR}$，TSR 被大幅压低
- $M_d = 0.3$（环境动态）：$f = 0.3 \times \text{TSR} + 0.7 \times 0.1 \times \text{TSR} = 0.37 \times \text{TSR}$

### 4.2 典型案例推演

| ID | M_d | TSR | f(M_d,TSR) | α_eff | 之前纯TSR的α | 已知最优α |
|:--:|:---:|:---:|:-----------:|:-----:|:-----------:|:---------:|
| 32 | 1.0 | 0.151 | 0.151 | 0.0006 | 0.0006 | 0.003~0.004 |
| 33 | 1.0 | 0.337 | 0.337 | 0.0013 | 0.0013 | 0.003~0.004 |
| 7 | 1.0 | 0.575 | 0.575 | 0.0023 | 0.0023 | 0.004 |
| 73 | 0.0 | 0.330 | 0.033 | 0.0001 | 0.0013 | 0.001 |
| 80 | 0.3 | 0.570 | 0.211 | 0.0008 | 0.0023 | 0.002 |
| 72 | 0.3 | 0.041 | 0.015 | 0.0001 | 0.0002 | — |
| 43 | 1.0 | 0.000 | 0.000 | 0.0000 | 0.0000 | — |

**关键改善**：
- **80号**（蒲公英）：α 从 0.0023 降到 0.0008 → 环境动态不再被当作可靠运动信号 ✅
- **73号**（仓库）：α 从 0.0013 降到 0.0001 → 静态场景几乎关闭注入 ✅

**仍存在的问题**：
- **32号、33号**：即使 M_d=1.0，α_eff 仍然偏小（0.0006、0.0013），因为 TSR 本身就低
  - 这反映了 TSR 对间歇性运动的低估
  - 可能需要额外机制：当 M_d=1.0 但 TSR 低时，给一个保底 α_floor

### 4.3 保底 α_floor 机制

当 M_d 确认了物体运动存在，但 TSR 因为间歇性运动而偏低时：

$$\alpha_{\text{eff}} = \max\left(\alpha_{\text{floor}} \times M_d, \; \alpha_{\min} + f(M_d, \text{TSR}) \times (\alpha_{\max} - \alpha_{\min})\right)$$

其中 $\alpha_{\text{floor}} = 0.002$（M_d 确认物体运动时的保底注入量）。

| ID | M_d | TSR | f | α_fusion | α_floor | **α_eff** | 已知最优α |
|:--:|:---:|:---:|:---:|:---------:|:-------:|:---------:|:---------:|
| 32 | 1.0 | 0.151 | 0.151 | 0.0006 | 0.0020 | **0.0020** | 0.003~0.004 |
| 33 | 1.0 | 0.337 | 0.337 | 0.0013 | 0.0020 | **0.0020** | 0.003~0.004 |
| 73 | 0.0 | 0.330 | 0.033 | 0.0001 | 0.0000 | **0.0001** | 0.001 |
| 80 | 0.3 | 0.570 | 0.211 | 0.0008 | 0.0006 | **0.0008** | 0.002 |

> α_floor 机制显著改善了物体运动型样本，同时不影响场景型样本的保护。

## 五、L3 门控：Quality Scale 与 Adaptive Gate

### 5.1 Quality Scale（已实现，保持不变）

基于 η_temporal 帧间余弦相似度的 sigmoid 映射：

```python
mean_cos = mean(cos(frame_f, frame_{f+1}))
scale = 0.1 + 0.9 * sigmoid(20.0 * (mean_cos - 0.05))
```

| mean_cos | scale | 含义 |
|:--------:|:-----:|------|
| >0.10 | ≈1.0 | ✅ 强引导 |
| ≈0.05 | ≈0.5 | ⚡ 中等引导 |
| <0.00 | ≈0.1 | ⚠️ 微弱引导 |

### 5.2 Adaptive Gate（已实现，保持不变）

```python
gate = 1 - sigmoid(temp * (cos(h_gen, h_ref) - 0.5))  # temp=5.0
λ_eff = λ_schedule[t] × quality_scale × gate
```

### 5.3 M_d 对 Quality Scale 的潜在修正（待验证）

**假设**：scene 类样本的 quality_scale 可能也偏高了（M_d 低 → 应降权）。

当前不做修改，等融合门控验证后再决定是否需要 M_d 修正 quality_scale。
