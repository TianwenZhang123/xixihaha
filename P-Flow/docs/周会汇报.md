# 上篇：方法与原理

## 1. 通俗故事 — "让 AI 精准重画任意视频"

### 一句话版本

给 AI 一段视频，让它仅凭「文字 + 噪声」重新生成一个尽可能一模一样的版本。我们的方法是一个渐进式框架，通过逐层添加信息（文字→噪声→嵌入），一步步逼近完美复现。

### 完整故事

想象你面前有一位画师（Wan2.1 文生视频模型），他只接受两样东西：一段文字描述和一张满是涂鸦的白纸（初始噪声）。你的目标是：让他画出和你手中参考视频完全一致的作品。

问题在于，一段文字远远不够。你说「一只橘猫在窗台打哈欠」，画师每次画出来的猫都不同——朝向、动作幅度、背景光线全凭自由发挥。文字只是一个模糊的意图传达。

**我们的解法是：用 flag 组合逐层逼近目标。**

**第一层（L1 — Prompt 优化）**：先让 VLM 看参考视频，写一段详尽描述，然后反复对比生成结果和原视频，迭代优化措辞。相当于给画师一份越来越精确的"剧本"。

**第二层（L2 — Noise Prior）**：文字到了极限后，画师的"起笔"就成了关键。我们从参考视频"倒推"出一张特殊的噪声图，再用 SVD 分离出纯运动成分，最后与随机噪声混合。就像给画师一张"带运动暗示的草稿纸"。

**第三层（L3 — Velocity Matching）**：文字和噪声都搞定后，还剩那些文字无法表达的隐含动态——特定的加速曲线、微妙的节奏感。我们通过优化一个 embedding 残差 Δe，让模型的"画笔运动方向"精确对齐参考视频定义的理想轨迹。

三层组合就像一个分辨率递增的编码系统：文字给出语义轮廓（"什么在动"），噪声先验提供结构引导（"从哪里开始动"），Δe 锁定精确轨迹（"怎么动"）。每一层都是前一层的残差补充，互不冗余。

## 2. 三层架构详解

### 2.1 Layer 1 — Hybrid Prompt Rewrite（文字优化）

**解决的问题**：纯文字描述的信息带宽有限，VLM 写出的 caption 偏"描述性"而非"指导性"，直接用于生成效果有限。

**方法**：VLM 观察参考视频 → 写出详尽描述 → LLM 改写为"指导性 prompt"（Hybrid 策略） → 位置优化（关键动作描述前置）。迭代多轮可进一步逼近文字极限。

**直觉**：从"一只猫在桌上跳下"到"A ginger cat crouches on a wooden desk, suddenly springs upward with all four paws leaving the surface simultaneously, arcs through the air, and lands softly before darting out of frame"——信息密度的提升直接转化为生成质量。

**关键参数**：使用 V4 iter1 最佳 prompt（`hybrid_iter_v4/captions_iter0`）。

**贡献**：CLIP +1.6%, XCLIP +3.7%（vs Baseline）。

---

### 2.2 Layer 2 — SVD Noise Prior（噪声先验）

**解决的问题**：文字到达极限后，初始噪声的随机性成为生成结果不可控的主要来源。需要为生成过程提供一个"方向性偏置"。

**通俗理解**：视频生成模型的工作方式是"从一团噪声出发，逐步去噪变成清晰画面"。如果噪声是完全随机的，模型每次走向的终点（生成结果）就不可控。我们的想法是：既然参考视频已经存在，能不能从它身上"偷"一点信息，藏到初始噪声里，让模型起步时就朝着正确方向走？

但直接把参考视频"倒推"回噪声空间再用，等于告诉模型答案——那就不是"重建"而是"复制"了。所以关键在于：只偷"运动方向"（猫往哪跳、手怎么挥），不偷"外观细节"（猫长什么样、背景是什么颜色）。

**方法**：三步走——

**Step 1 — Flow Matching Inversion（倒推噪声）**：把参考视频从"成品"倒退回"起点噪声"。好比拿到一幅画，反推画师最初的草稿纸长什么样。

数学上，这是沿 ODE 反向积分：

```
η_inv = z₀ + ∫₁⁰ v_θ(x_t, t, e₀) dt    (从视频 latent z₀ 出发，逆向走回噪声空间)
```

**Step 2 — SVD 两阶段滤波（手术刀分离）**：η_inv 里包含了参考视频的所有信息——外观和运动混在一起。我们用 SVD（奇异值分解）当作"频谱手术刀"，做两刀：

- **第一刀（空间维度）**：把数据按像素位置展开，做 SVD。大奇异值对应的是"全局外观"（整体形状、颜色分布），小奇异值对应"局部运动"。截掉前 10% 的大奇异值 → 去除外观。
- **第二刀（时间维度）**：把数据按帧展开，做 SVD。大奇异值对应"主要运动模式"（主体位移轨迹），小奇异值是噪声。保留前 90% → 保留运动结构。

```
Stage 1 (Spatial): reshape → (C·F, H·W), SVD, 截断前 ρ_s=0.1 奇异值 → 去掉外观
Stage 2 (Temporal): reshape → (C·H·W, F), SVD, 保留前 ρ_m=0.9 奇异值 → 留住运动
→ η_temporal（纯运动噪声：知道"往哪动"，不知道"长什么样"）
```

**Step 3 — Noise Blending（极低比例混合）**：将这份"运动暗示"以极低比例（α=0.004，信号权重仅 √α≈6.3%）混入随机噪声。就像往一杯白水里滴了一滴墨水——水还是基本透明的，但已经有了微妙的方向感。

```
η = √α · η_temporal + √(1-α) · η_random,  α = 0.004
```

**为什么这么低的比例就有效？** 因为扩散模型对初始噪声极其敏感——ODE 的起点哪怕偏移 0.1%，30 步积分后终点就会有肉眼可见的差异。6.3% 的运动先验信号已足够"引导"模型在正确的运动方向上生成，同时不破坏模型自身的创造力。

**关键参数**：α=0.004（5 点网格搜索确定），ρ_s=0.1, ρ_m=0.9。

**贡献**：CLIP +1.3%, XCLIP +3.3%（vs L1 alone）。

### 2.3 Layer 3 — Velocity Matching（速度场匹配）

**解决的问题**：L1 给了模型"画什么"的语义信息，L2 给了"从哪里开始画"的起点偏置。但生成过程中每一步的"画笔往哪个方向走"（即速度场），仍然完全由 text embedding 决定。文字描述再精确，也无法编码"猫跳起来的弧度恰好是 45 度"这种细粒度运动信息。我们需要一种方法，直接告诉模型每一步该往哪个方向画。

**通俗理解**：想象生成过程是一段从 A 点到 B 点的旅途。A 点是噪声（η_inv），B 点是参考视频（z₀）。模型生成视频就是从 A 走向某个终点——我们希望这个终点恰好是 B。

模型走路时每一步的方向由"指南针"（text embedding）决定。原始指南针 e₀ 只编码了文字含义，所以模型大致朝 B 的方向走，但路线不够精确。我们的做法是：给指南针加一个微小的"校准偏移" Δe，让它在每一步都精确指向 B。

怎么找到这个 Δe？反复测试。我们已经知道从 A 到 B 的理想方向（就是 B-A 这条直线），于是在路途中随机挑几个位置，问模型："你现在会往哪走？"如果它回答的方向和理想方向有偏差，我们就微调 Δe 让偏差变小。反复 30 次，Δe 就收敛了。

**核心思想**：在 text embedding 上学习一个残差 Δe，使模型在整个 ODE 轨迹上的速度预测 v_pred 对齐理想速度 v*。Δe 编码了文字无法表达的运动细节——加速曲线、节奏感、微妙的方向变化。

**方法**：

```
v* = z₀ - η_inv                          (理想速度：B 减 A，从噪声直达视频的方向)
x_t = (1-t)·η_inv + t·z₀                 (A 到 B 路途上 t 时刻的位置)
L_vel = E_{t} [ || v_θ(x_t, t, e₀+Δe) - v* ||² ]   (让模型预测的方向逼近理想方向)

优化: 30步 Adam + Cosine Annealing, lr=1e-3
注入: e_final = e₀ + es·Δe, es=0.02 (通过 text_encoder hook)
```

每步优化时，我们在 A→B 的直线上随机取一个位置 x_t，把它连同修正后的 embedding `e₀+Δe` 喂给模型，看模型预测的速度 v_pred 和理想速度 v* 差多少（MSE loss），然后只更新 Δe（模型完全冻结）。30 轮下来，Δe 学到了"让模型在每一步都精确指向参考视频"所需的所有微调信息。

最终生成时，Δe 只以 2% 的强度注入（`e_final = e₀ + 0.02·Δe`），相当于给指南针做了极轻微的校准——不改变模型的整体行为，只修正运动方向的细节。

**关键参数**：es=0.02（理论推导 + 网格验证），velocity_steps=30，lr=0.001。

**为什么 30 步就够**：我们使用条件反演（`prompt=caption`），反演出的 η_inv 已处在正确条件的 ODE 轨迹上，相当于 A 点本身就在正确道路的起点附近，Δe 只需做小幅修正。早期方案使用无条件反演（`prompt=""`），A 点偏离正确道路很远，需要 100 步才能把 Δe 拉回来。

**贡献**：CLIP +0.3%, XCLIP +0.5%（vs L1+L2）。

### 2.3.1 Δe 优化过程详解

整个 Velocity Matching 分为两个阶段：**Phase 1** 是离线优化——通过梯度下降求解出 Δe；**Phase 2** 是在线生成——把 Δe 注入到正常生成流程中引导模型。

#### Phase 1：优化出 Δe（30 步梯度下降）

**前置条件（三个输入）：**

在进入优化之前，系统已经准备好三样东西：

- `z₀`：参考视频经 VAE 编码后的 latent——目标视频在潜空间的表达，shape `(1, C, F, H, W)`
- `η_inv`：通过 Flow Matching Inversion 从 z₀ 反演出的噪声——把成品视频"逆播"回起点
- `e₀`：当前 caption 经 T5 text encoder 编码后的文本 embedding，shape `(1, 512, D)`

**核心目标：** 找到一个 Δe（与 e₀ 同 shape 的张量，初始化为全零），使得当模型接收 `e₀ + Δe` 作为文本条件时，模型在整个时间轴上的速度预测都逼近理想速度 `v* = z₀ - η_inv`。

**初始化阶段：**

```python
# Δe 初始化为全零——只有它可以被梯度更新
delta_e = torch.zeros_like(e0, requires_grad=True)

# 优化器只管理 delta_e 这一个变量，模型参数全部冻结
optimizer = Adam([delta_e], lr=1e-3)
scheduler = CosineAnnealingLR(optimizer, T_max=30)  # 余弦退火

# 一个常量：理想速度场，从头到尾不变
v_star = z0 - eta_inv  # shape: (1, C, F, H, W)
```

同时预计算两个辅助工具：

- **Padding Mask**：tokenize caption 后发现有效 token 只占 120~310 个（总长 512），剩下全是 padding。构建一个 mask `(1, 512, 1)`，有效位置为 1，padding 位置为 0——用于每步 backward 后将 padding 位置的梯度归零。
- **Motion Weight**：计算 v* 在帧维度的方差——运动剧烈的像素点 v* 帧间变化大，获得更高的 loss 权重。

**每一步的循环（共 30 步）：**

```
Step i = 0, 1, ..., 29:
```

**① 分层时间步采样（Stratified Sampling）**：将 [0, 1] 区间等分为 K=4 个 bin，每个 bin 内随机采一个 t。这保证每步都覆盖完整时间轴（不会 4 个 t 全扎堆在 0.3 附近），相当于用 4 个"探测点"均匀扫描整条 ODE 路径。

**② 对每个 t_k (k=0..3) 做前向传播**：

```python
e_current = e0 + delta_e                      # 当前修正后的 embedding
x_t = (1 - t_k) * eta_inv + t_k * z0         # 该时刻的线性插值状态
v_pred = model(x_t, t_k, e_current)           # 模型预测的速度
loss_k = mean((v_pred - v_star)² * motion_weight)  # 加权 MSE
```

通俗解释：在 A→B 的直线上取一个中间位置 x_t，问模型"此刻你会往哪走？"，然后和理想方向 v* 对比，差距就是 loss。

**③ 平均 K 个 loss 后反向传播**：

```python
loss = (loss_0 + loss_1 + loss_2 + loss_3) / 4
loss.backward()  # 梯度只流向 delta_e（模型参数 requires_grad=False）
```

**④ Padding Mask 梯度归零**：

```python
delta_e.grad.data *= padding_mask  # padding 位置梯度强制为 0
```

为什么？因为 padding token 在 cross-attention 中不应携带语义信息（参考 Padding Tone, NAACL 2025），如果允许 Δe 在 padding 位置积累数值，会浪费优化容量，甚至引入有害信号。

**⑤ Adam 更新 + 学习率衰减**：

```python
optimizer.step()   # Adam 更新 delta_e
scheduler.step()   # lr 从 1e-3 余弦衰减到 1e-4
```

**30 步后的结果：** Δe 从全零变成一个 L2 范数约 8~10 的张量。它的数值虽小（相对于 ||e₀|| ≈ 1448），但每一位都经过精确优化，携带了"让模型速度场对齐参考视频运动轨迹"的全部微调信息。

**收敛行为观察：**

```
step  0/30: L_vel=0.520, ||Δe||=0.12   ← 开始时模型预测偏差大
step 10/30: L_vel=0.210, ||Δe||=5.80   ← 快速下降阶段
step 20/30: L_vel=0.185, ||Δe||=8.10   ← 进入精调阶段
step 29/30: L_vel=0.178, ||Δe||=8.38   ← 收敛（loss 变化 <1%）
```

#### Phase 2：用 Δe 指导视频生成

拿到优化好的 Δe 后，进入正常的视频生成流程。注入方式是通过一个 **text encoder hook**——不修改任何模型代码或权重，仅在 text encoder 输出上做一次加法。

**Hook 注入机制：**

```python
def text_encoder_hook(module, input, output):
    """在 text encoder 输出上叠加 Δe"""
    hidden_states = output[0]                        # 原始 embedding e₀
    hidden_states = hidden_states + 0.02 * delta_e   # e_final = e₀ + 0.02·Δe
    return (hidden_states,) + output[1:]

# 生成前注册 hook，生成后立即移除
hook_handle = text_encoder.register_forward_hook(text_encoder_hook)
video = pipeline.generate(prompt, latents=η_blended, ...)
hook_handle.remove()
```

**注入强度为什么是 0.02？**

理论推导：前序实验确定最优注入量级为 ||injection|| ≈ 0.17~0.18。当前 ||Δe|| ≈ 8.5，所以 es = 0.18 / 8.5 ≈ 0.02。实际 `||0.02 × Δe|| ≈ 0.17`，相对于 `||e₀|| ≈ 1448` 仅为 0.012% 的扰动——微小但精准。

**为什么这么小的扰动有效？**

因为 Δe 不是随机噪声，它是经过 30 步精心优化的有方向性信号。在 transformer 的 cross-attention 中，text embedding 通过 K/V 投影影响每一帧每一个空间位置的注意力分布。即使 embedding 变化很小，经过 Q·Kᵀ 的点积放大后，attention score 的变化足以将去噪方向偏移到正确的运动轨迹上。30 步 ODE 去噪过程中这个偏移逐步累积，最终产生肉眼可见的运动对齐效果。

**生成过程中的完整数据流：**

```
1. Text Encoder 编码 prompt → e₀
2. Hook 拦截输出 → e_final = e₀ + 0.02·Δe（仅对 positive prompt，negative 不变）
3. Noise Blending → η = √0.004·η_temporal + √0.996·η_random（L2 提供的初始噪声）
4. ODE 去噪 30 步：每步 transformer 接收 (x_t, t, e_final) → 预测速度 v_pred
   ↳ 因为 e_final 携带了校准信息，v_pred 会微微偏向 v*（参考视频方向）
5. 30 步累积 → 生成视频的运动模式明显更接近参考视频
6. Hook 移除，pipeline 状态恢复
```

**关键细节：**

- Hook 只应用一次（第一次调用 text encoder 时，即 positive prompt 编码），CFG 的 negative prompt 编码不受影响。
- 如果 Δe 的序列长度与 hidden_states 不匹配（理论上不会，因为都统一为 512），会自动对齐（截断或补零）。
- 生成结束后 hook 立即移除，不影响后续调用。

**两阶段的关系总结：**

Phase 1 是"逆向求解"——已知想要的行为（v*），反推出需要什么样的文本条件偏移（Δe）。Phase 2 是"正向应用"——把求出的偏移以极低强度注入正常生成，让模型在不知不觉中被引导向正确的运动方向。整个过程不修改模型权重，只微调输入条件，是一种 test-time optimization。

### 2.4 Layer 3 v2 — 三项优化改进

60 步实验失败（指标全面退化）暴露了 v1 架构的三个根因：梯度方差大、能量分散到 padding、静态区域浪费优化。v2 在不增加步数的前提下从代码层面解决这三个问题。

#### 改进 1：分层多时间步采样（Stratified Multi-Timestep Sampling）

**问题**：v1 每步只采 1 个随机 t，30步仅覆盖 30 个时间点，梯度方向随机性高。

**方案**：将 [0, T_m] 分成 K=4 个等宽 bin，每 bin 内均匀采一个 t，每步 4 次 forward+backward 后平均梯度。覆盖密度从 30 提升到 120 点，方差下降 √K=2 倍。

**论文基础**：CVPR 2025 "Adaptive Non-Uniform Timestep Sampling" + NVIDIA CARV (2025)。

#### 改进 2：Padding-Aware 梯度 Mask

**问题**：全部 512 位置统一优化，但实际 caption 只占 120-310 token，能量被分散到无语义的 padding。

**方案**：tokenize caption 获取有效 token 长度 → 构建 padding_mask → 每步 backward 后将 padding 位置梯度置零。

**论文基础**：NAACL 2025 "Padding Tone"。Attention dump 证实 Wan2.1 DiT 中 padding 区域有 4-5x 异常高权重。

#### 改进 3：运动感知 Loss 加权（Motion-Aware Weighting）

**问题**：均匀 MSE 对所有像素同等权重，但视频复现核心是运动——静态背景即使 v_pred 有偏差也不影响感知质量。

**方案**：计算 v* 的帧间方差作为 per-pixel motion weight，高运动区域获得更高 loss 权重。

```
w(x,y) = 1 + λ · log(1 + Var_f(v*[:,:,f,x,y]) / μ)
L = mean(w_normalized · (v_pred - v*)²)
```

**论文基础**："Latent Temporal Discrepancy as Motion Prior" (2025) + Motion Inversion (SIGGRAPH 2025)。

**v2 贡献**：CLIP +0.2%, XCLIP +0.4%（vs v1）。在高运动困难样本上效果显著（#31 XCLIP +5.4%），但对低运动样本存在轻微退化风险。

## 3. 设计哲学与架构

### Pipeline as Flags

所有改动点都是可选的、正交的、可以任意组合的。你可以只用文字（baseline），也可以全部叠加（`--full`）。这使得消融实验变成简单的 flag 排列组合，无需维护多个代码分支。

### 架构概览

```
┌────────────────────────────────────────────────────────────────────────┐
│                           Pipeline Overview                              │
├──────────────────────────────────────┬─────────────────────────────────┤
│  Step 1: Load Reference Video        │  → ref_video tensor             │
│  Step 2: Caption (VLM if empty)      │  → prompt text                  │
│  Step 3: Noise Prior                 │  → (η_temporal, η_inv_raw)      │
│    ├─ Flow Matching Inversion        │     η_inv = ODE_reverse(z₀)     │
│    ├─ SVD Filtering                  │     η_temporal = SVD(η_inv)     │
│    └─ (returns raw η_inv for Step 3.5) │                               │
│  Step 3.5: Velocity Matching         │  → Δe                           │
│    └─ 30-step Δe optimization        │     v_θ(x_t,t,e₀+Δe) ≈ v*     │
│  Step 4: Generate Loop               │  → video frames                 │
│    ├─ Noise Blending (if --blend)    │     η = √α·η_t + √(1-α)·η_r   │
│    ├─ Embedding Hook (if --velocity) │     e_final = e₀ + 0.02·Δe     │
│    ├─ CFG Generation                 │     Wan2.1 diffusers pipeline   │
│    └─ VLM Iteration (if --iter)      │     compare → refine prompt     │
│  Step 5: Output                      │  → final.mp4 + metadata.json   │
└──────────────────────────────────────┴─────────────────────────────────┘
```

### 计算开销

```
Configuration                      DiT Equivalent Forwards    相对 Baseline
────────────────────────────────────────────────────────────────────────────
Baseline (caption only)            ~30                        1.0×
+inversion +svd +blend             ~80                        2.7×
+inversion +svd +blend +velocity   ~170                       5.7×
+all +iter10                       ~470                       15.7×
```

### 轻量化设计选择

相比早期方案（900+ DiT forwards），我们保留了核心优化目标但大幅简化了架构：去掉了 Content Disentanglement（条件反演已天然产生纯净 η_inv）、Position-Aware Gradient Scaling（实验收益有限）、Token Decoding（已证明有害）。同时将优化步数从 100 降到 30（条件反演起点质量高，收敛快），实现 7.6× 加速。

## 实验 2：Velocity Matching 首次验证（es=0.005）

> **所属层级**：L3（Velocity Matching）
> **实验目的**：首次引入 L3 层，验证 Velocity Matching 在 L2 基础上能否带来额外增益。同时修复 `max_sequence_length` bug。

### Bug 修复

**问题**：`_encode_prompt()` 调用 `pipe.encode_prompt()` 时未显式传入 `max_sequence_length`，默认值 226 与生成阶段的 512 不匹配。导致 Δe 在 226-token 空间优化后注入 512-token 空间，后 286 位全零，运动信息严重丢失。

## 实验 3：embed_strength 理论推导与网格搜索

> **所属层级**：L3（Velocity Matching）
> **实验目的**：确定 L3 层的核心超参 embed_strength (es)，使 Δe 注入量落在有效区间。

**时间**：2025-06-04

### 推导过程

注入公式：`e_final = e0 + es * Δe`，模型感知信号大小 = `||es * Δe||`。

前序实验验证的最优注入量区间：||injection|| = 0.18~0.29。

| 前序 es | ||Δe|| (前序) | ||injection|| | 效果 |
|---------|--------------|---------------|------|
| 0.005 | 36.25 | 0.18 | CLIP 最优 |
| 0.008 | 36.25 | 0.29 | XCLIP 最优 |
| 0.01 | 36.25 | 0.36 | 略过强 |

我们的方法中 ||Δe||~8.5（30步无 position-aware，仅前序的 1/4）。等效映射：

- CLIP 最优：es = 0.18 / 8.5 = 0.02
- XCLIP 最优：es = 0.29 / 8.5 = 0.034

### 网格搜索结果

```bash
for es in 0.02 0.05; do
    python run.py ... --velocity_full --alpha 0.004 --embed_strength $es ...
done
```

| 方案 | CLIP | XCLIP | ||es*Δe|| | 说明 |
|------|------|-------|-----------|------|
| noise prior only | 0.8953 | 0.7667 | 0 | baseline |
| velocity es=0.005 | 0.8990 | 0.7639 | ~0.04 | 太弱 |
| **velocity es=0.02** | **0.8981** | **0.7705** | **~0.17** | 最优 |
| velocity es=0.05 | 0.8965 | 0.7685 | ~0.43 | 略过强 |

### 结论

es=0.02 最优，XCLIP 首次超过 baseline（+0.38%），实际注入量 0.17 恰好落在前序实验最优区间下界，与理论一致。

## 实验 4：60 步实验与失败分析

> **所属层级**：L3（Velocity Matching）
> **实验目的**：探索优化步数上限（30→60），验证更多步数能否进一步提升。该实验失败并揭示了 v1 架构的根因问题。

**时间**：2025-06-03

### 结果

| 配置 | ||Δe|| avg | 注入量 (es*||Δe||) | CLIP | XCLIP |
|------|-----------|-------------------|------|-------|
| 30步, es=0.02 | ~8.5 | 0.17 | **0.8981** | **0.7705** |
| 60步, es=0.02 | ~12.5 | 0.25 | 0.8935 | 0.7628 |

### 失败原因

60步指标全面下降，三个根因：

1. **注入过强**：60步使 ||Δe|| 增大到 12.5，注入量 0.25 超出最优区间 0.17，落入"过强"区域。

2. **优化不稳定**：v1 每步只采 1 个随机时间步，梯度方差大。更多步数中方差累积，部分样本（31, 47）出现 loss 振荡（final_loss > 0.5），Δe 方向在不同时间步之间拉锯。

3. **能量分散**：全部 512 token 位置（含大量 padding）被均匀优化，有效 token 获得的梯度能量不足。

### 关键启示

问题不是"步数不够"，而是"每步优化效率低"。需要从代码层面改进：降低梯度方差、集中有效能量、关注运动区域。→ 催生 v2 三项改进。

| **Velocity v2 es=0.02 (L3)** | **0.8998** | **+0.0295** | **0.7736** | **+0.0572** | L1+L2+L3(v2) |

### 三层架构累积贡献

| 层 | 技术 | 增量贡献（vs 上一层） |
|----|------|---------|
| L1 | V4 Hybrid Prompt Rewrite | CLIP +1.4% |
| L2 | SVD Noise Prior (alpha=0.004) | CLIP +1.1%, XCLIP +2.4% |
| L3 v1 | Velocity Matching (es=0.02) | CLIP +0.3%, XCLIP +0.4% |
| L3 v2 | + Stratified + Padding Mask + Motion Weight | CLIP +0.2%, XCLIP +0.4% (vs v1) |

> 注：L3 v2 相比 v1 的全量增益较小（+0.4%），但在高运动困难样本上效果显著（样本 31 XCLIP +5.4%）。v2 的主要价值在于对困难样本的针对性提升，但对低运动样本存在轻微退化风险。

## 下一步改进方向

1. **Adaptive motion weight**：当 v* 帧间方差低于阈值时自动降低 motion_weight_strength
2. **降低 motion_weight_strength**：从 1.0 调到 0.3~0.5，温和化加权
3. **es 联合调参**：v2 的 padding mask 使有效 token 上的 ||Δe|| 更集中，最优 es 可能需要微调
4. **逐项消融**：确认三项改进各自的独立贡献，找出哪项导致退化
