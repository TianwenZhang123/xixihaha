## 1. Step 0：Baseline（纯 VLM Caption 直出）

Baseline 不使用任何改动点 flag，直接用 VLM 原始 caption 生成视频。

```bash
python run.py \
    --data_dir data/videos \
    --caption_dir data/captions_qwen \
    --output_dir outputs/step0_baseline \
    --sample_ids $SAMPLE_IDS \
    --seed 42 \
    --resume
```

**预期结果**：CLIP ≈ 0.8703，XCLIP ≈ 0.7164

---

## 2. Step 1：+Layer 1（V4 Hybrid Prompt Rewrite）

Layer 1 是外部预处理步骤——用 LLM 将 VLM 描述性 caption 改写为指导性 prompt。这是一次性操作，不迭代。

### 2.1 生成 Hybrid Caption

```bash
python scripts/rewrite_hybrid.py \
    --input-dir data/captions_qwen \
    --output-dir data/captions_hybrid \
    --backend dashscope \
    --model qwen-plus \
    --sample-ids $SAMPLE_IDS \
    --skip-existing
```

改写三原则：
1. 首词 = 主体名词（利用 UMT5 position 0 权重优势）
2. 注入时序动作链（initially...then...finally...）
3. 保留原始视觉描述词（颜色、材质、光线等不替换）

### 2.2 用 Hybrid Caption 生成视频

```bash
python run.py \
    --data_dir data/videos \
    --caption_dir data/captions_hybrid \
    --output_dir outputs/step1_L1_hybrid \
    --sample_ids $SAMPLE_IDS \
    --seed 42 \
    --resume
```

**预期结果**：CLIP ≈ 0.8842，XCLIP ≈ 0.7430（相对 Baseline：CLIP +1.6%，XCLIP +3.7%）

---

## 3. Step 2：+Layer 1 + Layer 2（SVD Noise Prior）

Layer 2 在 Layer 1 基础上叠加噪声先验：从参考视频反演噪声 → SVD 分离运动成分 → 与随机噪声混合。

```bash
python run.py \
    --data_dir data/videos \
    --caption_dir data/captions_hybrid \
    --output_dir outputs/step2_L1L2_noise_prior \
    --noise_prior \
    --alpha 0.004 \
    --sample_ids $SAMPLE_IDS \
    --seed 42 \
    --resume
```

`--noise_prior` 是快捷 flag，等价于 `--inversion --svd --blend`。

关键参数：
- `--alpha 0.004`：最优混合系数（5 点网格搜索确定，√α≈0.063 的运动信号权重）
- `--inversion_steps 50`：反演 ODE 步数（默认值，标准精度）
- `--rho_s 0.1`：空间 SVD 截断阈值（去内容）
- `--rho_m 0.9`：时间 SVD 保留阈值（保运动）

**预期结果**：CLIP ≈ 0.8953，XCLIP ≈ 0.7667（相对 Baseline：CLIP +2.9%，XCLIP +7.0%）

---

## 4. Step 3：+Layer 1 + Layer 2 + Layer 3（Velocity Field Matching）

Layer 3 在前两层基础上叠加速度场匹配：优化 embedding 残差 Δe，让生成过程的速度场对齐参考视频定义的理想轨迹。

### 4.1 基础 Velocity（v1 配置）

```bash
python run.py \
    --data_dir data/videos \
    --caption_dir data/captions_hybrid \
    --output_dir outputs/step3_L1L2L3v1 \
    --velocity_full \
    --alpha 0.004 \
    --embed_strength 0.02 \
    --sample_ids $SAMPLE_IDS \
    --seed 42 \
    --resume
```

`--velocity_full` 等价于 `--inversion --svd --blend --velocity`。

关键参数：
- `--embed_strength 0.02`：Δe 注入强度（理论推导 + 网格验证确定的最优值）
- `--velocity_steps 30`：优化步数（条件反演起点质量高，30 步足够收敛）
- `--velocity_lr 1e-3`：Adam 学习率 + Cosine Annealing

**预期结果**：CLIP ≈ 0.8981，XCLIP ≈ 0.7705（相对 Baseline：CLIP +3.2%，XCLIP +7.6%）

### 4.2 增强 Velocity v2（当前最优配置）

```bash
python run.py \
    --data_dir data/videos \
    --caption_dir data/captions_hybrid \
    --output_dir outputs/step3_L1L2L3v2_best \
    --velocity_full \
    --alpha 0.004 \
    --embed_strength 0.02 \
    --velocity_K 4 \
    --velocity_motion_weight 1.0 \
    --sample_ids $SAMPLE_IDS \
    --seed 42 \
    --resume
```

v2 相对 v1 的改进：
- `--velocity_K 4`：分层多时间步采样，覆盖密度 4×，梯度方差下降 ~2×
- `--velocity_motion_weight 1.0`：运动感知 Loss 加权，高运动区域获更高权重
- Padding-Aware 梯度 Mask（自动生效，无需额外 flag）

**预期结果**：CLIP ≈ 0.8998，XCLIP ≈ 0.7736（相对 Baseline：CLIP +3.4%，XCLIP +8.0%）

## 5. 评估

每一步完成后，使用统一评估脚本计算 CLIP 和 X-CLIP 指标：

```bash
python evaluation/run_clip_xclip_eval.py \
    --orig-dir data/videos \
    --gen-dir outputs/<STEP_OUTPUT_DIR> \
    --caption-dir data/captions_hybrid \
    --output-dir outputs/<STEP_OUTPUT_DIR>/eval_clip
```

### 5.1 一键评估所有步骤

```bash
for step_dir in step0_baseline step1_L1_hybrid step2_L1L2_noise_prior step3_L1L2L3v1 step3_L1L2L3v2_best; do
    echo "====== 评估: $step_dir ======"
    python evaluation/run_clip_xclip_eval.py \
        --orig-dir data/videos \
        --gen-dir outputs/$step_dir \
        --caption-dir data/captions_hybrid \
        --output-dir outputs/$step_dir/eval_clip
    echo ""
done
```

## 7. 常见问题

### Q1：alpha/embed_strength 默认值和最优值不一致？

`run.py` 中 `--alpha` 默认 0.001、`--embed_strength` 默认 0.005，这是历史初始值。实验验证最优值为 alpha=0.004、embed_strength=0.02，运行时需显式指定。

### Q2：Velocity 是否依赖 Noise Prior？

`--velocity` 需要 `--inversion`（Δe 优化依赖反演噪声 η_inv 构建目标速度场 v* = z₀ - η_inv）。建议直接用 `--velocity_full`。

### Q3：如何只跑单个视频做快速测试？

```bash
python run.py \
    --video data/videos/31.mp4 \
    --caption "$(cat data/captions_hybrid/31.txt)" \
    --velocity_full --alpha 0.004 --embed_strength 0.02 \
    --output_dir outputs/single_test
```

### Q4：如何恢复中断的批量实验？

加 `--resume` flag，程序会跳过已有输出的样本。

### Q5：每一步的计算开销？

| 配置 | DiT Equivalent Forwards | 相对 Baseline | 单样本耗时 (A800) |
|------|------------------------|--------------|------------------|
| Baseline | ~30 | 1.0× | ~30s |
