# 切分镜（Shot Boundary Detection）调研

## 背景与动机

P-Flow 当前针对 5 秒短视频设计。要将其扩展到长视频（15s+），核心思路是：

```
长视频 → 切分镜（SBD） → 多个短片段 → 每段独立 P-Flow 处理 → 跨镜头 Smoothing → 合成长视频
```

"切分镜"在学术界对应的任务叫做 **Shot Boundary Detection (SBD)**，目标是自动检测视频中镜头切换的位置，将连续视频分割为独立的镜头片段（shot）。

---

## 一、深度学习 SBD 方法

### 1.1 TransNetV2（推荐，当前工业标准）

- **论文**：*TransNet V2: An effective deep network architecture for fast shot transition detection*
- **发表**：arXiv 2008.04838（2020），ACM MM 2024 正式版
- **开源**：[github.com/soCzech/TransNetV2](https://github.com/soCzech/TransNetV2)，PyPI: `transnetv2-pytorch`

**架构设计：**

TransNetV2 采用 Stacked Dilated Dense 3D CNN（DDCNN cells）架构：

1. **输入预处理**：所有帧统一缩放到 48×27×3（极小分辨率，保证速度）
2. **滑动窗口**：每 100 帧为一个 window，取中间 50 帧的预测结果（前后 25 帧 overlap）
3. **DDCNN Cells**：6 个堆叠的 Dilated Dense 3D 卷积块，dilation rate 递增（1,2,4,8...），感受野覆盖整个 100 帧窗口
4. **双预测头**：
   - Frame-level Head：逐帧输出 shot boundary 概率（主预测）
   - Multi-frame Head：预测转场片段的起止范围（训练辅助，推理不用）
5. **Frame Similarity Head**：3 个 AvgPooling 分支 → 空间平均 → 前后 50 帧 cosine distance，作为辅助特征

**训练策略：**

- 数据：ClipShots 数据集中随机挑选 2 个镜头片段（各 100 帧），在随机时间点渲染合成转场（硬切、淡入淡出、擦除等）
- 增强：随机旋转、饱和度/对比度/亮度/色相变换

**性能指标（F1 Score）：**

| 数据集 | F1 |
|--------|-----|
| ClipShots | 77.9 |
| BBC Planet Earth | 96.2 |
| RAI | 93.9 |

**推理速度**：250 fps（单 GPU），适合大规模视频批量处理

**使用方式**：
```python
from transnetv2_pytorch import TransNetV2

model = TransNetV2()
video_frames = ...  # (N, 3, 27, 48) uint8 tensor
predictions = model(video_frames)  # (N,) 每帧的 boundary 概率
```

### 1.2 AutoShot（CVPR 2023 Workshop, NAS）

- **论文**：*AutoShot: A Short Video Dataset and State-of-the-Art Shot Boundary Detection*
- **发表**：CVPR 2023 Workshop on Neural Architecture Search
- **开源**：[github.com/wentaozhu/AutoShot](https://github.com/wentaozhu/AutoShot)

**核心贡献：**

1. **SHOT 数据集**：853 个短视频，11,606 个镜头标注（200 个测试视频含 2,716 个高质量标注）
2. **NAS 搜索**：在融合 3D ConvNet + Transformer 的搜索空间中用 NAS 自动搜索最优 SBD 架构
3. **性能**：在 SHOT 数据集上 F1 比 TransNetV2 高 4.2%

**适用场景**：短视频风格的快节奏切换（抖音/快手类内容）

---

### 1.3 OmniShotCut（2025 最新 SOTA）

- **论文**：*OmniShotCut: Holistic Relational Shot Boundary Detection with Shot-Query Transformer*
- **发表**：arXiv 2604.24762（2025 年 4 月）
- **开源**：[github.com/UVA-Computer-Vision-Lab/OmniShotCut](https://github.com/UVA-Computer-Vision-Lab/OmniShotCut)

**核心创新：**

将 SBD 重新定义为**结构化关系预测**任务，不仅检测边界位置，还识别转场类型和镜头间关系：

1. **Shot-Query Transformer**：可学习的 shot query 向量 + Dense Video Transformer
2. **联合建模**：同时预测 intra-shot relations（镜头内部特征）和 inter-shot relations（镜头间转场关系）
3. **合成数据生成**：自动化转场渲染流水线，覆盖溶解、淡入淡出、擦除等所有类型
4. **新 Benchmark**：114 个视频（动漫/Vlog/游戏/体育/短视频），180K+ 帧，2700+ 标注

**优势**：最全面的 SBD 方案，检测 + 分类 + 关系建模一体化

## 二、工具类方案（轻量/工程向）

### 2.1 PySceneDetect

- **地址**：[github.com/Breakthrough/PySceneDetect](https://github.com/Breakthrough/PySceneDetect)
- **安装**：`pip install scenedetect[opencv]`

**两种检测模式：**

| 模式 | 原理 | 适用场景 |
|------|------|----------|
| `detect-content` | 计算相邻帧 HSV/RGB 差异，超过阈值（默认 27）判定为切换 | 硬切 |
| `detect-threshold` | 帧亮度与黑色级别比较 | 淡入淡出 |

**特点：**
- 纯 CPU OpenCV 实现，无需 GPU
- 集成 ffmpeg/mkvmerge 可自动分割视频为独立文件
- 适合快速原型验证

**使用方式**：
```python
from scenedetect import detect, ContentDetector
scene_list = detect("video.mp4", ContentDetector(threshold=27))
# scene_list: [(start_time, end_time), ...]
```

## 三、场景级分割（Scene Segmentation）

### 3.1 BaSSL（ACCV 2022）

- **论文**：*BaSSL: Boundary-aware Self-Supervised Learning for Video Scene Segmentation*
- **开源**：[github.com/kakaobrain/bassl](https://github.com/kakaobrain/bassl)

**与 SBD 的区别：**

- SBD：检测每一个镜头切换点（shot-level）
- Scene Segmentation：把多个连续镜头归为一个"场景"（scene-level，语义更高）

**方法：**

利用伪边界设计三个 boundary-aware pretext tasks 做自监督预训练：
1. 最大化 intra-scene 相似性
2. 最小化 inter-scene 相似性
3. 在 MovieNet-SSeg 数据集上 fine-tune

**与 P-Flow 的关系**：如果长视频中希望按"语义场景"而非单个硬切来分组处理（比如一个对话场景包含多个正反打镜头但应该一起处理），BaSSL 更合适。

## 四、多镜头视频生成中的分镜设计（参考架构）

这些论文的"分镜处理"设计思路，对 P-Flow 从 5s 扩展到长视频有直接参考价值：

### 4.1 HoloCine（CVPR 2026 Highlight）

- **论文**：*HoloCine: Holistic Generation of Cinematic Multi-Shot Long Video Narratives*
- **开源**：[github.com/yihao-meng/HoloCine](https://github.com/yihao-meng/HoloCine)

**关键设计：**
- **Window Cross-Attention**：每个 shot 的 text prompt 精准定位到对应帧窗口
- **Sparse Inter-Shot Self-Attention**：镜头内全 attention，镜头间稀疏连接（只在关键帧之间建立联系）

**对 P-Flow 的启示**：可以在每个 5s window 内保持原始 pipeline 不变，用稀疏注意力连接相邻 window 实现跨镜头一致性。

### 4.2 VideoGen-of-Thought / VGoT（NeurIPS 2025）

- **论文**：*VideoGen-of-Thought: Step-by-step generating multi-shot video with minimal manual intervention*
- **开源**：[github.com/DuNGEOnmassster/VideoGen-of-Thought](https://github.com/DuNGEOnmassster/VideoGen-of-Thought)

**Pipeline**：
```
单句输入 → LLM 脚本生成（拆分 shot 描述）→ 关键帧生成 → Shot-level 视频生成 → Smoothing
```

用 IP Embedding 做跨镜头身份一致性。

**对 P-Flow 的启示**：先用 SBD 切出原视频的 shot → 每个 shot 独立用 P-Flow 重建 → 用 smoothing 机制做跨 shot 拼接。

### 4.3 MultiShotMaster（CVPR 2026，快手 Kling 团队）

- **论文**：*MultiShotMaster: A Controllable Multi-Shot Video Generation Framework*
- **开源**：[github.com/KlingAIResearch/MultiShotMaster](https://github.com/KlingAIResearch/MultiShotMaster)

**关键设计：**
- 两种新 **RoPE 变体**扩展单镜头模型到多镜头：
  - Shot-aware RoPE：对不同 shot 的帧赋予独立的位置编码起点
  - Cross-shot RoPE：在 shot 边界处引入特殊位置标记
- 自动化数据 curation：从长视频提取多镜头数据 + Gemini-2.5 标注跨镜头一致性

**对 P-Flow 的启示**：位置编码的设计可以让 P-Flow 的 Wan2.1 backbone 区分不同 shot 的时序关系。

### 4.4 ShotAdapter（CVPR 2025）

- **论文**：*ShotAdapter: Text-to-Multi-Shot Video Generation with Diffusion Models*
- **开源**：[shotadapter.github.io](https://shotadapter.github.io/)

**方法**：轻量 adapter 微调预训练 T2V 模型，实现 shot-specific text conditioning，一次生成整个多镜头视频（full attention 跨所有 shot）。

### 4.5 Long Context Tuning / LCT（ICCV 2025）

- **论文**：*Long Context Tuning for Video Generation*
- **地址**：[arxiv.org/abs/2503.10589](https://arxiv.org/abs/2503.10589)

**方法**：直接把单镜头扩散模型的 context window 扩展到场景级：
- 交错 3D 位置嵌入 + 异步噪声策略
- 支持联合生成和自回归生成
- 无需额外参数

**对 P-Flow 的启示**：最优雅的长视频方案——不切分镜，而是直接扩展 P-Flow 的时间窗口。但需要额外训练。

## 五、方案选型建议

| 方案 | 适用场景 | 精度 | 速度 | 工程复杂度 |
|------|----------|------|------|-----------|
| TransNetV2 | 通用长视频切分 | 高 | 250fps | 低（pip install） |
| AutoShot | 短视频/快切 | 更高 | 中 | 中 |
| OmniShotCut | 需要转场分类 | 最高 | 中 | 中 |
| PySceneDetect | 快速验证/轻量 | 中 | 中 | 极低 |
| BaSSL | 语义场景分割 | — | 慢 | 高 |

**P-Flow 推荐路径：**

1. **开发阶段**：PySceneDetect 快速验证 pipeline 可行性
2. **生产阶段**：TransNetV2 作为默认 SBD 后端（精度+速度平衡最好）
3. **进阶**：若需转场类型信息（如区分硬切和淡入淡出以选择不同的跨镜头融合策略），升级 OmniShotCut

│  ┌──────────────────┐                                                │
│  │  Shot Detection  │  ← TransNetV2 / PySceneDetect                  │
│  │  切分镜模块       │                                                │
│  └──────────────────┘                                                │
│       │                                                               │
│       ▼ shots = [(0, 4.8s), (4.8s, 9.2s), (9.2s, 15.0s)]           │
│                                                                       │
│  ┌──────────────────┐                                                │
│  │  Shot Filtering  │  过滤过短/过长片段，必要时 pad/trim 到 5s       │
│  └──────────────────┘                                                │
│       │                                                               │
│       ▼                                                               │
│  ┌──────────────────────────────────────────────┐                    │
│  │  Per-Shot P-Flow (并行/串行)                   │                    │
│  │  ┌────────┐ ┌────────┐ ┌────────┐           │                    │
│  │  │ Shot 1 │ │ Shot 2 │ │ Shot 3 │  ...      │                    │
│  │  │ P-Flow │ │ P-Flow │ │ P-Flow │           │                    │
│  │  └────────┘ └────────┘ └────────┘           │                    │
│  └──────────────────────────────────────────────┘                    │
│       │                                                               │
│       ▼                                                               │
│  ┌──────────────────┐                                                │
│  │  Cross-Shot      │  帧融合 / overlap blending / smoothing          │
│  │  Stitching       │                                                │
│  └──────────────────┘                                                │
│       │                                                               │
│       ▼                                                               │
│  输出: 完整长视频                                                      │

└─────────────────────────────────────────────────────────────────────┘
