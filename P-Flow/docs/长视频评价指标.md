## 1. 帧级保真度指标（有参考）

这类指标需要 ground truth 参考视频，逐帧对比。适合 P-Flow 这种有明确参考视频的复现任务。

### 1.1 PSNR（Peak Signal-to-Noise Ratio）

**含义**：基于均方误差的像素级失真度量。

**计算方法**：

```
MSE = (1/N) * Σ(I_ref(i) - I_gen(i))²

PSNR = 10 * log₁₀(MAX² / MSE)
```

其中 MAX 为像素最大值（8-bit 图像为 255），N 为总像素数。

**视频级聚合**：对所有帧的 PSNR 取平均。

**范围**：越高越好，典型范围 20-40 dB。

**局限**：与人眼感知相关性差，对结构性失真不敏感。

---

### 1.2 SSIM（Structural Similarity Index）

**含义**：从亮度、对比度、结构三方面度量图像相似性，模拟人眼对结构信息的敏感性。

**计算方法**：

```
SSIM(x, y) = [l(x,y)]^α · [c(x,y)]^β · [s(x,y)]^γ

其中：
  l(x,y) = (2μ_x·μ_y + C₁) / (μ_x² + μ_y² + C₁)       # 亮度
  c(x,y) = (2σ_x·σ_y + C₂) / (σ_x² + σ_y² + C₂)       # 对比度
  s(x,y) = (σ_xy + C₃) / (σ_x·σ_y + C₃)                 # 结构

  C₁ = (K₁·L)², C₂ = (K₂·L)², C₃ = C₂/2
  L = 255, K₁ = 0.01, K₂ = 0.03
```

**视频级聚合**：逐帧计算 SSIM 后取平均。注意应先转灰度再计算。

**范围**：[0, 1]，越高越好。

---

### 1.3 LPIPS（Learned Perceptual Image Patch Similarity）

**含义**：使用深度网络（VGG/AlexNet）的中间层特征计算感知距离，与人眼感知高度相关。

**计算方法**：

```
LPIPS(x, y) = Σ_l  w_l · ||φ_l(x) - φ_l(y)||₂²

其中：
  φ_l(x) = 网络第 l 层对图像 x 的特征图（归一化后）
  w_l = 该层的学习权重（在人类感知判断数据上训练）
  求和遍历网络的多个中间层（VGG: conv1_2 ~ conv5_4）
```

**视频级聚合**：逐帧计算 LPIPS 后取平均。

**范围**：越低越好（0 表示完全相同）。

**优势**：目前帧级质量评估中与人类感知最对齐的指标。

## 2. 时序一致性指标（长视频核心）

**这是长视频评估最关键的维度。** 长视频相比短视频，最大的问题就是时序上的漂移（drift）、闪烁（flicker）和身份不一致（identity inconsistency）。

### 2.1 Warping Error（光流翘曲误差）

**含义**：通过光流将前一帧 warp 到后一帧位置，衡量 warp 结果与实际后一帧的差异。直接反映时序连贯性。

**计算方法**：

```
Step 1: 用 RAFT 估计帧 I_t → I_{t+1} 的光流场 F_{t→t+1}
Step 2: 将 I_t 按光流 warp：I_t^{warp} = Warp(I_t, F_{t→t+1})
Step 3: 计算翘曲误差（仅在非遮挡区域 M）：

  E_warp = (1/|M|) · Σ_{p∈M} ||I_t^{warp}(p) - I_{t+1}(p)||₂

Step 4: 遮挡 mask M 通过前向-后向一致性检测：
  M(p) = 1  if ||F_{t→t+1}(p) + F_{t+1→t}(p + F_{t→t+1}(p))|| < threshold
```

**视频级聚合**：所有相邻帧对的 E_warp 取平均。

**所用模型**：RAFT（Recurrent All-Pairs Field Transforms, ECCV 2020）。

**范围**：越低越好。

**适用**：EvalCrafter 的 `warping_error` 指标直接使用此方法。

---

### 2.2 Subject Consistency（主体一致性）— VBench

**含义**：评估视频中主体（人、动物、物体）的外观身份在时间轴上是否保持一致。长视频中主体"变形"或"身份漂移"是最常见的问题。

**计算方法**：

```
Step 1: 对视频每一帧提取 DINO-ViT 特征（使用 DINOv2 ViT-B/14）
        f_t = DINO(I_t)  →  [CLS] token 作为帧级特征

Step 2: 计算相邻帧间余弦相似度：
        sim_t = cos(f_t, f_{t+1}) = (f_t · f_{t+1}) / (||f_t|| · ||f_{t+1}||)

Step 3: 对所有相邻帧对取平均：
        Subject_Consistency = (1/(T-1)) · Σ_{t=1}^{T-1} sim_t
```

**范围**：[0, 1]，越高越好。

**为什么用 DINO 而非 CLIP**：DINO 的自监督训练使其对物体的视觉外观/纹理/形状极其敏感（实例级），而 CLIP 更偏向语义级别。因此 DINO 更适合检测"同一只猫是否始终是同一只猫"这类身份一致性问题。

**VBench-Long 扩展**：对长视频先做语义场景分割，在每个 clip 内计算 in-clip consistency，同时从每个 clip 抽取代表帧计算 cross-clip consistency，最后用分位数融合。

---

### 2.3 Background Consistency（背景一致性）— VBench

**含义**：评估视频背景是否保持稳定一致（无不合理变化）。

**计算方法**：

```
Step 1: 对视频每一帧提取 CLIP 图像特征（ViT-L/14）
        f_t = CLIP_image(I_t)

Step 2: 计算每帧与第一帧的余弦相似度：
        sim_t = cos(f_t, f_1)

Step 3: 同时计算相邻帧间相似度：
        sim_adj_t = cos(f_t, f_{t+1})

Step 4: 综合取平均：
        Background_Consistency = mean([sim_1, sim_2, ..., sim_{T-1}])
```

**范围**：[0, 1]，越高越好。

**为什么用 CLIP 而非 DINO**：背景通常是场景级语义（"海滩"、"城市"），CLIP 的语义编码能力更适合捕捉场景级变化。

---

### 2.4 Temporal Flickering（时间闪烁）— VBench

**含义**：检测局部高频细节的不真实抖动。这在长视频中尤为明显——即使整体运动流畅，但纹理、光照、小物件可能帧间剧烈跳动。

**计算方法**：

```
Step 1: 读取视频所有帧（仅对静态/低运动视频有效）
Step 2: 计算相邻帧的逐像素绝对差：
        diff_t = |I_t - I_{t+1}|    （逐像素 L1）

Step 3: 对每对帧计算 MAE：
        MAE_t = mean(diff_t)

Step 4: 转换为得分（越高越好）：
        Temporal_Flickering = (255.0 - mean(MAE_1, ..., MAE_{T-1})) / 255.0
```

**范围**：[0, 1]，越高越好（1 = 完全无闪烁）。

**注意**：此指标主要针对应该静止的区域。对高运动视频需要配合光流 mask 使用。

---

### 2.5 CLIP Temporal Score（帧间语义一致性）

**含义**：用 CLIP 图像编码器衡量相邻帧间的语义跳变程度。EvalCrafter 的 `clip_temp_score`。

**计算方法**：

```
Step 1: 对每一帧提取 CLIP 图像特征
        f_t = CLIP_image(I_t)

Step 2: 计算相邻帧余弦相似度：
        sim_t = cos(f_t, f_{t+1})

Step 3: 取平均：
        CLIP_Temp_Score = (1/(T-1)) · Σ sim_t
```

**范围**：[0, 1]，越高越好。

**与 Subject Consistency 区别**：CLIP Temporal Score 衡量整帧语义一致性，Subject Consistency 用 DINO 专注于主体身份。

---

### 2.6 Face Consistency Score（人脸一致性）

**含义**：针对包含人脸的视频，检测跨帧人脸身份是否一致。EvalCrafter 指标。

**计算方法**：

```
Step 1: 使用人脸检测模型（如 RetinaFace）检测每帧人脸
Step 2: 使用人脸识别模型（如 ArcFace）提取人脸 embedding
        e_t = ArcFace(crop_face(I_t))

Step 3: 计算相邻帧人脸 embedding 余弦相似度：
        sim_t = cos(e_t, e_{t+1})

Step 4: 对所有有效帧对取平均：
        Face_Consistency = mean(sim_t for t where face detected)
```

**范围**：[0, 1]，越高越好。

**适用**：任何包含人脸的长视频，检测是否出现"换脸"现象。

## 3. 分布级指标（无参考）

衡量生成视频集合与真实视频集合的分布距离，不需要逐帧配对。

### 3.1 FVD（Fréchet Video Distance）

**含义**：视频生成领域最经典的分布评估指标。类比图像领域的 FID，使用 I3D 网络提取时空特征。

**计算方法**：

```
Step 1: 准备两组视频——真实视频集 R 和生成视频集 G（每组 ≥2048 个）
Step 2: 对每个视频取固定帧数（通常 16 帧），通过预训练 I3D 网络提取特征：
        f_i = I3D(video_i)    →  输出 400-dim 特征向量

Step 3: 分别计算两组特征的均值和协方差：
        μ_r, Σ_r = mean(f_R), cov(f_R)
        μ_g, Σ_g = mean(f_G), cov(f_G)

Step 4: 计算 Fréchet 距离：
        FVD = ||μ_r - μ_g||² + Tr(Σ_r + Σ_g - 2·(Σ_r · Σ_g)^{1/2})
```

**所用模型**：I3D（Inflated 3D ConvNet），在 Kinetics-400 上预训练。

**范围**：越低越好（0 = 完全相同分布）。

**已知局限（ICLR 2025 "Beyond FVD"）**：
- I3D 特征空间非高斯分布，违反 Fréchet 距离的高斯假设
- I3D 对时序扰动严重不敏感（content bias：即使打乱帧顺序 FVD 变化不大）
- 需要 ≥2048 个样本才能稳定，小规模实验不可靠
- 16 帧的输入窗口限制了对长视频的评估能力

---

### 3.2 JEDi（Joint Embedding Distance）— FVD 替代品

**含义**：ICLR 2025 提出的新一代分布评估指标，解决 FVD 三大局限。

**计算方法**：

```
Step 1: 使用自监督视频模型（如 VideoMAE/V-JEPA）提取特征（替代 I3D）
        f_i = VideoMAE(video_i)

Step 2: 使用 kernel-based 距离替代 Fréchet 距离：
        JEDi = MMD²(P_r, P_g)   （Maximum Mean Discrepancy）
        
        MMD²(P, Q) = E[k(x,x')] + E[k(y,y')] - 2·E[k(x,y)]
        其中 k 为高斯核函数 k(x,y) = exp(-||x-y||²/(2σ²))

Step 3: 使用 permutation test 计算统计显著性
```

**优势**：
- 不假设高斯分布
- 对时序扰动敏感
- 样本效率高（~256 个视频即可稳定）
- 与人类评估相关性更强

---

### 3.3 FVMD（Fréchet Video Motion Distance）

**含义**：只评估运动质量的分布指标，解耦了内容与运动。

**计算方法**：

```
Step 1: 提取视频运动特征（光流统计量或运动编码器输出）
        m_i = MotionEncoder(video_i)  或  m_i = StatisticalFlow(video_i)

Step 2: 与 FVD 类似计算 Fréchet 距离：
        FVMD = ||μ_r^m - μ_g^m||² + Tr(Σ_r^m + Σ_g^m - 2·(Σ_r^m · Σ_g^m)^{1/2})
```

**优势**：FVD 存在严重的 content bias（对内容敏感、对运动不敏感），FVMD 专注运动质量。

---

### 3.4 IS（Inception Score）

**含义**：评估生成帧的多样性和真实性。

**计算方法**：

```
IS = exp(E_x[KL(p(y|x) || p(y))])

其中：
  p(y|x) = Inception 网络对单帧 x 的类别概率分布（应尖锐→真实）
  p(y) = 所有生成帧的边缘分布（应均匀→多样）
```

**范围**：越高越好。

**局限**：不考虑时序信息，本质是帧级指标。

## 4. 语义对齐指标（文本/视频匹配）

### 4.1 CLIP Score

**含义**：衡量生成视频与文本 prompt 的语义对齐程度。

**计算方法**：

```
Step 1: 用 CLIP 文本编码器提取 prompt 特征：
        f_text = CLIP_text(prompt)

Step 2: 对视频每帧用 CLIP 图像编码器提取特征：
        f_t = CLIP_image(I_t)

Step 3: 计算文本-图像余弦相似度：
        sim_t = cos(f_text, f_t) = (f_text · f_t) / (||f_text|| · ||f_t||)

Step 4: 对所有帧取平均（或取中间帧）：
        CLIP_Score = (1/T) · Σ_t sim_t

Step 5: 可选乘以缩放因子：
        CLIP_Score_final = 100 · max(sim, 0)
```

**所用模型**：OpenAI CLIP ViT-L/14 或 ViT-B/32。

**范围**：越高越好（典型范围 0.2-0.35）。

---

### 4.2 X-CLIP Score（时序感知的视频-文本对齐）

**含义**：引入时序建模的视频级语义对齐。P-Flow 实验中已使用的 `orig_gen_xclip`。

**计算方法**：

```
Step 1: 从视频均匀采样 N 帧（如 8/16/32 帧）
Step 2: X-CLIP 视频编码器处理帧序列，输出视频级特征：
        f_video = X-CLIP_video([I_1, I_2, ..., I_N])
        （内含 cross-frame attention 建模时序关系）

Step 3: X-CLIP 文本编码器处理 prompt：
        f_text = X-CLIP_text(prompt)

Step 4: 计算余弦相似度：
        X-CLIP_Score = cos(f_video, f_text)
```

**优势**：相比逐帧 CLIP 取平均，X-CLIP 能感知时序动态（如"从左走到右"这类运动描述）。

---

### 4.3 ViCLIP Score（视频原生 CLIP）

**含义**：OpenGVLab 的视频原生对比学习模型，直接在视频-文本对上训练。VBench 的 Overall Consistency 和 Temporal Style 维度使用。

**计算方法**：

```
Step 1: 从视频均匀采样 8 帧
Step 2: 通过 ViCLIP 视频编码器（ViT 架构 + temporal attention）：
        f_video = ViCLIP_video([I_1, ..., I_8])

Step 3: 通过 ViCLIP 文本编码器：
        f_text = ViCLIP_text(prompt)

Step 4: 计算余弦相似度：
        ViCLIP_Score = cos(f_video, f_text)
```

**与 X-CLIP 区别**：ViCLIP 是从头在 10M 视频-文本对上训练的视频原生模型（基于 InternVideo），时序建模能力更强。

---

### 4.4 BLIP-BLEU（生成 Caption 对比）

**含义**：先对生成视频做自动描述，再计算描述与原始 prompt 的文本相似度。EvalCrafter 指标。

**计算方法**：

```
Step 1: 从生成视频抽取关键帧
Step 2: 用 BLIP-2 对关键帧生成 caption：
        caption = BLIP2_caption(I_keyframe)

Step 3: 计算 caption 与原始 prompt 的 BLEU 分数：
        BLIP_BLEU = BLEU(caption, prompt)
        
        BLEU-4 = BP · exp(Σ_{n=1}^{4} (1/4) · log(p_n))
        BP = min(1, exp(1 - ref_len/gen_len))
        p_n = n-gram precision
```

**范围**：[0, 1]，越高越好。

## 5. 运动质量指标

### 5.1 Motion Smoothness（运动平滑度）— VBench

**含义**：评估物体运动是否平滑自然（无突兀跳动）。

**计算方法**：

```
Step 1: 取三帧序列 (I_{t-1}, I_t, I_{t+1})
Step 2: 使用帧插值模型 AMT 从 I_{t-1} 和 I_{t+1} 生成预测中间帧：
        I_t^{pred} = AMT(I_{t-1}, I_{t+1})

Step 3: 计算预测帧与实际帧的相似度（SSIM 或 LPIPS）：
        score_t = SSIM(I_t, I_t^{pred})   或   1 - LPIPS(I_t, I_t^{pred})

Step 4: 对所有中间帧取平均：
        Motion_Smoothness = (1/(T-2)) · Σ_{t=2}^{T-1} score_t
```

**原理**：如果运动是平滑的，那么中间帧应该可以从前后帧准确插值得到。如果运动突兀，插值结果与实际帧差异大。

**所用模型**：AMT（All-pairs Multi-field Transforms）帧插值模型。

**范围**：[0, 1]，越高越好。

---

### 5.2 Dynamic Degree（动态程度）— VBench

**含义**：评估视频是否包含足够的运动（避免"静态图片式"输出）。

**计算方法**：

```
Step 1: 使用 RAFT 计算相邻帧间光流 F_t
Step 2: 计算每帧对的光流幅度均值：
        flow_mag_t = mean(||F_t(p)||₂)    对所有像素 p

Step 3: 判断是否有足够帧的运动超过阈值：
        is_dynamic = count(flow_mag_t > threshold) >= count_num

Step 4: 输出二值或连续分数：
        Dynamic_Degree = proportion of videos deemed dynamic
        或 = mean(flow_mag_t) 作为连续量
```

**参数**：`threshold` 和 `count_num` 由 VBench 配置文件指定。

**所用模型**：RAFT 光流模型。

---

### 5.3 Flow Score（光流合理性）— EvalCrafter

**含义**：评估生成视频中光流场的合理性和自然程度。

**计算方法**：

```
Step 1: 用 RAFT 估计生成视频的光流场序列 {F_1, F_2, ..., F_{T-1}}
Step 2: 计算光流场的统计特征：
        - 光流幅度均值和方差
        - 光流方向一致性
        - 光流场的平滑度（梯度）

Step 3: 与真实视频的光流统计分布对比，或直接用绝对值评估：
        Flow_Score = aggregate(flow statistics)
```

---

### 5.4 Action Score（动作识别分数）— EvalCrafter

**含义**：使用预训练动作识别模型评估视频中动作是否可被正确识别。

**计算方法**：

```
Step 1: 从 prompt 中提取目标动作标签（如 "running", "jumping"）
Step 2: 将生成视频输入动作识别模型（如 VideoMAE/TimeSformer）
Step 3: 获取动作分类概率：
        p(action | video) = ActionRecognizer(video)

Step 4: 检查目标动作的置信度：
        Action_Score = p(target_action | generated_video)
```

**范围**：[0, 1]，越高越好。

---

### 5.5 Motion AC Score（运动准确性分数）— EvalCrafter

**含义**：通过动作分类 top-k 准确率衡量运动表达的准确性。

**计算方法**：

```
Step 1: 使用 VideoMAE 或类似模型对生成视频做动作分类
Step 2: 检查目标动作是否在 top-k 预测中：
        Motion_AC = 1 if target_action in top_k_predictions else 0

Step 3: 对所有测试视频取平均：
        Motion_AC_Score = mean(Motion_AC_i for all videos)
```

## 6. 多镜头/叙事级指标（长视频专属）

**这是长视频区别于短视频评估的核心维度。** 长视频不仅要单帧好看、帧间连贯，还需要跨镜头的叙事一致性、情节推进合理性。

### 6.1 NarrLV 叙事评估体系（ICLR 2026）

NarrLV 是首个全面评估长视频生成模型**叙事表达能力**的 benchmark。受电影叙事理论启发。

#### 核心概念：TNA（Temporal Narrative Atom，时间叙事原子）

TNA 是视频中保持连续视觉呈现的基本叙事单元。一个 TNA 内主体、场景、动作保持一致；TNA 的数量衡量叙事丰富度。

#### 指标 1：Narrative Element Fidelity（R_fid，叙事元素忠实度）

**含义**：检查单个 TNA 内的内容是否准确表达了 prompt 中描述的叙事元素。

**计算方法**：

```
Step 1: 将 prompt 分解为多个叙事元素 E = {e_1, e_2, ..., e_n}
        （包括：主体属性、动作、场景、情绪等）

Step 2: 对生成视频的每个 TNA，使用 MLLM 生成关于每个元素的 QA 对：
        Q_i = "Does the video show [element e_i]?"
        
Step 3: 用 MLLM（如 GPT-4V / Qwen-VL）回答问题并打分：
        score_i = MLLM_QA(video_segment, Q_i)  ∈ {0, 1}

Step 4: 计算忠实度：
        R_fid = (1/n) · Σ_{i=1}^{n} score_i
```

**范围**：[0, 1]，越高越好。

#### 指标 2：Narrative Unit Coverage (R_cov)

评估视频是否涵盖了 prompt 中描述的所有叙事单元（TNA）。

```
计算方法：
Step 1: 从 prompt 中提取所有 TNA 单元列表 {TNA_1, TNA_2, ..., TNA_m}

Step 2: 对生成视频进行语义分段，识别哪些 TNA 被视觉呈现：
        covered_i = MLLM_detect(video, TNA_i)  ∈ {0, 1}

Step 3: 计算覆盖度：
        R_cov = (1/m) · Σ_{i=1}^{m} covered_i
```

**范围**：[0, 1]，越高越好。反映长视频是否「丢失」了叙事内容。

#### 指标 3：Narrative Unit Coherence (R_coh)

评估相邻 TNA 之间的逻辑和视觉过渡是否自然连贯。

```
计算方法：
Step 1: 将视频按 TNA 边界切分为 segments {s_1, s_2, ..., s_m}

Step 2: 对每对相邻 segment (s_i, s_{i+1}) 评估过渡质量：
        Q = "Is the transition from [TNA_i description] to [TNA_{i+1} description] 
             natural and coherent?"
        coh_i = MLLM_QA(concat(s_i[-k:], s_{i+1}[:k]), Q)  ∈ [1, 5]

Step 3: 计算连贯度：
        R_coh = (1/(m-1)) · Σ_{i=1}^{m-1} (coh_i - 1) / 4
```

**范围**：[0, 1]，越高越好。是长视频叙事质量的核心指标。

### 6.2 DirectorBench 分镜诊断性指标（2025）

面向分钟级 multi-shot 长视频的诊断性评估框架，使用多 Agent 模拟不同用户偏好。

#### Within-Shot Quality（镜头内质量）

评估单个镜头内的视觉质量和一致性。

```
计算方法：
使用 MLLM Agent 对每个 shot 独立打分：
        ws_score_i = Agent_eval(shot_i, shot_prompt_i)

维度包括：
- Visual fidelity（视觉真实感）
- Prompt adherence（是否遵循该 shot 的描述）
- Internal consistency（shot 内部一致性）

Within-Shot Quality = (1/N) · Σ_{i=1}^{N} ws_score_i
```

#### Between-Shot Transition Quality（镜头间转换质量）

**当前长视频生成的最大瓶颈**——论文发现平均仅 0.256，最佳 workflow 也仅 0.356。

```
计算方法：
Step 1: 提取每对相邻 shot 的边界帧区域：
        boundary_i = (shot_i[-k_frames:], shot_{i+1}[:k_frames])

Step 2: MLLM Agent 评估转换质量：
        trans_score_i = Agent_eval(boundary_i, transition_type_i)

评估维度：
- Temporal smoothness（时间平滑度）
- Narrative logic（叙事逻辑性）
- Style continuity（风格延续性）
- Character persistence（人物身份保持）

Transition Quality = (1/(N-1)) · Σ_{i=1}^{N-1} trans_score_i
```

**范围**：[0, 1]，越高越好。

#### Prompt-Level User Demand Fulfillment（用户需求满足度）

```
计算方法：
对整段视频评估是否满足了用户的完整创作意图：
        fulfillment = Agent_eval(full_video, full_script, user_profile)

考虑因素：
- 是否包含所有要求的 shots
- 整体叙事是否通顺
- 是否符合该 user_profile 的偏好（如专业导演 vs 普通用户）

当前最佳模型平均得分 0.71。
```

## 7. MSVBench 多镜头视频评估（ACL 2026 Findings）

20 个子指标，四大维度的混合评估框架（LMM + 专家模型）。与人类判断 Spearman 相关性达 94.4%。

### 7.1 Visual Quality 维度

```
子指标：
- Frame Aesthetics: LAION aesthetic predictor 逐帧打分取均值
- Resolution Quality: 基于清晰度/噪声水平的技术质量评估
- Color Harmony: 色彩和谐度评估
- Artifact Detection: 生成伪影（毛刺、变形）检测
```

### 7.2 Story Video Alignment 维度

```
子指标：
- Script Adherence: 每个 shot 是否遵循脚本描述
- Character Appearance Match: 角色外观是否与参考图一致
- Scene Setting Match: 场景设定是否正确
- Action Execution: 动作执行是否准确
- Dialogue Timing: 对话时机是否合理

计算方法：LMM 对视频-脚本对进行 QA 评估
```

### 7.3 Video Consistency 维度（跨镜头！）

```
子指标：
- Character Identity Consistency: 同一角色在不同镜头中身份是否一致
- Scene Continuity: 场景切换是否合理连续
- Object Persistence: 物体是否在不该消失时消失
- Style Consistency: 视觉风格是否统一
- Temporal Logic: 时间逻辑是否正确（如白天→黑夜不应突然跳转）

计算方法：
- 专家模型：DINO/CLIP 特征跨 shot 余弦相似度
- LMM 评估：多模态模型整体判断
- 混合得分 = α · expert_score + (1-α) · LMM_score
```

### 7.4 Motion Quality 维度

```
子指标：
- Motion Naturalness: 运动自然度（AMT 帧插值对比）
- Camera Movement Smoothness: 镜头运动平滑度
- Action Continuity: 跨 cut 动作连续性
- Physics Plausibility: 物理合理性
```

## 8. VideoScore 模型评分（TIGER-Lab）

基于 Mantis-8B-Idefics2 训练的端到端视频质量评估模型，直接输出 1.0-4.0 分数。

### 五个评估维度

```
1. Visual Quality (VQ):
   评估画面清晰度、色彩准确性、无伪影程度
   输入：视频帧序列（最多 48 帧）
   输出：1.0-4.0 连续分数

2. Temporal Consistency (TC):
   评估帧间一致性、无闪烁、主体稳定
   关注：物体形状/颜色是否突变

3. Dynamic Degree (DD):
   评估视频是否有合理的运动量
   区分：静态幻灯片 vs 真实动态视频

4. Text-to-Video Alignment (TVA):
   评估生成内容是否符合文本描述
   关注：语义准确性、细节完整性

5. Factual Consistency (FC):
   评估视频内容是否符合常识和物理规律
   检测：违反重力、比例失调、逻辑矛盾

使用方法：
        from transformers import AutoModelForCausalLM
        model = AutoModelForCausalLM.from_pretrained("TIGER-Lab/VideoScore-v1.1")
        # 输入视频帧 + 评估 prompt → 输出 5 维分数
```

**优势**：轻量（单卡推理）、无需参考视频、与人类判断高度一致（Spearman 77.1）。

## 9. P-Flow 15s 扩展推荐指标方案

### 9.1 核心评估方案

| 评估层面 | 推荐指标 | 理由 | 计算成本 |
|---------|---------|------|----------|
| 帧级保真度 | LPIPS + SSIM | 有参考视频可逐帧对比 | 低 |
| 时序一致性 | Warping Error + Subject Consistency (DINO) | 长视频核心痛点是漂移 | 中 |
| 语义保持 | X-CLIP（已有）+ ViCLIP | 捕获时序语义 | 中 |
| 运动质量 | Flow Score + Motion Smoothness | 15s 中运动自然度 | 中 |
| 整体分布 | JEDi（替代 FVD） | FVD 对长视频不可靠 | 高 |
| MLLM 评分 | VideoScore 5 维度 | 模拟人类感知 | 低 |
| 多镜头一致性 | MSVBench Video Consistency 子集 | 跨 shot 身份保持 | 中 |

### 9.2 针对 P-Flow 的特殊指标

```
1. 长程身份漂移检测（Long-Range Identity Drift）:
   计算方法：每隔 1s 提取关键帧，计算首帧与所有后续帧的 DINO 余弦相似度
   绘制 drift curve: sim(frame_0, frame_t) vs t
   健康标准：曲线斜率 > -0.01/s（即 15s 内相似度下降不超过 0.15）

2. 运动模式保持度（Motion Pattern Fidelity）:
   计算方法：原始视频与生成视频的光流场余弦相似度
   flow_sim_t = cosine(RAFT(orig_t, orig_{t+1}), RAFT(gen_t, gen_{t+1}))
   Motion Fidelity = mean(flow_sim_t) for all t

3. 跨窗口缝合质量（Cross-Window Stitching Quality）:
   如果用滑动窗口生成 15s，需检测窗口边界处的不连续：
   boundary_error = MAE(gen_{window1_end}, gen_{window2_start})
   + flow_discontinuity at boundary
```
