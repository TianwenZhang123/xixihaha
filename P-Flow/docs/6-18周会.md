## 一、Wan2.1 模型输入与 P-Flow 可操作接口

### 1.1 Wan2.1 Pipeline 接受的完整输入

P-Flow 使用 diffusers 官方 `WanPipeline`，通过标准 `__call__` 接口驱动生成。根据代码（`src/pipeline.py:1568-1582`），实际传入的参数如下：

| 参数 | 类型 | 官方默认值 | P-Flow 使用值 | 说明 |
|------|------|--------|--------|------|
| `prompt` | str | None | VLM caption | 文本描述，经内部 UMT5 text encoder 编码为 token embeddings，作为 cross-attention 条件 |
| `negative_prompt` | str | None | 硬编码 NEGATIVE_PROMPT | 负面提示词，用于 CFG 引导 |
| `height` | int | 480 | 480 | 视频高度（像素） |
| `width` | int | 832 | 832 | 视频宽度（像素） |
| `num_frames` | int | 81 | 81 | 生成帧数（VAE 压缩后时间维 F=21） |
| `guidance_scale` | float | 5.0 | 5.0 | Classifier-Free Guidance 强度 |
| `num_inference_steps` | int | **50** | **30** | Flow Matching 去噪步数（P-Flow 自行设为 30） |
| `generator` | torch.Generator | None | seed=42 | 随机数生成器（控制 η_random） |
| `output_type` | str | **"np"** | **"pt"** | 输出格式（官方默认 numpy，P-Flow 用 PyTorch tensor） |
| `latents` | Tensor / None | None | SVD z_T / None | **初始噪声 z_T**，形状 `[1, 16, F, H, W]`。若为 None 则内部随机采样 N(0,1) |
| `callback_on_step_end` | Callable / None | None | 灰盒回调 / None | 每步去噪后的回调函数（灰盒操作入口） |
| `callback_on_step_end_tensor_inputs` | list | ["latents"] | ["latents"] | 回调可访问的 tensor 名称 |

> **验证结论（对照 [huggingface/diffusers main 分支](https://github.com/huggingface/diffusers/blob/main/src/diffusers/pipelines/wan/pipeline_wan.py)）**：上述 12 个参数**全部是 WanPipeline 官方 API 原生支持的参数**，P-Flow 没有新增任何自定义参数。官方 `__call__` 共有 18 个参数，P-Flow 未使用的 6 个为：`guidance_scale_2`（二阶段 transformer 专用）、`num_videos_per_prompt`、`prompt_embeds`、`negative_prompt_embeds`、`return_dict`、`attention_kwargs`、`max_sequence_length`。P-Flow 仅在 `num_inference_steps`（50→30）和 `output_type`（"np"→"pt"）上使用了非默认值。

### 1.2 三个操作接口

根据代码逻辑，P-Flow 将 Wan2.1 作为几乎不改动的黑盒/灰盒使用，真正可操作的接口有三个：

**接口 1：prompt（文本条件）**—— L1 层操作对象。prompt 以字符串形式传入，由 WanPipeline 内部调用 UMT5 编码。P-Flow 不直接传 `prompt_embeds`（`encode_prompt` 仅在 inversion 阶段单独使用）。

**接口 2：latents（初始噪声）**—— L2 层操作对象。通过 inversion + SVD 滤波 + blend 构造结构化初始噪声，替代默认的纯随机 N(0,1)。这是纯黑盒下唯一能传递参考视频信息的通道。

**接口 3：灰盒操作入口**——在 Wan2.1 的生成 pipeline 中，有两个可用的灰盒机制：

- **`callback_on_step_end`**（diffusers 回调）：每步 denoise **之后** 拿到当前 latents，在噪声空间施加外部约束。历史机制，当前未使用。
- **`register_forward_hook`**（PyTorch hook）：在 DiT transformer 的指定层（mid=层10~19）挂载 forward hook，在**前向传播内部**捕获并修改中间层特征。这是当前 **FI（Feature Injection）** 的实现方式——在特征空间将生成特征 h_gen 与参考特征 h_ref 做加权混合：`h_injected = (1-λ_eff)·h_gen + λ_eff·h_ref`。

两种机制的空间不同：callback 操作 latents（噪声空间），hook 操作中间层激活（特征空间）。FI 选择 hook 是因为它不修改 ODE 路径（只改内部特征），与 L2 的初始噪声偏置天然正交、无冲突。

此外，inversion 阶段会绕过 `__call__` 直接调用 `pipe.transformer`（前向传播计算速度场）和 `pipe.vae`（视频编码/解码），但这属于预处理阶段，不影响生成 pipeline 的行为。

## 二、SVD Noise Prior 的设计动机与作用

### 2.1 P-Flow 论文中 SVD 的核心作用

P-Flow（论文 Section 3.3）明确指出，SVD noise prior 的**核心作用不是"引导运动方向"，而是"稳定迭代 prompt 优化的收敛"**：

- **问题**：P-Flow 的核心 pipeline 是迭代 prompt 优化——每轮用 VLM 评估生成结果与参考视频的差异，据此改写 prompt，再重新生成。但如果每轮生成使用完全随机的 z_T，则视觉效果的变化来源混杂了"prompt 变化"和"噪声随机性"两个因素，优化器无法区分，收敛困难。
- **解决**：通过 inversion + SVD 提取参考视频的时序结构作为确定性分量 η_temporal，以极小比例（α=0.001~0.004）blend 进 z_T。这提供了一个**跨迭代的稳定锚点**——每轮生成的 z_T 都包含相同的微弱确定性偏置，使生成结果的变化**主要归因于 prompt 变化而非噪声随机性**，从而稳定优化方向。
- **平衡**：`√(1-α) ≈ 0.998`，z_T 仍然 99.8% 是随机的，保留了足够的探索多样性。如果 α 过大（如固定噪声），则陷入局部最优无法探索。

### 2.2 我们实验中 SVD 的实际效果

在我们的实验设定中（非迭代优化，单次生成），SVD v1 的效果体现为：η_temporal 携带的参考视频时序结构作为 z_T 的微弱方向偏置，被 30 步 flow matching 积分过程累积放大，最终使生成视频在运动模式上与参考视频更相似。实测 CLIP +2.4%, XCLIP +5.1%（vs 纯随机噪声 baseline）。

关键发现（来自 10+ 组消融实验）：T2V 模型对 z_T 中的结构化偏离极度敏感。所有试图增强注入的改进均失败，注入量越大效果越差。α=0.004（实际方向偏移 ~2%）是纯黑盒下的绝对天花板。

## 三、当前有效策略总结

### 3.1 纯黑盒最优：原始 Caption + SVD v1

- **L1（Prompt）**：保留 VLM 原始 caption 不改写，运动描述模糊（"the boat moves"），给 SVD 留出信息补充空间
- **L2（SVD）**：两阶段 SVD 滤波（时间保运动），v1 模式不做 renorm，α=0.004 极小剂量 blend
- **结果**：CLIP 0.8964, XCLIP 0.7874（当前黑盒最优）

### 3.2 L1 有效改进：v9 删减+VLM补充

- **策略**：LLM 对原始 caption 做纯删减（去 preamble、hedging、summary），再用本地 Qwen2.5-VL-7B 看视频帧补充视觉细节，**不碰运动描述**
- **与 SVD 兼容**：不描述运动 → 不与 SVD 偏置冲突
- **结果**：XCLIP +1.3%，CLIP 持平

### 3.3 灰盒方向：Feature Injection（特征注入）

- **原理**：在 Wan2.1 DiT transformer 的指定层（mid=层10~19）挂载 PyTorch `register_forward_hook`，在每步去噪的前向传播内部，将当前生成特征 h_gen 与预缓存的参考特征 h_ref 做加权混合：`h_injected = (1-λ_eff)·h_gen + λ_eff·h_ref`
- **自适应门控**：`λ_eff = λ × gate × quality_scale`，其中 gate 基于 h_gen 与 h_ref 的余弦相似度（cos 高→gate 小，防止过度注入），quality_scale 来自 η_temporal 帧间一致性（运动一致性高的样本允许更强注入）
- **EMA 特征平滑**：跨步对参考特征做时序 EMA 平滑（ema_decay=0.7），减少相邻步注入信号的跳变
- **状态**：λ=0.05 + adaptive gate + quality_scale + EMA 平滑，当前最强 L3 方案，XCLIP +13.6%

### 问题概况
#### 1、不同场景的SVD和FI的引导量是不同的，目前做的前十个效果最佳的case集中在人物、动物运动等场景，一些空间镜头类的场景用当前的注入量效果不佳
#### 2、有些case中（大概1/5）根据prompt t2v生成的运动方向和FI注入的方向相差过大（目前采取自适应门控的方式解决，生成过程中计算二者余弦夹角，如果相差过大就不进行注入）
#### 针对上述问题想出两种解决方案：1、用默认配置过滤一遍，效果不好的case集中起来用其他的注入量再过滤一遍，删除所有配置均表现不佳的样本 2、采取自适应注入，识别当前的场景分类以及计算一些方向夹角等内容，根据现在实验的结果，根据不同种类的样本选择不同的注入量

## 五、关键发现与结论

### 5.1 SVD 的双重角色

在 P-Flow 原论文的迭代优化 pipeline 中，SVD 的核心作用是**提供跨迭代的确定性锚点**（stabilize optimization convergence）。在我们的单次生成实验中，SVD 的作用退化为**提供运动方向的微弱统计偏置**。两种作用都要求 α 极小——前者是为了保留探索多样性，后者是因为 T2V 模型无法容忍 z_T 的结构化偏离。

### 5.2 L1 与 L2 的结构性矛盾

SVD 先验本质是"运动信息补充器"——当 prompt 说不清怎么动时有效（如原始 VLM caption）。当 prompt 已精确描述运动（v7e），SVD 偏置变为干扰。正确分工：L1 只做结构优化不碰运动，L2 负责运动引导。

### 5.3 纯黑盒天花板

10+ 组消融实验证明：η_temporal 的一切信息（内容、幅度、相位）在强注入时对 T2V 模型均有害。α=0.004 是绝对天花板——信号弱到"不产生破坏"是其有效的前提条件。

### 5.4 灰盒方向是突破口

Feature Injection 绕过"只能改 z_T"的黑盒限制，通过 `register_forward_hook` 在 DiT 前向传播的**内部**修改中间层特征。与 P-Flow 原文的"仅影响起点"（L2）形成本质区别：FI 影响的是生成路径的"内部语义表示"而非噪声空间的位置，构成独立贡献。且 FI 不依赖 ref_trajectory 的位置信息，无需与 L2 的 z_T 起点对齐，天然适合与 L2 叠加。

## 六、SVD v1 详细流程

```
参考视频 V_ref
    │
    ▼
[VAE Encode] → z_0, shape [1, 16, 21, 60, 104]
    │
    ▼
[Flow Matching Inversion] → η_inv (反演噪声, t=0→t=1, Euler 30 steps)
    │   调用 pipe.transformer + pipe.vae，计算 ODE 积分的逆过程
    ▼
[SVD Stage 1: 空间去内容]
    │   reshape (C×F, H×W) → SVD → 去除 top-k 空间主成分(占 90% 能量)
    ▼
[SVD Stage 2: 时间保运动]
    │   reshape (C×H×W, F) → SVD → 只保留 top-k 时间主成分(占 90% 能量)
    │   输出 η_temporal, std ≈ 0.28~0.41
    ▼
[Noise Blend] → z_T = √α·η_temporal + √(1-α)·η_random
    │   α=0.004, 实际方向偏移 ≈ 0.063 × 0.3 ≈ 2%
    ▼
[WanPipeline.__call__(prompt, latents=z_T, ...)]
    │   30 步 flow matching 去噪
    ▼
生成视频
```

## 七、Feature Injection (FI) 自适应门控实验（6-15）

> **实验时间**：2026-06-15 19:29~20:20
>
> **核心目标**：验证 L2(SVD) + L3(FI 特征注入) 的叠加效果。FI 在 DiT 中间层注入参考视频特征，用自适应门控动态调整注入强度。

### 7.1 FI 设计原理

**特征注入（Feature Injection）**：在 Wan2.1 的 DiT transformer 中，对指定层（mid=层10~19）挂载 PyTorch forward hook，在每步去噪的前向传播中，将当前生成激活 h_gen 与预缓存的参考激活 h_ref 做加权混合：

```python
h_injected = (1 - λ_eff) * h_gen + λ_eff * h_ref
```

**自适应门控**：
```python
cos_sim = cosine_similarity(h_gen, h_ref)     # 特征余弦相似度
gate = 1.0 - sigmoid(temp * (cos_sim - 0.5))    # cos 高→gate 小
λ_eff = λ * gate * quality_scale                # 实际注入强度
```
- quality_scale：来自 η_temporal 帧间余弦相似度的软门控，运动一致性高的样本允许更强注入
- gate：特征越接近参考→注入越少，防止过度约束

### 8.4 核心发现：不同场景类型与注入量的关系

根据P-Flow论文(2603.22091v1)的分析，**α（SVD blend系数）和λ（FI注入强度）对不同类型的视频确实应该不同**：

#### 1. 论文中的关键依据

论文Table 4的超参数分析明确指出：

| 设置 | FID-VID↓ | FVD↓ | Dyn. Degree↑ |
|------|----------|------|---------------|
| w/o SVD Projection (ρs=0, ρm=1.0) | 33.25 | 1052.80 | 0.58 |
| Random Noise Only (α=0) | 32.74 | 923.67 | 0.73 |
| Enhanced Noise (α=0.001, ρs=0.1, ρm=0.9) | 29.32 | 784.51 | **0.94** |
| Enhanced Noise (α=0.01, ρs=0.1, ρm=0.9) | 29.21 | 803.35 | 0.88 |

**论文默认 α=0.001**，而我们的实验用的是 **α=0.004**（4倍于论文默认值）！

论文还指出：
- α=0.001：运动动态最佳(Dyn=0.94)，FVD最低
- α=0.01：外观保真度略好(FID-VID更低)，但运动动态下降(Dyn=0.88)
- **α越大，temporal noise影响越强，但可能过度偏移导致与prompt运动方向冲突**

#### 2. 不同场景类型的最优α不同

| 场景类型 | 最优α | 原因 |
|----------|-------|------|
| **物体运动型** (动物、人物、物理) | α=0.003~0.005 | SVD提取的运动方向与prompt一致，可以适当放大 |
| **场景/镜头运动型** (FPV、航拍、隧道) | α=0.0005~0.001 | SVD提取的运动方向可能与prompt冲突，需极度保守 |
| **异常活动型** (纸飞机、火焰精灵) | α=0.001~0.002 | 运动语义较抽象，SVD提取方向不确定 |

#### 3. 不同场景类型的最优λ不同

| 场景类型 | 最优λ | 原因 |
|----------|-------|------|
| **物体运动型** | λ=0.05 (当前) | mid层特征与prompt语义一致，注入有效 |
| **场景/镜头运动型** | λ=0.02~0.03 | mid层外观特征与运动prompt冲突，需降低或换层 |
| **异常活动型** | λ=0.03~0.04 | 运动语义抽象，中等注入最安全 |

#### 4. 论文的核心洞察

论文Section 3.3明确指出：**η_temporal 保留了dominant motion information，同时抑制了static和appearance-dependent details**。这意味着：

- 对于**物体运动型**视频：dominant motion = 物体运动方向 → η_temporal与prompt一致 → α可以大
- 对于**场景/镜头运动型**视频：dominant motion = 镜头/场景变化 → η_temporal可能与prompt描述的物体运动冲突 → α必须小

论文选择 α=0.001 是因为它在Open-VFX数据集上达到最佳平衡。但Open-VFX主要是**视觉特效**（爆炸、压缩、膨胀），运动模式与物体运动型视频更接近。我们的数据集包含大量**场景穿越型**视频，运动模式完全不同。

#### 5. 为什么场景型样本 α 太大就崩？

论文 Section 3.3 指出 η_temporal 保留了 dominant motion information。核心问题在于：

- **物体运动型**（动物、人类、物理）：dominant motion = 物体移动方向 → 与 prompt 一致 → α 可以大
- **场景/镜头运动型**（FPV、航拍、隧道）：dominant motion = 镜头/场景变化方向 → 与 prompt 描述的运动语义冲突 → α 必须小

这就是为什么后20样本中 73（-0.129）、80（-0.140）、105（-0.093）这些场景型样本的 SVD 增益大幅为负——α=0.004 把错误的运动方向注入得太强了。

#### 6. 深入分析：日志中的 direction_shift 证据

从 run_log.txt 可以提取每个样本的 `direction_shift` 指标（越大说明 temporal noise 对混合噪声的影响越大）：

| 样本 | direction_shift | SVD ΔXC | 类型 |
|------|----------------|---------|------|
| 73 | 0.0223 | -0.1290 | scene |
| 80 | 0.0187 | -0.1396 | scene+camera |
| 105 | 0.0215 | -0.0932 | scene+camera |
| 111 | 0.0251 | +0.2791 | scene+camera (异常高) |
| 32 | 0.0198 | +0.1133 | animal |
| 33 | 0.0201 | +0.1981 | human |

**关键发现**: direction_shift 在0.018~0.025之间，看起来数值差异不大，但SVD效果天壤之别。这说明问题不在于temporal noise的强度，而在于其**方向**是否与prompt一致。direction_shift 相近的情况下：
- 物体运动型：temporal方向与prompt一致 → 正增益
- 场景型：temporal方向与prompt冲突 → 负增益

因此单纯靠 direction_shift 不能区分好坏，需要额外的方向对齐度指标（如 mean_cos 或 prompt-motion cosine similarity）。

#### 7. 论文迭代优化 vs 我们的单次生成

**重要差异**: 论文 P-Flow 的框架是**迭代优化**（10次迭代），每次生成后用VLM分析差异再优化prompt。而我们的实验是**单次生成**（无迭代）。这意味着：

- 论文中即使α偏大，后续迭代可以通过prompt调整来弥补
- 我们没有这种纠错机制，α的选择必须更保守
- **结论**: 我们应该用比论文更小的α，而不是更大

### 9.7 核心发现

#### 1. 🔥 低α（0.001）整体优于默认α（0.004）

- Exp2（α=.001+FI）CLIP均值0.9160 > 默认0.9147，XCLIP均值0.7429 > 默认（纯SVD+FI约0.73+）
- α从0.004降到0.001，对CLIP指标有0.1%~0.8%的提升
- 但对XCLIP指标的影响因样本而异，并非简单的"越小越好"

#### 2. 🎯 不同样本对α的敏感度差异巨大（最核心发现！）

- **Sample 73（仓库/室内场景）**：α从0.001→0.002，XCLIP暴跌0.075！说明场景型视频对时序噪声极其敏感
- **Sample 111（森林航拍）**：α增大反而提升0.086！说明快速镜头运动可能需要更多时序先验
- **Sample 32（动物）**：几乎不受α影响，物体运动的视频自带较好的时序结构
- **结论：不存在全局最优α，必须自适应**

#### 3. 📉 单纯降α并不能修复所有场景型负增益

- Sample 73：即使α降到0.001，XCLIP仍为0.6435（Baseline=0.7535），负增益-0.110仅比默认-0.103略好
- Sample 80：α=0.002+FI时XCLIP=0.7149，比默认0.6622提升0.053，但仍低于Baseline
- **根本原因**：场景型视频的SVD temporal方向与prompt运动语义存在冲突，降α只是减轻伤害，不能扭转方向

#### 4. ⚡ FI λ=0.05→0.03 几乎无差别

- Exp2(λ=0.05) vs Exp5(λ=0.03)：CLIP几乎相同，XCLIP微弱下降0.0017
- 原因：自适应门控已将有效λ压到0.005-0.01，调λ_max边际效果极小

#### 5. 🔬 Sample 111 是关键例外

- 111是"场景+镜头运动"型但SVD增益极高（+0.283）
- 独特之处：森林航拍视频的dominant motion就是镜头运动本身，与prompt描述一致
- α=0.002 SVD only时XCLIP最高(0.7900)，说明其temporal信号方向正确，可以承受更大的α

### 9.8 原因深度分析

#### 为什么α对不同视频效果差异如此大？

**根本原因：SVD提取的η_temporal代表的是"dominant motion direction"，而不是"desired motion direction"**

1. **物体运动型（32-动物）**：dominant motion = 物体移动 → 与prompt"两只金毛在雪地玩耍"一致 → α大也不伤
2. **场景静止型（73-仓库）**：dominant motion ≈ 0（室内几乎无运动）→ η_temporal主要是噪声 → α越大注入越多噪声 → 严重破坏
3. **场景+快速镜头（111-航拍）**：dominant motion = 镜头运动 → 与prompt"无人机在森林上方飞行"一致 → α越大效果越好
4. **场景+慢速镜头（80-蒲公英）**：dominant motion = 微弱的镜头推近 → 与prompt"蒲公英种子头部特写"弱相关 → 中等α最佳

**数学解释**：

η_mixed = √(1-α)·η_random + √α·η_temporal

当η_temporal方向与prompt运动方向一致时：α↑ → 生成更遵循原始运动 → XC↑
当η_temporal方向与prompt运动方向冲突时：α↑ → 生成偏离prompt运动指令 → XC↓
当η_temporal信号弱（≈噪声）时：α↑ → 注入随机时序噪声 → 破坏时序一致性

**为什么单纯降α不能完全解决问题？**

73号样本在α=0.001时XC=0.6435仍远低于Baseline=0.7535，说明即使极小α，SVD still注入了有害的时序方向。问题本质不是"注入量太多"，而是"注入方向错误"。自适应α能减轻但不能消除这个问题。

#### 为什么mean_cos不是完美的α预测指标？

- mean_cos衡量的是FI特征空间中生成特征与参考特征的相似度
- 它反映的是"外观保真度"，而不是"运动方向正确性"
- Sample 111的mean_cos=0.067（不高），但α增大反而更好，因为其运动方向与prompt一致
- **真正需要的指标是：η_temporal方向与prompt运动语义的对齐度**

## 十、Phase 2: Spatial Blend (β) 消融实验（2026-06-18）

> **实验时间**：2026-06-18 09:00~09:42
>
> **核心目标**：验证 SVD Stage 1 分离的外观分量 η_spatial 对"完全复原原视频"是否有帮助

### 10.1 动机

SVD 两阶段滤波中，Stage 1 去掉了外观/内容信息（η_spatial），只保留 Stage 2 的时间运动信息（η_temporal）。对于"运动迁移"场景，丢弃外观是正确的；但对于"完全复原原视频"场景，外观信息应该有用。通过 β 参数将 η_spatial 回注混合噪声：

```
η = √α · η_temporal + √β · η_spatial + √(1-α-β) · η_random
```

### 10.2 关键发现：η_spatial 量级不匹配问题

首次实验（β=0.001, 无 renorm）结果**全线崩溃**：

| 指标 | β=0 基线 | β=0.001 无 renorm | 变化 |
|------|:---:|:---:|:---:|
| CLIP | 0.9037 | 0.8284 | **-8.3%** ❌❌ |
| XCLIP | 0.8138 | 0.6897 | **-15.2%** ❌❌❌ |

**根本原因**：η_spatial 的 std (~0.9~1.2) 远大于 η_temporal 的 std (~0.28~0.41)

```
修复前的实际注入量:
  √α × η_temporal = 0.0632 × 0.35 = 0.022  (运动)
  √β × η_spatial  = 0.0316 × 1.0  = 0.032  (外观) ← 反而更大!

修复后 (renorm η_spatial 使其 std 与 η_temporal 一致):
  √α × η_temporal          = 0.0632 × 0.35 = 0.022  (运动)
  √β × η_spatial_renormed = 0.0316 × 0.35 = 0.011  (外观) ← 正确地更小
```

**修复**：注入 η_spatial 前将其 std renorm 到与 η_temporal 一致，使 α 和 β 的语义对等。

## 十一、下一步计划

### 11.1 短期：β 增量验证

β=0.001 整体正增益但太微弱，需要验证更大 β 的效果：

| 实验 | β值 | α值 | 总注入 | 预期 |
|------|------|------|--------|------|

| B | 0.003 | 0.004 | 0.007 | 较强外观注入，可能开始对 46 号等场景型样本产生伤害 |
| C | 0.002 | 0.002 | 0.004 | 运动和外观各一半，总注入不变 |

### 11.2 中期：TSR-guided 自适应 α

Phase 1 已证明不同样本的最优 α 差异巨大（73号 α 极度敏感 vs 32号 α 不敏感）。TSR (Temporal Signal Reliability) 可以自动调节 α：

```
α_adaptive = α_min + TSR × (α_max - α_min)
```

- 高 TSR（动物奔跑、航拍飞行）→ α 大 → 运动注入强
- 低 TSR（静态室内、微弱镜头）→ α 小甚至 0 → 避免注入有害噪声

代码已实现，需要跑实验验证。

### 11.3 长期方向：场景感知的注入策略

当前 β 对所有样本统一注入外观，但实际效果因场景而异。未来可能需要：

1. **外观注入也做门控**：基于 η_spatial 与 prompt embedding 的相关性，只注入与生成目标一致的外观信息
2. **FI 层级选择**：场景型样本可能需要换层注入（如 early 层更多外观细节）
3. **联合 α-β 优化**：让 α（运动）和 β（外观）都由 TSR 之类的指标自适应决定
