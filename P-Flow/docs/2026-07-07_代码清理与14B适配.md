## 二、FI 注入机制详解

### 2.1 注入层范围

FI 注入 DiT 中间 1/3 层的 cross-attention 输出：

| 模型 | DiT 总层 | mid 范围 | 注入层数 | 占比 |
|------|:--:|------|:--:|:--:|
| 1.3B | 30 | 10~19 | 10 | 33% |
| 14B | 40 | 13~26 | 13 | 33% |

- **早期层(0-9/12)**：边缘、纹理 — 不注入，交 T2V 自由生成
- **中期层(10-19/13-26)**：物体身份、结构、运动模式 — **FI 注入**
- **晚期层(20-29/27-39)**：高频细节 — 不注入，避免与 prompt 冲突

### 2.2 时间步分布（middle_peak 调度）

30 步生成过程中，FI 注入强度按正弦曲线分布：

```
λ_sched(i) = fi_lambda × sin(π×i/29)

early 0-9:  avg_sched=0.0223, 占 FI 总能量 24.2%
mid  10-19: avg_sched=0.0476, 占 FI 总能量 51.6%
late 20-29: avg_sched=0.0223, 占 FI 总能量 24.2%
```

峰值在 step 14-15（λ=0.05），首尾归零。

### 2.3 反演缓存 → 生成注入（完整链路）

```
反演 50 步:
  ├─ eta_inv (x_t 终值)      → SVD 输入 → η_temporal → z_T
  ├─ trajectory {t: x_t}      → FI 回放备用
  └─ fi_ref_features          → FI 核心缓存
      └─ {gen_step_0: {layer10: tensor, ..., layer19: tensor}, ...}
         30 steps × 10 layers = 300 tensors (1.3B)

生成 30 步:
  每步 callback:
    step_features = ref_features[step_index]
    for layer in [10..19]:
        h_ref = step_features[layer]    ← 从缓存取出
        h_gen = cross_attn(latent, t)   ← 当前生成
        cos = cosine_sim(h_gen, h_ref)
        gate = 1 - sigmoid(5×(cos-0.5))  ← 余弦门控 (保留语义)
        λ = sin_schedule × gate × QS    ← 三层门控
        h_out = (1-λ)·h_gen + λ·h_ref   ← 残差注入
```

关键数据（204 样本统计）：
- 缓存 300 tensors/样本，生成时 100% 使用（零浪费）
- with_ref: 30.0/30 步 (100%)，no_ref: 0.0/30 步 (0%)
- QS 范围 [0.109, 0.867]，均值 0.493，永不归零

### 2.4 FI 注入量 vs 指标的帕累托关系

```
FI↑ → SSIM↑/LPIPS↓/DINO↑（低层感知好）
     → CLIP↓/XCLIP↓       （语义退化）

三层门控设计在两者间取平衡：
  sin_schedule: 中段语义步最强
  cosine_gate:  特征相似时主动衰减（26% 步 cos>0.7，gate→0.18-0.27）
  QS:           原运动弱时保守注入
```

## 三、14B 内存适配

### 3.1 问题

14B 在单 A800(80GB) 上反演 OOM，原因：

| 资源 | 占用量 | 上限 |
|------|--------|------|
| GPU 显存 | 模型 38GB + 激活 ~9GB = ~47GB | 80GB ✅ |
| CPU 内存 | FI 缓存 13层×30步 = ~130GB | 120GB ❌ |

### 3.2 尝试的方案

| 方案 | 效果 | 结果 |
|------|------|------|
| 原地更新 `x_t.add_()` | 减少 GPU 碎片 | GPU 稳住了，CPU 仍 OOM |
| `attention_slicing` | 降低 forward 峰值 | 有效但还不够 |
| FI 落盘 `torch.save()` | 省 CPU 用磁盘 | 系统盘/数据盘均满 |
| RAM 预算 + 轨迹回放 | 预算内缓存+按需补齐 | 回放时 CPU 仍超 |
| 生成时按需回放 | 单步用完即释放 | 复杂度过高 |

### 3.3 最终方案：时间步截断

**保持 mid=1/3 层原则不变**，仅缓存前 23/30 生成步。

```python
# 配置
fi_max_cache_step = 23  # 默认 30 (1.3B 不受影响)

# 效果
缓存量: 23×13 = 299 tensors ≈ 100GB ✅ 120GB 内存够
FI 能量: 保留 75.8%（丢弃的步骤 24-29 仅贡献 24.2% 能量）
后期余弦门控本身也衰减，丢步影响极小
```

新增参数：
- `PFlowConfig.fi_max_cache_step = 30`
- `--fi_max_cache_step` CLI 参数
- `configs/default.toml` 中 `max_cache_step = 30`

## 五、当前架构总览

```
P-Flow 三层架构:

L1 (Prompt):      LLM 场景理解 → S2/S3 改写 → 原始vs改写二版本择优
L2 (SVD 噪声先验): 反演50步 → 两阶段SVD(ρ_s=0.1,ρ_m=0.9) → sigmoid自适应α → 渐进多尺度+回退
L3 (FI 特征注入):  mid 1/3层 cross-attn → 三层门控:
    ├─ 中峰调度(sin)
    ├─ 余弦门控(τ=5.0, 保语义)
    └─ QS(k=20.0, 视频级自适应)

核心超参 (3个): lambda=0.05, k=20.0, tau=5.0
零训练，模型无关
```
