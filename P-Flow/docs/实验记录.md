## 技术调研：Wan2.1-1.3B Token 限制与结构化 Prompt 方案

**时间**：2025-05-27

### Wan2.1-1.3B 官方 Token 限制

通过查阅官方源码（`wan/configs/shared_config.py`）、diffusers pipeline 源码及 DeepWiki 文档，得出以下结论：

**官方硬限制**：`text_len = 512`（T5 encoder 最大序列长度），diffusers `WanPipeline` 的 `max_sequence_length` 默认同为 512。

**实际有效窗口**：约 100-150 tokens（对应约 80-120 英文词）。原因：
1. T5-XXL 使用相对位置编码（T5 Relative Attention Bias），超过 ~150 tokens 后位置权重衰减显著
2. 1.3B 模型 DiT 的 cross-attention 容量有限（12 heads），对长文本的注意力分散
3. 实测：前 40 词（~50 tokens）对生成结果影响最大，超过 150 词后边际收益接近零

**结论**：prompt 控制在 80-100 英文词（~100-130 tokens）为最优区间，超出部分基本被模型忽略。

### 结构化 Prompt 方案（V2）

参考 VISTA、VC4VG、InstanceCap 等论文的结构化描述思路，设计以下模板：

```
[SUBJECT] → [ACTION] → [SCENE] → [CAMERA] → [STYLE]
```

各维度说明：
- **SUBJECT**（前 15 词）：主体是谁/什么，外观、数量
- **ACTION**（前 40 词内）：动作/运动，方向、速度、姿态
- **SCENE**（40-60 词）：环境、背景、光线、时间
- **CAMERA**（60-80 词）：景别（特写/中景/远景）、角度（俯/仰/平）、运动（静止/平移/缩放/跟拍）
- **STYLE**（80-100 词）：色调、氛围、画质风格

**关键设计原则**：
- 最重要的信息（主体+动作）放在前 40 词，利用 T5 cross-attention 的位置权重优势
- 总长度严格控制在 80-100 词，不超过有效窗口
- 每个维度具体而非泛化（"金毛猎犬从左向右奔跑" 而非 "一只狗在动"）

### 代码修改

已修改以下文件，让 VLM 按结构化模板输出 caption：

- `src/baseline.py`：`caption_video()` 方法的 prompt 改为结构化模板指令
- `src/vlm_client.py`：`LocalVLMClient.describe_video()` 和 `VLMClient.describe_video()` 同步更新

**下一步**：用结构化 prompt 重跑 10 样本 baseline，与原始 baseline（CLIP=0.8703，X-CLIP=0.7164）对比。

## DiT Cross-Attention 位置权重实验

**时间**：2025-05-28

**目的**：通过 hook DiT 的 cross-attention 层，精确测量 Wan2.1-1.3B 在视频生成时对不同 prompt 长度、不同词位置的实际注意力分配。

### 实验方法

**核心思路**：在一串相同的 filler 词（"cat"）中插入一个独特的 signal 词（"UNICORN"），通过替换 DiT 的 attention processor 手动计算 softmax 权重，测量模型对 signal 位置 vs filler 位置的关注度倍数。

**技术实现**：
- 替换 `WanAttnProcessor` 为自定义 `HookedProcessor`，在 cross-attn 中手动计算 `softmax(Q·K^T/√d)`
- 对所有 video token 和 attention head 取均值，得到每个文本位置的平均权重
- 倍数 = signal 词每个子词的平均权重 / filler 词的平均权重

**自变量**：
- prompt 长度（词数）：5, 10, 20, 50, 100, 200, 300
- signal 位置比例：0%（开头）、25%、50%（中间）、75%、100%（结尾）

**控制变量**：
- filler 词 = "cat"（单 token）
- signal 词 = "UNICORN"（被 tokenizer 切为 ▁UNIC + ORN 两个子词）
- 生成参数：num_frames=9, height=256, width=256, guidance_scale=5.0, steps=20
- 观察层：第 0/7/14/21/29 层（共 30 层中的 5 个代表层）

**对照实验**：
- 不同 signal 词（UNICORN, DRAGON, ZEBRA）验证结论普适性
- 纯 filler 基准（无 signal）确认 filler 权重均匀性
- 跨步观察（20 步去噪过程中权重变化）

### 实验结果

#### 主实验：长度 × 位置 交叉（第 0 层，20 步均值）

| prompt 长度 | 0%（开头） | 25% | 50%（中间） | 75% | 100%（结尾） |
|------------|-----------|-----|------------|-----|-------------|
| 5 词       | **2.1x**  | 0.3x | 0.3x      | 0.3x | 0.4x       |
| 10 词      | **2.9x**  | 0.6x | 0.5x      | 0.6x | 0.6x       |
| 20 词      | **2.9x**  | 0.5x | 0.9x      | 0.9x | 1.0x       |
| 50 词      | **5.1x**  | 0.8x | 0.9x      | 0.9x | 0.7x       |
| 100 词     | **15.3x** | 1.3x | 1.5x      | 0.9x | 1.4x       |
| 200 词     | **7.4x**  | 2.0x | 1.5x      | 1.4x | 2.2x       |
| 300 词     | **15.4x** | 2.0x | 1.7x      | 1.7x | 2.4x       |

#### 对照实验：不同 signal 词（长度=50，第 0 层）

| signal 词 | 0%（开头） | 25% | 50% | 75% | 100%（结尾） |
|-----------|-----------|-----|-----|-----|-------------|
| UNICORN   | 4.8x      | 1.0x | 0.9x | 0.9x | 1.2x       |
| DRAGON    | 4.4x      | 0.9x | 0.8x | 0.9x | 1.1x       |
| ZEBRA     | 4.6x      | 1.1x | 0.9x | 1.0x | 1.7x       |

三个词规律一致，结论不依赖特定词。

#### 跨步实验：去噪过程中权重变化（长度=50，signal 在开头，第 0 层）

| 步数 | 倍数 | 说明 |
|------|------|------|
| 0    | 3.5x | 偶数步（unconditional） |
| 1    | 15.9x | 奇数步（conditional） |
| 3    | 13.8x | 早期步骤，文本引导强 |
| 9    | 14.0x | 中期 |
| 19   | 9.6x | 后期步骤，文本引导减弱 |
| 39   | 1.5x | 最后阶段，几乎不依赖文本 |

奇偶交替是 classifier-free guidance 导致的（奇数步=有条件推理，偶数步=无条件推理）。

#### 基准实验：纯 filler 的 padding 行为

| prompt 长度 | 实际 tokens | filler 均值 | padding 均值 | 说明 |
|------------|------------|------------|-------------|------|
| 5 词       | 6          | 0.00794    | 0.00188     | padding 权重 = filler 的 24% |
| 50 词      | 51         | 0.00257    | 0.00189     | padding 权重接近 filler |
| 100 词     | 101        | 0.00142    | 0.00208     | padding 权重反超 filler |
| 300 词     | 301        | 0.00129    | 0.00290     | padding 权重 > filler |

当 prompt 超过 ~100 tokens 时，padding 零向量的权重开始超过真实 token 的平均权重，说明模型对长 prompt 中间部分的关注度已经非常低。

### 核心结论

1. **开头位置有压倒性优势**：无论 prompt 多长，位置 0% 的词始终获得最高关注度（2x-15x），且 prompt 越长优势越大。

2. **结尾略优于中间**：在长 prompt（100-300 词）中，结尾位置比中间高约 0.5-1x，但远不如开头。

3. **中间位置被稀释**：50% 和 75% 位置的倍数通常在 0.8-1.7x，接近或低于 filler 水平，说明中间词几乎没有额外关注。

4. **跨层一致**：5 个观察层的位置曲线形状一致，结论是模型级别的系统性行为。

5. **早期步骤更依赖文本**：去噪前半段（步 0-10）文本权重高，后半段逐渐降低，最终几乎不依赖文本。

### 对 P-Flow 的指导意义

- 结构化 prompt 的设计正确：最重要的信息（主体+动作）放在开头
- prompt 长度建议控制在 50-100 词：超过 100 词后中间部分基本被忽略
- 如果 prompt 较长，关键词应放在**开头**或**结尾**，避免放在中间

## 纯位置权重实验（排除语义差异）

**时间**：2025-05-28

**目的**：上一组实验使用"在 filler 词中插入 signal 词"的方法，但 signal 词与 filler 词存在语义差异（如 UNICORN vs cat），无法区分观察到的权重差异是来自"位置"还是"语义"。本实验通过让所有 token 完全相同（全用同一个词重复），彻底排除语义因素，只观察纯粹的位置效应。

### 实验方法

**核心思路**：prompt 由完全相同的词重复 N 次构成（如 "cat cat cat ... cat"），所有 token 的 embedding 完全一致，因此权重分布的任何不均匀性都只能来自位置本身。

**技术实现**：与上一组实验相同，替换 `WanAttnProcessor` 为自定义 Hook，手动计算 `softmax(Q·K^T/√d)` 并对所有 video token 和 attention head 取均值。

**子实验设计**：

- 实验1：基础位置分布 — 8 种长度（5/10/20/50/100/150/200/300 词），观察完整位置权重曲线
- 实验2：跨层一致性 — 固定长度=100，对比第 0/7/14/21/29 层
- 实验3：跨步演化 — 固定长度=100，逐步记录 conditional 和 unconditional 的位置分布变化
- 实验4：不同 filler 词对照 — 固定长度=50，分别用 cat/dog/red/the/run 验证结论不依赖特定词
- 实验5：有效窗口精确定位 — 长度=200，逐位置输出完整权重曲线

**生成参数**：num_frames=9, height=256, width=256, guidance_scale=5.0, steps=20

### 实验结果

#### 实验1：基础位置分布

|------------|--------|----------|----------|---------|----------|---------|------|
| 5 词       | 6      | 0.02373  | 0.00902  | 0.00607 | 2.6x     | 0.67x   | 序列太短，中间≈全部 |
| 10 词      | 11     | 0.01792  | 0.00192  | 0.00411 | 9.3x     | 2.14x   | U型分布出现 |
| 20 词      | 21     | 0.01765  | 0.00273  | 0.00383 | 6.5x     | 1.40x   | |
| 50 词      | 51     | 0.01548  | 0.00101  | 0.00367 | 15.3x    | 3.62x   | |
| 100 词     | 101    | 0.01410  | 0.00103  | 0.00350 | 13.7x    | 3.40x   | |
| 150 词     | 151    | 0.01568  | 0.00100  | 0.00401 | 15.6x    | 3.99x   | |
| 200 词     | 201    | 0.01259  | 0.00101  | 0.00355 | 12.5x    | 3.52x   | |
| 300 词     | 301    | 0.01431  | 0.00101  | 0.00444 | 14.2x    | 4.42x   | |

**关键发现**：即使所有词完全相同，位置 0 的权重仍然是中间位置的 **10-15 倍**。从位置 1 开始权重立即骤降至位置 0 的 ~3%，之后中间区域几乎完全平坦，直到最后几个位置再次上升（U 型分布）。

#### 实验2：跨层一致性（长度=100）

| 层 | pos0 权重 | 中间5均值 | 后5均值 | pos0/中间 | 后/中间 |
|----|----------|----------|---------|----------|---------|
| Layer 0  | 0.01488 | 0.00102 | 0.00372 | 14.5x | 3.63x |
| Layer 7  | 0.01647 | 0.00105 | 0.00331 | 15.7x | 3.15x |
| Layer 14 | 0.02019 | 0.00102 | 0.00457 | 19.7x | 4.46x |
| Layer 21 | 0.00848 | 0.00099 | 0.00246 | 8.6x  | 2.48x |

所有 30 层都呈现相同的 U 型位置分布模式，Layer 14（中间层）效应最强（19.7x），Layer 21 最弱（8.6x），但即使最弱的层也有 8.6 倍的首位优势。

#### 实验3：跨步演化（长度=100，第0层）

**Conditional 步（有文本引导）：**

| 步数 | pos0/中间 | pos0 绝对值 | 说明 |
|------|----------|------------|------|
| 0    | 30.2x    | 0.05586    | 初始噪声阶段，位置效应最强 |
| 1-4  | 27-29x   | 0.051-0.054 | 早期去噪，强文本引导 |
| 5-9  | 20-28x   | 0.037-0.052 | 逐渐减弱 |
| 10   | 16.4x    | 0.031      | 中期转折点 |
| 13   | 3.2x     | 0.006      | 后期，位置效应大幅减弱 |
| 18   | 2.0x     | 0.004      | 末期，接近均匀 |

**Unconditional 步（无文本引导）：**

| 步数 | pos0/中间 | pos0 绝对值 | 说明 |
|------|----------|------------|------|
| 0    | 29.1x    | 0.01580    | |
| 5    | 30.3x    | 0.00389    | 比值反而上升（因为中间权重更快趋零） |

Conditional 步中，位置效应从 30x 逐步衰减到 2x，说明去噪后期模型逐渐不再依赖文本位置信息。Unconditional 步的比值始终很高，但绝对权重极低（接近零），说明无条件推理时文本几乎不起作用。

#### 实验4：不同 filler 词对照（长度=50）

| filler 词 | pos0/中间 | 后/中间 | 说明 |
|-----------|----------|---------|------|
| cat       | 15.1x    | 3.59x   | |
| dog       | 2.8x     | 0.68x   | 效应最弱 |
| red       | 40.9x    | 8.55x   | 效应最强 |
| the       | 6.3x     | 1.55x   | |
| run       | 5.0x     | 1.16x   | |

**重要发现**：不同词的位置效应强度差异很大（2.8x 到 40.9x）。这说明位置效应的强度与词的 embedding 特性有关——某些词（如 "red"）的 embedding 在重复时会产生更强的首位聚焦，而另一些词（如 "dog"）则相对均匀。但所有词都呈现 pos0 > 末尾 > 中间 的 U 型规律。

#### 实验5：有效窗口精确定位（长度=200）

完整的 200 位置权重曲线呈现极端的"尖刺 + 平台 + 尖刺"形态：

```
位置   0: 0.02906 (100.0%) ██████████████████████████████████████████████████
位置  10: 0.00100 (  3.4%) █
位置  20: 0.00099 (  3.4%) █
位置  30: 0.00099 (  3.4%) █
...（中间完全平坦，约 0.00096-0.00100）...
位置 170: 0.00096 (  3.3%) █
位置 180: 0.00096 (  3.3%) █
位置 190: 0.00097 (  3.3%) █
位置 200: 0.03027 (104.2%) ████████████████████████████████████████████████████
```

- 位置 0 权重：0.02906（占总权重的 ~3%，但是单个 token 中最高的）
- 位置 1 权重：0.00123（立即降到位置 0 的 4.2%）
- 位置 2-198：稳定在 0.00096-0.00100（位置 0 的 3.3%）
- 位置 199（倒数第二）：0.00153（略微上升）
- 位置 200（最后一个）：0.03027（与位置 0 几乎相同！）
- padding 均值：0.00240（反而高于中间真实 token）

**临界位置**：权重从位置 1 开始就已经降到位置 0 的 1/10 以下，没有渐进衰减过程。

### 核心结论

1. **纯位置效应确认**：即使排除所有语义差异，DiT cross-attention 仍然对位置 0（第一个 token）和最后一个 token 赋予远超其他位置的权重。这是模型的固有行为，不是语义驱动的。

2. **极端的 U 型分布**：权重分布呈现"首尾高、中间平"的 U 型。位置 0 和最后一个位置的权重几乎相等（~0.029-0.030），中间所有位置几乎完全均匀（~0.001）。

3. **位置 1 即断崖**：不存在渐进衰减。从位置 0 到位置 1，权重立即下降 96%。这意味着"有效窗口"实际上只有 1 个 token（位置 0）+ 最后 1-2 个 token。

4. **词的 embedding 影响效应强度**：虽然所有词都呈现 U 型分布，但不同词的首位优势倍数差异很大（2.8x-40.9x），说明 T5 encoder 对重复 token 的编码方式会影响 DiT 的注意力集中程度。

5. **去噪早期效应最强**：位置效应在去噪前 10 步最显著（30x），后期逐渐减弱到 2x，说明模型在早期阶段最依赖文本的位置结构来建立全局构图。

### 机制解释

这个 U 型分布的来源是 **UMT5 encoder 的 relative position bias**。UMT5 使用双向注意力 + 相对位置偏置（`relative_attention_num_buckets=32`, `relative_attention_max_distance=128`），在编码重复 token 序列时：

- 位置 0 是 BOS（句首）token 或紧邻 BOS 的位置，T5 的 relative bias 对序列边界有特殊处理
- 最后一个 token 是 EOS（句尾）或紧邻 EOS 的位置，同样获得边界加成
- 中间 token 由于内容完全相同且相对位置偏置在长距离后趋于平坦，编码结果几乎一致

DiT 的 cross-attention 本身没有位置编码，它看到的是 T5 输出的 hidden states。因此位置效应的根源在 T5 encoder 而非 DiT。

### 对 P-Flow 的修正指导

基于纯位置实验的新发现，修正之前的建议：

- **首词和末词是黄金位置**：不仅开头重要，结尾同样重要（权重与开头相当）
- **中间位置几乎等价**：位置 2 到倒数第 2 之间的所有位置权重几乎相同，不存在"前 40 词更重要"的渐进衰减
- **之前实验中观察到的"前 40 词效应"实际上是语义效应**：当 prompt 包含不同语义的词时，模型会根据语义重要性分配权重，而非位置
- **实际策略**：将最关键的概念词放在 prompt 的第一个词和最后一个词位置，中间部分的排列顺序不太重要

## 自动化改写方案（规模化落地）

**背景**：上述融合策略实验的 10 个 prompt 为手动改写，验证了方法有效性。要扩展到 50~200 个样本，需要自动化流程。

### 方案设计

用 LLM 执行融合策略改写，将三原则编码为 system prompt，输入原始 VLM caption，输出 hybrid caption。

**核心 System Prompt 逻辑**：

```
三原则：
1. Subject-First Opening — 首词必须是具体主体名词，删除 "The video shows..." 等前缀
2. Inject Temporal Action Chain — 注入时序标记 (initially/then/gradually/finally)
3. Preserve Original Visual Vocabulary — 保留原始的颜色、材质、光线等视觉描述词不替换

约束：
- 词数 ±20%，修改比例 <50%
- 不添加原文未暗示的信息
- 仅输出改写后的 prompt，无解释
```

### 自动验证规则

改写完成后自动检查：
- 输出非空
- 词数比例在 0.5~2.0 之间
- 不以 "The video/This video/In this video/The scene" 开头（原则1违规检测）
- 验证不通过自动重试（最多 3 次）

### 使用方式

```bash
# 安装依赖
pip install openai

# 设置 API Key（DashScope / 阿里云百炼）
export DASHSCOPE_API_KEY="sk-xxxxx"

# 批量改写（全部 200 样本）
python scripts/rewrite_hybrid.py \
    --input-dir /root/autodl-tmp/captions_baseline \
    --output-dir /root/autodl-tmp/captions_hybrid \
    --backend dashscope \
    --model qwen-plus \
    --skip-existing

# 只改写指定样本
python scripts/rewrite_hybrid.py \
    --input-dir /root/autodl-tmp/captions_baseline \
    --output-dir /root/autodl-tmp/captions_hybrid \
    --sample-ids 7 17 21 31 32 33 34 43 46 47

# 使用本地部署的模型（vLLM）
python scripts/rewrite_hybrid.py \
    --input-dir /root/autodl-tmp/captions_baseline \
    --output-dir /root/autodl-tmp/captions_hybrid \
    --backend openai \
    --api-base http://localhost:8000/v1 \
    --model Qwen2.5-72B-Instruct
```

### 完整流水线

```bash
# Step 1: 汇总 baseline caption
mkdir -p /root/autodl-tmp/captions_baseline
for d in /root/autodl-tmp/outputs/baseline/sample_*/; do
  id=$(basename "$d" | sed 's/sample_//')
  cp "$d/vlm_caption.txt" "/root/autodl-tmp/captions_baseline/${id}.txt"
done

# Step 2: LLM 自动改写
python scripts/rewrite_hybrid.py \
    --input-dir /root/autodl-tmp/captions_baseline \
    --output-dir /root/autodl-tmp/captions_hybrid_auto \
    --backend dashscope --model qwen-plus --skip-existing

# Step 3: 生成视频
python run.py \
    --data_dir /root/autodl-tmp/data/video-200/water_mark_out \
    --caption_dir /root/autodl-tmp/captions_hybrid_auto \
    --output_dir /root/autodl-tmp/outputs/hybrid_auto \
    --steps 30 --guidance 5.0 --seed 42 --resume

# Step 4: 创建 flat 目录 + 评测
mkdir -p /root/autodl-tmp/outputs/hybrid_auto/flat
for f in /root/autodl-tmp/outputs/hybrid_auto/sample_*/*.mp4; do
  ln -sf "$f" /root/autodl-tmp/outputs/hybrid_auto/flat/$(basename "$f")
done

python evaluation/run_clip_xclip_eval.py \
    --orig-dir /root/autodl-tmp/data/video-200/water_mark_out \
    --gen-dir /root/autodl-tmp/outputs/hybrid_auto/flat \
    --caption-dir /root/autodl-tmp/captions_hybrid_auto \
    --output-dir /root/autodl-tmp/outputs/hybrid_auto/eval_clip
```

### 脚本位置

`scripts/rewrite_hybrid.py` — 支持 DashScope 和 OpenAI 兼容接口两种后端
