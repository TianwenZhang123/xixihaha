## 技术调研：Wan2.1-1.3B Token 限制与结构化 Prompt 方案

**时间**：2025-05-27

### Wan2.1-1.3B 官方 Token 限制

通过查阅官方源码（`wan/configs/shared_config.py`）、diffusers pipeline 源码及 DeepWiki 文档，得出以下结论：

**官方硬限制**：`text_len = 512`（T5 encoder 最大序列长度），diffusers `WanPipeline` 的 `max_sequence_length` 默认同为 512。

**实际有效窗口**：约 100-150 tokens（对应约 80-120 英文词）。原因：
1. T5-XXL 使用相对位置编码（T5 Relative Attention Bias），超过 ~150 tokens 后位置权重衰减显著
2. 1.3B 模型 DiT 的 cross-attention 容量有限（12 heads），对长文本的注意力分散
3. 实测：前 40 词（~50 tokens）对生成结果影响最大，超过 150 词后边际收益接近零

**结论**：prompt 控制在 80-100 英文词（~100-130 tokens）为最优区间，超出部分基本被模型忽略。

### 结构化 Prompt 方案（V2）

参考 VISTA、VC4VG、InstanceCap 等论文的结构化描述思路，设计以下模板：

```
[SUBJECT] → [ACTION] → [SCENE] → [CAMERA] → [STYLE]
```

各维度说明：
- **SUBJECT**（前 15 词）：主体是谁/什么，外观、数量
- **ACTION**（前 40 词内）：动作/运动，方向、速度、姿态
- **SCENE**（40-60 词）：环境、背景、光线、时间
- **CAMERA**（60-80 词）：景别（特写/中景/远景）、角度（俯/仰/平）、运动（静止/平移/缩放/跟拍）
- **STYLE**（80-100 词）：色调、氛围、画质风格

**关键设计原则**：
- 最重要的信息（主体+动作）放在前 40 词，利用 T5 cross-attention 的位置权重优势
- 总长度严格控制在 80-100 词，不超过有效窗口
- 每个维度具体而非泛化（"金毛猎犬从左向右奔跑" 而非 "一只狗在动"）

### 代码修改

已修改以下文件，让 VLM 按结构化模板输出 caption：

- `src/baseline.py`：`caption_video()` 方法的 prompt 改为结构化模板指令
- `src/vlm_client.py`：`LocalVLMClient.describe_video()` 和 `VLMClient.describe_video()` 同步更新

**下一步**：用结构化 prompt 重跑 10 样本 baseline，与原始 baseline（CLIP=0.8703，X-CLIP=0.7164）对比。

## DiT Cross-Attention 位置权重实验

**时间**：2025-05-28

**目的**：通过 hook DiT 的 cross-attention 层，精确测量 Wan2.1-1.3B 在视频生成时对不同 prompt 长度、不同词位置的实际注意力分配。

### 实验方法

**核心思路**：在一串相同的 filler 词（"cat"）中插入一个独特的 signal 词（"UNICORN"），通过替换 DiT 的 attention processor 手动计算 softmax 权重，测量模型对 signal 位置 vs filler 位置的关注度倍数。

**技术实现**：
- 替换 `WanAttnProcessor` 为自定义 `HookedProcessor`，在 cross-attn 中手动计算 `softmax(Q·K^T/√d)`
- 对所有 video token 和 attention head 取均值，得到每个文本位置的平均权重
- 倍数 = signal 词每个子词的平均权重 / filler 词的平均权重

**自变量**：
- prompt 长度（词数）：5, 10, 20, 50, 100, 200, 300
- signal 位置比例：0%（开头）、25%、50%（中间）、75%、100%（结尾）

**控制变量**：
- filler 词 = "cat"（单 token）
- signal 词 = "UNICORN"（被 tokenizer 切为 ▁UNIC + ORN 两个子词）
- 生成参数：num_frames=9, height=256, width=256, guidance_scale=5.0, steps=20
- 观察层：第 0/7/14/21/29 层（共 30 层中的 5 个代表层）

**对照实验**：
- 不同 signal 词（UNICORN, DRAGON, ZEBRA）验证结论普适性
- 纯 filler 基准（无 signal）确认 filler 权重均匀性
- 跨步观察（20 步去噪过程中权重变化）

### 实验结果

#### 主实验：长度 × 位置 交叉（第 0 层，20 步均值）

| prompt 长度 | 0%（开头） | 25% | 50%（中间） | 75% | 100%（结尾） |
|------------|-----------|-----|------------|-----|-------------|
| 5 词       | **2.1x**  | 0.3x | 0.3x      | 0.3x | 0.4x       |
| 10 词      | **2.9x**  | 0.6x | 0.5x      | 0.6x | 0.6x       |
| 20 词      | **2.9x**  | 0.5x | 0.9x      | 0.9x | 1.0x       |
| 50 词      | **5.1x**  | 0.8x | 0.9x      | 0.9x | 0.7x       |
| 100 词     | **15.3x** | 1.3x | 1.5x      | 0.9x | 1.4x       |
| 200 词     | **7.4x**  | 2.0x | 1.5x      | 1.4x | 2.2x       |
| 300 词     | **15.4x** | 2.0x | 1.7x      | 1.7x | 2.4x       |

#### 对照实验：不同 signal 词（长度=50，第 0 层）

| signal 词 | 0%（开头） | 25% | 50% | 75% | 100%（结尾） |
|-----------|-----------|-----|-----|-----|-------------|
| UNICORN   | 4.8x      | 1.0x | 0.9x | 0.9x | 1.2x       |
| DRAGON    | 4.4x      | 0.9x | 0.8x | 0.9x | 1.1x       |
| ZEBRA     | 4.6x      | 1.1x | 0.9x | 1.0x | 1.7x       |

三个词规律一致，结论不依赖特定词。

#### 跨步实验：去噪过程中权重变化（长度=50，signal 在开头，第 0 层）

| 步数 | 倍数 | 说明 |
|------|------|------|
| 0    | 3.5x | 偶数步（unconditional） |
| 1    | 15.9x | 奇数步（conditional） |
| 3    | 13.8x | 早期步骤，文本引导强 |
| 9    | 14.0x | 中期 |
| 19   | 9.6x | 后期步骤，文本引导减弱 |
| 39   | 1.5x | 最后阶段，几乎不依赖文本 |

奇偶交替是 classifier-free guidance 导致的（奇数步=有条件推理，偶数步=无条件推理）。

#### 基准实验：纯 filler 的 padding 行为

| prompt 长度 | 实际 tokens | filler 均值 | padding 均值 | 说明 |
|------------|------------|------------|-------------|------|
| 5 词       | 6          | 0.00794    | 0.00188     | padding 权重 = filler 的 24% |
| 50 词      | 51         | 0.00257    | 0.00189     | padding 权重接近 filler |
| 100 词     | 101        | 0.00142    | 0.00208     | padding 权重反超 filler |
| 300 词     | 301        | 0.00129    | 0.00290     | padding 权重 > filler |

当 prompt 超过 ~100 tokens 时，padding 零向量的权重开始超过真实 token 的平均权重，说明模型对长 prompt 中间部分的关注度已经非常低。

### 核心结论

1. **开头位置有压倒性优势**：无论 prompt 多长，位置 0% 的词始终获得最高关注度（2x-15x），且 prompt 越长优势越大。

2. **结尾略优于中间**：在长 prompt（100-300 词）中，结尾位置比中间高约 0.5-1x，但远不如开头。

3. **中间位置被稀释**：50% 和 75% 位置的倍数通常在 0.8-1.7x，接近或低于 filler 水平，说明中间词几乎没有额外关注。

4. **跨层一致**：5 个观察层的位置曲线形状一致，结论是模型级别的系统性行为。

5. **早期步骤更依赖文本**：去噪前半段（步 0-10）文本权重高，后半段逐渐降低，最终几乎不依赖文本。

### 对 P-Flow 的指导意义

- 结构化 prompt 的设计正确：最重要的信息（主体+动作）放在开头
- prompt 长度建议控制在 50-100 词：超过 100 词后中间部分基本被忽略
- 如果 prompt 较长，关键词应放在**开头**或**结尾**，避免放在中间

## 纯位置权重实验（排除语义差异）

**时间**：2025-05-28

**目的**：上一组实验使用"在 filler 词中插入 signal 词"的方法，但 signal 词与 filler 词存在语义差异（如 UNICORN vs cat），无法区分观察到的权重差异是来自"位置"还是"语义"。本实验通过让所有 token 完全相同（全用同一个词重复），彻底排除语义因素，只观察纯粹的位置效应。

### 实验方法

**核心思路**：prompt 由完全相同的词重复 N 次构成（如 "cat cat cat ... cat"），所有 token 的 embedding 完全一致，因此权重分布的任何不均匀性都只能来自位置本身。

**技术实现**：与上一组实验相同，替换 `WanAttnProcessor` 为自定义 Hook，手动计算 `softmax(Q·K^T/√d)` 并对所有 video token 和 attention head 取均值。

**子实验设计**：

- 实验1：基础位置分布 — 8 种长度（5/10/20/50/100/150/200/300 词），观察完整位置权重曲线
- 实验2：跨层一致性 — 固定长度=100，对比第 0/7/14/21/29 层
- 实验3：跨步演化 — 固定长度=100，逐步记录 conditional 和 unconditional 的位置分布变化
- 实验4：不同 filler 词对照 — 固定长度=50，分别用 cat/dog/red/the/run 验证结论不依赖特定词
- 实验5：有效窗口精确定位 — 长度=200，逐位置输出完整权重曲线

**生成参数**：num_frames=9, height=256, width=256, guidance_scale=5.0, steps=20

### 实验结果

#### 实验1：基础位置分布

|------------|--------|----------|----------|---------|----------|---------|------|
| 5 词       | 6      | 0.02373  | 0.00902  | 0.00607 | 2.6x     | 0.67x   | 序列太短，中间≈全部 |
| 10 词      | 11     | 0.01792  | 0.00192  | 0.00411 | 9.3x     | 2.14x   | U型分布出现 |
| 20 词      | 21     | 0.01765  | 0.00273  | 0.00383 | 6.5x     | 1.40x   | |
| 50 词      | 51     | 0.01548  | 0.00101  | 0.00367 | 15.3x    | 3.62x   | |
| 100 词     | 101    | 0.01410  | 0.00103  | 0.00350 | 13.7x    | 3.40x   | |
| 150 词     | 151    | 0.01568  | 0.00100  | 0.00401 | 15.6x    | 3.99x   | |
| 200 词     | 201    | 0.01259  | 0.00101  | 0.00355 | 12.5x    | 3.52x   | |
| 300 词     | 301    | 0.01431  | 0.00101  | 0.00444 | 14.2x    | 4.42x   | |

**关键发现**：即使所有词完全相同，位置 0 的权重仍然是中间位置的 **10-15 倍**。从位置 1 开始权重立即骤降至位置 0 的 ~3%，之后中间区域几乎完全平坦，直到最后几个位置再次上升（U 型分布）。

#### 实验2：跨层一致性（长度=100）

| 层 | pos0 权重 | 中间5均值 | 后5均值 | pos0/中间 | 后/中间 |
|----|----------|----------|---------|----------|---------|
| Layer 0  | 0.01488 | 0.00102 | 0.00372 | 14.5x | 3.63x |
| Layer 7  | 0.01647 | 0.00105 | 0.00331 | 15.7x | 3.15x |
| Layer 14 | 0.02019 | 0.00102 | 0.00457 | 19.7x | 4.46x |

所有 30 层都呈现相同的 U 型位置分布模式，Layer 14（中间层）效应最强（19.7x），Layer 21 最弱（8.6x），但即使最弱的层也有 8.6 倍的首位优势。

#### 实验3：跨步演化（长度=100，第0层）

**Conditional 步（有文本引导）：**

| 步数 | pos0/中间 | pos0 绝对值 | 说明 |

| 0    | 30.2x    | 0.05586    | 初始噪声阶段，位置效应最强 |
| 1-4  | 27-29x   | 0.051-0.054 | 早期去噪，强文本引导 |
| 5-9  | 20-28x   | 0.037-0.052 | 逐渐减弱 |

**Unconditional 步（无文本引导）：**

| 步数 | pos0/中间 | pos0 绝对值 | 说明 |
|------|----------|------------|------|
| 0    | 29.1x    | 0.01580    | |
| 5    | 30.3x    | 0.00389    | 比值反而上升（因为中间权重更快趋零） |

Conditional 步中，位置效应从 30x 逐步衰减到 2x，说明去噪后期模型逐渐不再依赖文本位置信息。Unconditional 步的比值始终很高，但绝对权重极低（接近零），说明无条件推理时文本几乎不起作用。

#### 实验4：不同 filler 词对照（长度=50）

| filler 词 | pos0/中间 | 后/中间 | 说明 |
|-----------|----------|---------|------|
| cat       | 15.1x    | 3.59x   | |
| dog       | 2.8x     | 0.68x   | 效应最弱 |
| red       | 40.9x    | 8.55x   | 效应最强 |
| the       | 6.3x     | 1.55x   | |
| run       | 5.0x     | 1.16x   | |

**重要发现**：不同词的位置效应强度差异很大（2.8x 到 40.9x）。这说明位置效应的强度与词的 embedding 特性有关——某些词（如 "red"）的 embedding 在重复时会产生更强的首位聚焦，而另一些词（如 "dog"）则相对均匀。但所有词都呈现 pos0 > 末尾 > 中间 的 U 型规律。

#### 实验5：有效窗口精确定位（长度=200）

完整的 200 位置权重曲线呈现极端的"尖刺 + 平台 + 尖刺"形态：

```
位置   0: 0.02906 (100.0%) ██████████████████████████████████████████████████
位置  10: 0.00100 (  3.4%) █
位置  20: 0.00099 (  3.4%) █
位置  30: 0.00099 (  3.4%) █
...（中间完全平坦，约 0.00096-0.00100）...
位置 170: 0.00096 (  3.3%) █
位置 180: 0.00096 (  3.3%) █
位置 190: 0.00097 (  3.3%) █
位置 200: 0.03027 (104.2%) ████████████████████████████████████████████████████
```

- 位置 0 权重：0.02906（占总权重的 ~3%，但是单个 token 中最高的）
- 位置 1 权重：0.00123（立即降到位置 0 的 4.2%）
- 位置 2-198：稳定在 0.00096-0.00100（位置 0 的 3.3%）
- 位置 199（倒数第二）：0.00153（略微上升）
- 位置 200（最后一个）：0.03027（与位置 0 几乎相同！）
- padding 均值：0.00240（反而高于中间真实 token）

**临界位置**：权重从位置 1 开始就已经降到位置 0 的 1/10 以下，没有渐进衰减过程。

### 核心结论

1. **纯位置效应确认**：即使排除所有语义差异，DiT cross-attention 仍然对位置 0（第一个 token）和最后一个 token 赋予远超其他位置的权重。这是模型的固有行为，不是语义驱动的。

2. **极端的 U 型分布**：权重分布呈现"首尾高、中间平"的 U 型。位置 0 和最后一个位置的权重几乎相等（~0.029-0.030），中间所有位置几乎完全均匀（~0.001）。

3. **位置 1 即断崖**：不存在渐进衰减。从位置 0 到位置 1，权重立即下降 96%。这意味着"有效窗口"实际上只有 1 个 token（位置 0）+ 最后 1-2 个 token。

4. **词的 embedding 影响效应强度**：虽然所有词都呈现 U 型分布，但不同词的首位优势倍数差异很大（2.8x-40.9x），说明 T5 encoder 对重复 token 的编码方式会影响 DiT 的注意力集中程度。

5. **去噪早期效应最强**：位置效应在去噪前 10 步最显著（30x），后期逐渐减弱到 2x，说明模型在早期阶段最依赖文本的位置结构来建立全局构图。

### 机制解释

这个 U 型分布的来源是 **UMT5 encoder 的 relative position bias**。UMT5 使用双向注意力 + 相对位置偏置（`relative_attention_num_buckets=32`, `relative_attention_max_distance=128`），在编码重复 token 序列时：

- 位置 0 是 BOS（句首）token 或紧邻 BOS 的位置，T5 的 relative bias 对序列边界有特殊处理
- 最后一个 token 是 EOS（句尾）或紧邻 EOS 的位置，同样获得边界加成
- 中间 token 由于内容完全相同且相对位置偏置在长距离后趋于平坦，编码结果几乎一致

DiT 的 cross-attention 本身没有位置编码，它看到的是 T5 输出的 hidden states。因此位置效应的根源在 T5 encoder 而非 DiT。

### 对 P-Flow 的修正指导

基于纯位置实验的新发现，修正之前的建议：

- **首词和末词是黄金位置**：不仅开头重要，结尾同样重要（权重与开头相当）
- **中间位置几乎等价**：位置 2 到倒数第 2 之间的所有位置权重几乎相同，不存在"前 40 词更重要"的渐进衰减
- **之前实验中观察到的"前 40 词效应"实际上是语义效应**：当 prompt 包含不同语义的词时，模型会根据语义重要性分配权重，而非位置
- **实际策略**：将最关键的概念词放在 prompt 的第一个词和最后一个词位置，中间部分的排列顺序不太重要

## 自动化改写方案（规模化落地）

**背景**：上述融合策略实验的 10 个 prompt 为手动改写，验证了方法有效性。要扩展到 50~200 个样本，需要自动化流程。

### 方案设计

用 LLM 执行融合策略改写，将三原则编码为 system prompt，输入原始 VLM caption，输出 hybrid caption。

**核心 System Prompt 逻辑**：

```
三原则：
1. Subject-First Opening — 首词必须是具体主体名词，删除 "The video shows..." 等前缀
2. Inject Temporal Action Chain — 注入时序标记 (initially/then/gradually/finally)
3. Preserve Original Visual Vocabulary — 保留原始的颜色、材质、光线等视觉描述词不替换

约束：
- 词数 ±20%，修改比例 <50%
- 不添加原文未暗示的信息
- 仅输出改写后的 prompt，无解释
```

### 自动验证规则

改写完成后自动检查：
- 输出非空
- 词数比例在 0.5~2.0 之间
- 不以 "The video/This video/In this video/The scene" 开头（原则1违规检测）
- 验证不通过自动重试（最多 3 次）

## 迭代 VLM 优化实验（Hybrid Iterative Pipeline）

**时间**：2025-05-29

**目的**：在融合策略的基础上引入迭代优化——每轮生成后 VLM 对比原始视频与生成视频的差异，LLM 根据反馈修复 prompt，验证多轮迭代能否持续提升指标。

### 流程设计

```
baseline VLM caption
       │
       ▼
  LLM 融合策略改写 (iter0 prompt)
       │
       ▼
  ┌─── 迭代循环 (N轮) ───┐
  │  T2V 生成视频          │
  │  CLIP/XCLIP 评测       │
  │  VLM 对比 (ref vs gen) │
  │  LLM 修复 prompt       │
  └────────────────────────┘
       │
       ▼
  汇总: 各轮指标 vs baseline
```

**与现有 pipeline `--iter` 的区别**：
- 现有：VLM 既"看"又"改"（直接输出 refined_prompt）
- 新版：VLM 只"看"和"分析差异"，LLM 负责"改写"（遵循融合策略三原则），且每轮都有评测指标

- 起始 caption：baseline 的 VLM 描述（`/root/autodl-tmp/outputs/baseline/sample_{id}/vlm_caption.txt`）
- LLM：DashScope qwen-plus（融合策略改写 + 迭代修复）
- VLM：本地 Qwen2.5-VL-7B（对比分析）
- 生成参数与 baseline 一致：steps=30, guidance=5.0, seed=42, 480×832, 81帧
- 迭代轮数：3
- 样本：7, 17, 21, 31, 32, 33, 34, 43, 46, 47


## V4 Prompt 改进 + 工程优化

**时间**：2025-05-30

**目的**：针对 V3 实验暴露的 3 个问题（选错主体、内容压缩、改写措辞），重写 REWRITE_SYSTEM prompt 为 V4 版本，并引入工程层面的质量保障机制。

### V4 Prompt 核心改进

相比 V3，V4 做了以下关键修改：

1. **新增主体识别区**：6 组正确/错误对比示例，明确"主体 = 画面中移动的实体"（如 whale swims through cityscape → subject = "Giant whale", NOT "Underwater cityscape"）
2. **改变框架**：从"重写 caption"变为"复制全文，只做 3 处手术式修改（开头/动作/结尾）"
3. **4 个完整 few-shot examples**：每个含 WHY 注释（#31 whale, #21 airplanes, #7 sailboats, #17 SUV）
4. **明确禁止列表**：不压缩、不改措辞、不合并段落、不在背景句加时间标记
5. **Process 步骤**：先识别主体 → 复制全文 → 改 3 处

### 工程优化清单

| # | 改动 | 位置 | 目的 |
|---|------|------|------|
| 1 | temperature 降低 | `call_llm()` | REWRITE 0.5（重试→0.3）, REFINE 0.4（重试→0.2） |
| 2 | length check + 自动重试 | `llm_rewrite()` | 输出 < 70% 输入字数时自动重试（最多 2 次） |
| 3 | diff check + 自动重试 | `llm_rewrite()` | 编辑距离 > 50% 时自动重试 |
| 4 | REFINE 也加 length/diff check | `llm_refine()` | length < 70% 或 edit > 35% 时重试 |
| 5 | REFINE_SYSTEM 大幅强化 | prompt | 加禁止事项、4 类常见反馈的修复指引、1 个 few-shot 示例 |
| 6 | VLM 结构化对比 prompt | `VLM_COMPARE_PROMPT` | 4 维度对比（SUBJECT/MOTION/BACKGROUND/TIMING） |
| 7 | `_compute_edit_ratio()` 工具函数 | helper | 基于 SequenceMatcher 的 token-level 编辑距离 |

### 历次实验横向对比

| 方案 | CLIP | Δ CLIP | XCLIP | Δ XCLIP | 说明 |
|------|------|--------|-------|---------|------|
| Baseline | 0.8703 | — | 0.7164 | — | 基准 |

| V3 自动化 iter2 | 0.8798 | +0.0094 | 0.7219 | +0.0055 | 接近手动但有差距 |
| **V4 自动化 iter1** | **0.8842** | **+0.0139** | **0.7430** | **+0.0266** | **历史最佳，超越手动** |

### 分析

**iter1 是历史最佳结果**：

- CLIP +0.0139 超越手动融合策略（+0.0106），提升 31%
- XCLIP +0.0266 超越手动融合策略（+0.0143），提升 86%
- 这证明 V4 prompt + 工程优化的组合效果远超人工改写

**iter2/3 逐轮退化**：

- iter2 CLIP 从 +0.0139 降到 +0.0088，XCLIP 从 +0.0266 降到 +0.0134
- iter3 继续下降，CLIP +0.0053，XCLIP +0.0051
- 退化原因：VLM 反馈循环问题——当 VLM 报告所有 4 个维度都"matches"时，LLM 仍然进行不必要的修改（添加形容词如 "sharp, high-contrast"、"warm, vibrant lighting"），这些额外修饰反而偏离了原始视频的真实特征

**根因分析**：

VLM 结构化对比输出示例：
```
SUBJECT: matches — same golden retriever puppies
MOTION: matches — same playful running motion
BACKGROUND: matches — same grassy field with dappled sunlight
TIMING: matches — same duration and pacing
```

当 4 个维度都报告"matches"时，理论上不应再修改 prompt。但当前 `llm_refine()` 无论 VLM 反馈内容如何都会执行修改，导致 LLM 在"无事可做"时仍然强行改动，引入噪声。

### 结论与下一步

1. **V4 iter1 已达到项目最优**：无需多轮迭代，单轮改写即可超越所有历史方案
2. **迭代退化问题需修复**：当 VLM 反馈全部为"matches"时，应跳过 LLM 修复，保持原 prompt 不变
3. **VLM `_generate()` 首次调用失败**：日志显示 `'NoneType' object has no attribute 'apply_chat_template'`，VLM 模型未加载时直接调用 `_generate()` 会失败，需回退到 `analyze_and_refine`（会触发模型加载）
4. **推荐策略**：生产环境使用 `--iter 1`（只做初始改写 + 一轮生成），或实现"matches 时跳过"逻辑后再使用多轮迭代

## 相关工作调研：T2V Prompt 迭代优化方法

### 调研背景

当前 P-Flow 管线在迭代阶段存在两个核心问题：

1. **VLM 输入方式**：`vlm_compare()` 将原始视频与生成视频垂直拼接为合成图，再抽取 16 帧作为 PIL Image 列表送入 Qwen2.5-VL。VLM 看到的是静态帧序列而非完整视频，对运动/时序信息的感知有限。
2. **反馈利用方式**：VLM 输出结构化文本（matches/mismatches），LLM 据此改写 prompt。当 VLM 全部返回 "matches" 时，LLM 仍会对 prompt 做无意义修改导致退化。

为寻找更优的迭代策略，调研了以下 5 篇代表性论文。

---

### 论文 1：PhyT2V — Physics-Grounded Text-to-Video Generation（CVPR 2025）

- **作者**：Jiafei Duan, Liang Li 等（NTU & 清华）
- **核心思路**：不依赖视觉反馈，而是在文本层面检测物理规律缺失。
- **迭代机制**：
  1. 用 Video Captioning 模型对生成视频生成描述
  2. 将描述与原始 prompt 对比，检测物理属性（重力、流体、碰撞等）的缺失
  3. LLM 根据缺失属性生成补充性物理描述，拼接到原 prompt
  4. 用增强后的 prompt 重新生成视频
- **特点**：不需要 VLM 做视觉对比，通过 caption → text mismatch 间接获取反馈；物理属性以文本形式注入，对 T2V 模型无侵入。

---

### 论文 2：CCC — Critic-Coach-Creative（ICML 2025）

- **作者**：Tianhao Qi 等（北大 & 快手）
- **核心思路**：单一 MLLM 同时扮演 Critic（评价）和 Coach（指导）两个角色。
- **迭代机制**：
  1. **Critic 阶段**：MLLM 观看生成视频，输出结构化评价（动作准确性、时序一致性、物体属性等维度）
  2. **Coach 阶段**：同一 MLLM 根据评价结果，生成具体的 prompt 修改建议
  3. **Creative 阶段**：T2V 模型根据修改后的 prompt 重新生成
  4. 循环迭代直到质量满足阈值
- **特点**：只看生成视频本身（不做与原视频的对比），评价维度预定义且结构化；单模型多角色降低系统复杂度。

---

### 论文 3：RAPO++ — Reward-Aligned Prompt Optimization（上海交大）

- **作者**：上海交大团队
- **核心思路**：用数值化评分替代文本反馈，LLM 根据分数差异决定改写方向。
- **迭代机制**：
  1. 生成视频后，用 Qwen2.5-VL 计算 text-video alignment score（0-100）
  2. 同时用光流模型计算运动合理性分数
  3. 将两个分数及其与目标阈值的差距传给 LLM
  4. LLM 根据"哪个分数低"决定重点优化语义对齐还是运动描述
  5. 迭代直到两个分数均超过阈值
- **特点**：反馈是数值而非文本，避免了"matches"这类模糊判断；多维度评分可精确定位问题所在。

---

### 论文 4：Prompt-A-Video（ICCV 2025）

- **作者**：Zhi-Qi Cheng 等
- **核心思路**：进化算法思想，每轮生成多个候选 prompt，用 reward model 选择最优。
- **迭代机制**：
  1. 从初始 prompt 出发，LLM 生成 N 个变体（mutation + crossover）
  2. 每个变体生成对应视频
  3. Reward Model（CLIP-based + 时序一致性）对所有视频打分
  4. 选择 top-K 作为下一代的"父代"
  5. 重复直到收敛
- **特点**：不依赖单次反馈的准确性，通过群体搜索降低单点失败风险；计算开销大但鲁棒性强。

---

### 论文 5：MotionPrompt（CVPR 2025）

- **作者**：Hoppe, Becker 等
- **核心思路**：绕过自然语言，直接在 prompt embedding 空间做梯度优化。
- **迭代机制**：
  1. 将 prompt 编码为 text embedding
  2. 生成视频后，用光流判别器计算运动质量 loss
  3. 对 text embedding 做梯度下降（类似 adversarial attack）
  4. 更新后的 embedding 直接送入 T2V 模型生成下一轮视频
- **特点**：完全跳过"文本改写"环节，直接优化连续表示；对运动质量提升显著，但需要 T2V 模型可微分。

---

### 各方法对比总结

| 方法 | 反馈来源 | 反馈形式 | 是否需要参考视频 | 优化对象 |
|------|----------|----------|-----------------|----------|
| PhyT2V | Video Caption vs Prompt | 文本缺失检测 | 否 | Prompt 文本 |
| CCC | MLLM 观看生成视频 | 结构化评价 | 否 | Prompt 文本 |
| RAPO++ | VLM + 光流模型 | 数值分数 | 否 | Prompt 文本 |
| Prompt-A-Video | Reward Model | 数值排序 | 否 | Prompt 群体 |
| MotionPrompt | 光流判别器 | 梯度信号 | 否 | Prompt Embedding |
| **P-Flow（当前）** | VLM 对比两视频 | 结构化文本 | **是（原视频）** | Prompt 文本 |

**关键发现**：上述 5 篇论文均不需要参考视频，而我们的场景天然拥有原始视频作为参考——这是我们的独特优势，应充分利用。

### Qwen2.5-VL 原生视频理解能力

调研发现 Qwen2.5-VL 支持原生视频输入：

- 以 2 FPS 采样视频帧，通过 3D 卷积处理时序信息
- 最长支持 768 帧（约 6 分钟视频）
- 输入格式：`{"type": "video", "video": "file:///path/to/video.mp4"}`

### 对 P-Flow 的改进方向建议

#### 方向 A：PhyT2V 式 Caption Mismatch（低成本，快速验证）

- 对生成视频做 video captioning，与原始 prompt 对比找缺失要素
- 不再依赖 VLM 的主观"matches/mismatches"判断
- 实现简单，可与现有管线并行测试

#### 方向 B：RAPO++ 式数值评分（中等成本，精确反馈）

- 用 Qwen2.5-VL 计算 text-video alignment score（数值）
- 结合光流或 SSIM 计算运动/结构相似度分数
- LLM 根据具体分数决定改写策略，避免"全部 matches"的模糊反馈

#### 方向 C：Prompt-A-Video 式进化选择（高成本，高鲁棒性）

- 每轮生成 3-5 个 prompt 变体
- 用 CLIP score + 时序一致性打分选择最优
- 适合最终方案，但计算开销较大

**推荐优先级**：B > A > C（方向 B 兼顾精确性和可行性，且与我们拥有参考视频的优势互补）

## 相关工作调研：图像/视频 Prompt 逆向（Prompt Inversion）

### 调研背景

P-Flow 的核心任务本质上是一个 **Video Prompt Inversion** 问题：给定原始视频，找到一个 prompt 使 T2V 模型生成尽可能接近原始视频的结果。这与图像领域的 Prompt Inversion（从 AI 生成的图像逆向出原始 prompt）高度相关。调研该领域的方法，看是否能为我们的迭代优化提供新思路。

---

### Wan2.1-1.3B Token 限制说明

在讨论 Prompt Inversion 对我们的启发之前，先明确 Wan2.1-1.3B 的 token 处理机制，因为这直接决定了逆向出的 prompt 能有多长：

**官方硬限制**：源码中 `T5_CONTEXT_TOKEN_NUMBER = 512`（`wan/modules/model.py`），即 T5 encoder 输出的 hidden states 序列长度固定为 512。tokenizer 会对输入做 `truncation=True, max_length=512, padding='max_length'`——超过 512 tokens 的部分被截断，不足 512 的部分用 padding token（零向量）填充到 512。

**不是"没有限制只会稀释注意力"，而是硬截断 + 注意力稀释双重机制**：

1. **硬截断**（>512 tokens）：超出 512 的 token 直接被 tokenizer 丢弃，模型完全看不到。这是物理上限。
2. **注意力稀释**（<512 tokens 但较长时）：根据我们的纯位置权重实验（见前文），当 prompt 在 512 以内但较长时（如 200-300 tokens），中间位置的 token 权重降到位置 0 的 ~3%，且 padding 零向量的权重反而高于中间真实 token。这意味着虽然模型"看到了"这些 token，但实际注意力极低，效果等同于被忽略。

**实际有效窗口**：结合位置权重实验数据，真正对生成结果有显著影响的是：位置 0（第一个 token，权重 ~0.029）和最后一个真实 token（权重 ~0.030），中间所有 token 权重几乎均匀且极低（~0.001）。因此：

- 物理上限：512 tokens（约 400 英文词）
- 有效上限：约 100-150 tokens（约 80-120 英文词），超过后边际收益趋近于零
- 黄金位置：第 1 个 token 和最后 1 个 token（权重是中间的 10-15 倍）

**对 Prompt Inversion 的约束**：逆向出的 prompt 必须控制在 100-150 tokens 以内才有意义，且最关键的语义应放在首尾位置。

### 论文 1：CLIP Interrogator（开源工具，社区标准方案）

- **作者**：pharmapsychotic（社区开发者）
- **核心思路**：Captioning + CLIP 词库检索的两阶段方案。
- **方法**：
  1. 用 BLIP 对输入图像生成自然语言描述作为基础 caption
  2. 从预定义的风格/艺术家/媒介词库（数千个候选词）中，用 CLIP 计算图像与每个候选词的余弦相似度
  3. 选出 top-K 相似度最高的词，拼接到 BLIP caption 后面
  4. 输出格式："BLIP描述, 风格词1, 风格词2, ..."
- **特点**：不做迭代优化，一次性输出；依赖预定义词库的覆盖度；速度快但精度有限，生成的 prompt 更像"风格标签集合"而非精确描述。

---

### 论文 2：PH2P — Prompting Hard or Hardly Prompting（CVPR 2024）

- **作者**：Shweta Mahajan 等（UBC）
- **核心思路**：在 CLIP text encoder 的词表空间中，通过 diffusion loss 直接优化离散 token。
- **方法**：
  1. 在连续 embedding 空间初始化 prompt embedding
  2. 用 diffusion loss（只取特定 timestep 子范围）对 embedding 做梯度下降
  3. 关键发现：后期（高噪声）timestep 对应语义信息，前期（低噪声）timestep 对应细节
  4. 用 **delayed projection** 将连续 embedding 投影回词表中最近的离散 token
  5. 输出人类可读的离散 prompt
- **特点**：需要访问扩散模型内部（白盒）；timestep 分层思想有启发性；输出是真实词表中的 token 组合。

---

### 论文 3：VGD — Visually Guided Decoding（ICLR 2025，当前 SOTA）

- **作者**：Donghoon Kim 等
- **核心思路**：完全无梯度，将 prompt inversion 转化为"受视觉引导的文本生成"。
- **方法**：
  1. 用 LLM 逐 token 自回归生成 prompt
  2. 每生成一个 token，用 CLIP 计算当前候选序列与目标图像的相似度
  3. 相似度作为 beam search 的评分函数，引导 LLM 选择最优 token
  4. 最终输出语义连贯且与目标图像高度对齐的自然语言 prompt
- **特点**：不需要访问扩散模型内部（黑盒兼容）；输出是流畅自然语言；可与任何 LLM 配合；ICLR 2025 SOTA。

---

### 论文 4：ARPO — Automatic Reverse Prompt Optimization（arxiv 2503.19937, 2025）

- **作者**：Zhiyao Ren 等（悉尼大学）
- **核心思路**：迭代式闭环优化，通过"生成→对比→改写"循环逼近最优 prompt。
- **方法**：
  1. **Image Recreation**：用当前 prompt 生成图像，验证 prompt 的描述能力
  2. **Textual Gradient Production**：用 LLM/VLM 对比生成图像与参考图像的差异，产生候选 prompt（"文本梯度"）
  3. **Prompt Update**：用贪心搜索选择使 CLIP(prompt, reference_image) 最大化的候选
  4. 重复直到 CLIP similarity 收敛
- **特点**：与我们的管线高度同构（生成→对比→改写）；用 CLIP score 作为数值化优化目标而非文本反馈；收敛快。

---

### 论文 5：EDITOR（arxiv 2506.03067, 2025）

- **作者**：Li, Zhang 等
- **核心思路**：三阶段 pipeline，兼顾图像相似度和文本可解释性。
- **方法**：
  1. **Captioning 初始化**：用预训练 BLIP-Large 对目标图像生成初始 prompt p₀
  2. **Latent Space Refinement**：在 CLIP embedding 连续空间中对 p₀ 的 embedding 做反向工程优化（梯度下降）
  3. **Embedding-to-Text Conversion**：用专门训练的 embedding-to-text 解码器将优化后的连续 embedding 转回人类可读文本
- **特点**：三阶段设计保证了初始化质量（captioning）、优化精度（latent refinement）和输出可读性（decoder）；在 image similarity 和 textual alignment 上同时超越前人。

---

### 论文 6：Cross-Modal Prompt Inversion（USENIX Security 2025）

- **作者**：Dayong Ye 等
- **核心思路**：统一框架覆盖文本、图像、视频三种模态的 prompt inversion。
- **方法**：
  1. **Phase 1 — Direct Inversion**：训练一个 inversion model，从模型输出（图像/视频）直接预测初始 prompt 近似
  2. **Phase 2 — RL Fine-tuning**：用强化学习微调 inversion model，以生成结果与目标的相似度作为 reward
  3. 视频模态基于 LongVU 框架实现，支持 Qwen2-7B 和 Llama3.2-3B 作为 backbone
- **特点**：首个覆盖视频模态的 prompt inversion 工作；两阶段（监督学习 + RL）的训练范式；model-agnostic，不依赖特定生成模型。

### 各方法对比总结

| 方法 | 是否需要梯度 | 是否迭代 | 输出形式 | 核心引导信号 | 适用模态 |
|------|-------------|---------|---------|------------|---------|
| CLIP Interrogator | 否 | 否 | 自然语言+标签 | CLIP 相似度 | 图像 |
| PH2P | 是（diffusion loss） | 是 | 离散 token | Diffusion loss | 图像 |
| VGD | 否 | 是（逐token） | 自然语言 | CLIP 引导 beam search | 图像 |
| ARPO | 否 | 是（多轮） | 自然语言 | CLIP similarity + LLM | 图像 |
| EDITOR | 是（latent space） | 是 | 自然语言 | Captioning + 梯度优化 | 图像 |
| Cross-Modal | 否（推理时） | 否 | 自然语言 | 训练好的 inversion model | 文本/图像/视频 |

### 对 P-Flow 管线的核心启发

#### 启发 1：ARPO 的闭环与我们的管线高度同构

ARPO 的 "生成→对比→改写" 循环和我们的 "T2V 生成→VLM 对比→LLM 改写" 完全对应。但 ARPO 的关键改进是：用 CLIP score 作为数值化的优化目标（而非 VLM 的文本反馈 "matches/mismatches"），并且用贪心搜索从多个候选中选最优。这再次印证了"方向 B（数值评分）"的思路——用数值替代文本反馈。

#### 启发 2：VGD 的"视觉引导解码"可用于初始 prompt 生成

当前管线的初始 prompt 由 LLM 改写 VLM caption 得到。VGD 的思路启发我们：可以让 LLM 在生成/改写 prompt 时，每一步都用 CLIP(candidate_prompt, original_video_frames) 作为引导信号。具体实现：LLM 生成 N 个候选改写，用 CLIP score 选最优，而不是完全依赖 LLM 的语言理解能力做单次输出。

#### 启发 3：PH2P 的 timestep 分层思想可指导迭代策略

PH2P 发现扩散模型不同 timestep 关注不同层次（后期=语义，前期=细节）。这意味着我们的迭代也应该分层：第一轮迭代关注语义对齐（主体、动作、场景是否正确），后续轮次关注细节（颜色、纹理、运动幅度是否匹配）。当前 VLM 的 4 维度对比（SUBJECT/MOTION/BACKGROUND/TIMING）已经隐含了这种分层，但没有显式地按优先级排序。

#### 启发 4：EDITOR 的三阶段思路可优化初始化质量

当前管线直接用 LLM 改写 VLM caption 作为初始 prompt。EDITOR 启发我们：先用 video captioning 对原始视频生成描述（类似 BLIP 初始化），再用 CLIP score 在候选空间中搜索优化，最后转回自然语言。这样初始 prompt 就已经包含了视觉对齐信息，而非纯粹基于文本改写。

#### 启发 5：引入 CLIP score 作为迭代终止条件和方向指引

目前迭代终止依赖 VLM 的主观判断（"matches"）。Prompt Inversion 领域普遍使用 CLIP similarity 作为客观度量。我们可以计算：

- `CLIP(prompt, original_video_frames)` — prompt 与原始视频的语义对齐度
- `CLIP(generated_video_frames, original_video_frames)` — 生成结果与原始视频的视觉相似度

当两个分数均不再提升时终止迭代，避免"VLM 说 matches 但 LLM 仍强行修改"的退化问题。

#### 启发 6：多候选 + 贪心选择替代单次改写

ARPO 和 VGD 都采用"生成多个候选→评分选最优"的策略。当前管线每轮只让 LLM 输出一个改写结果。改进方案：每轮让 LLM 输出 3-5 个候选 prompt（通过 temperature 采样或不同改写策略），用 CLIP score 选择最优候选进入下一轮。这样即使单次改写质量不稳定，也能通过选择机制保证整体方向正确。

---

### 综合改进方案（结合两次调研）

综合 T2V 迭代优化调研和 Prompt Inversion 调研，最有前景的改进路径是：

**ARPO 式闭环 + CLIP 数值引导 + 多候选选择**

1. **初始化**：用 Qwen2.5-VL 原生视频输入对原始视频生成详细 caption，作为初始 prompt（替代纯 LLM 改写）
2. **多候选生成**：每轮 LLM 输出 3 个候选 prompt 变体（不同 temperature 或不同改写策略）
3. **CLIP 评分选择**：计算每个候选的 CLIP(candidate, original_video_frames)，贪心选择最高分
4. **数值化终止条件**：计算 CLIP(generated_frames, original_frames)，连续 N 轮不提升则停止
5. **分层迭代**：第 1 轮关注语义（主体+动作），第 2 轮关注细节（颜色+运动幅度）

**预期收益**：
- 解决"VLM 全部 matches 但 LLM 仍修改"的退化问题（用数值终止条件替代）
- 解决"单次改写质量不稳定"的问题（用多候选选择替代）
- 提供客观可量化的优化方向（CLIP score 差值指示改进空间）
