## 一、项目定位

VMAD（Video Motion Asset Decomposition）= 视频 Prompt Inversion。给定参考视频和冻结 T2V 模型，找到最优输入条件使模型忠实再现参考视频。

**三层架构**：
- Layer 1 (Text)：P-Flow V4 LLM 改写（Subject-First + Temporal Action Chain + Preserve Visual Vocabulary）
- Layer 2 (Δe)：Position-Aware Velocity Field Matching → 在 T5 hidden states 空间优化残差 embedding
- Layer 3 (η_inv)：Flow Matching Inversion + SVD → Noise Prior Blending

**核心模型**：Wan2.1-T2V-1.3B-Diffusers, 30 steps, guidance=5.0, seed=42, 480×832, 81帧

## 三、关键技术发现

### Bug 修复（2025-05-31）
1. `apply_noise_prior` 无条件调用 → 加 `if cfg.use_blend:` 保护
2. `motion_text` 无条件拼接垃圾 token → 加 `and cfg.use_token_decode` 保护
3. 修复后 VMAD 三层全关 = P-Flow（CLIP 差异 < 0.001）

### Layer 3 失败根因
- **不是** generator state offset（FlowMatchEulerDiscreteScheduler 是确定性的，去噪循环不消耗 generator）
- **是** latents 生成路径差异：外部 `torch.randn` vs pipeline 内部 `randn_tensor`，即使同一 generator 状态也产生完全不同的随机数
- 修复方案：让 pipeline 正常 `prepare_latents`，然后外部 blend eta_motion 进去（未实施）

### V4 Caption 与 Δe 互补关系
- V4 caption 饱和 CLIP 维度（帧级外观）
- Δe 主攻 XCLIP 维度（时序运动），在 V4 基础上 +0.0321
- V4 下最优 alpha 左移到 0.005~0.008（因 e₀ 已接近目标，需更小修正）

### cfg.alpha 耦合问题
同一参数控制 L2（effective_strength = strength × alpha）和 L3（alpha_scaled = alpha × strength）。当前 L3 关闭故不影响，后续需解耦。

### Seed 策略差异
P-Flow: `seed = cfg.seed + sample_id`; VMAD: `seed = cfg.seed`（固定 42）
