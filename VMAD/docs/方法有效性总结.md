# VMAD 方法有效性总结

> 截止：2025-06-01 | 验证样本：#7（单样本，待多样本确认）

---

## 有效方法

### 1. V4 Caption（P-Flow Layer 1）

| 项目 | 内容 |
|------|------|
| 原理 | LLM 对 VLM caption 做 3 处手术式修改：主体前置 + 时序动作链 + 运动词收尾 |
| 主攻维度 | CLIP（帧级外观） |
| 增量 | +0.0247 CLIP, +0.0183 XCLIP（vs baseline caption, sample #7） |
| 10 样本验证 | ✅ 通过（0.8703→0.8842 CLIP, 0.7164→0.7430 XCLIP） |
| Caption 来源 | `/root/autodl-tmp/outputs/hybrid_iter_v4/captions_iter0/{id}.txt` |

**关键参数**：无需调参，LLM 改写是离线一次性操作。

---

### 2. Layer 2 — Δe Embedding Hook

| 项目 | 内容 |
|------|------|
| 原理 | Velocity Field Matching 优化残差 Δe，通过 text encoder hook 注入到 T5 输出 |
| 主攻维度 | XCLIP（时序运动），CLIP 在 V4 caption 下几乎饱和 |
| 增量（V4 基础） | α=0.005: +0.0049 CLIP, +0.0281 XCLIP; α=0.008: -0.0002 CLIP, +0.0321 XCLIP |
| 10 样本验证 | ❌ 待做（仅 sample #7） |

**有效参数**：

```yaml
# Extract 阶段参数
num_opt_steps: 200          # VFM 优化步数
T_m: 1.0                    # 时间步上界（全步匹配）
position_aware: true        # Position-Aware 梯度缩放
learning_rate: 0.01         # AdamW lr（代码默认）

# Apply 阶段参数
alpha: 0.005                # ★CLIP 最优（双指标均衡）
alpha: 0.008                # ★XCLIP 最优（时序运动极值）
strength: 1.0               # 固定不动
effective_strength: alpha   # 实际注入 = strength × alpha

# 必须关闭的开关
use_blend: false            # --no-blend（Layer 3 有害）
use_token_decode: false     # --no-token_decode（Layer 1 乱码）
use_svd: false              # --no-svd（未验证独立效果）
use_disentangle: false      # --no-disentangle（未验证独立效果）
```

**推荐运行命令**：
```bash
cd /root/autodl-tmp/videofake/VMAD

# Extract + Apply（需要 V4 caption 重新 extract Δe）
python run_batch_extract.py \
    --video-dir /root/autodl-tmp/data/video-200/water_mark_out \
    --caption-dir /root/autodl-tmp/outputs/hybrid_iter_v4/captions_iter0 \
    --output-dir /root/autodl-tmp/outputs/vmad_v4_l2_best \
    --sample-ids 7 17 21 31 32 33 34 43 46 47 \
    --alpha 0.005 --num_opt_steps 200 \
    --no-svd --no-disentangle --no-blend --no-token_decode \
    --content SELF --resume -v

# Apply-only（换 alpha，复用 assets）
python run_batch_extract.py --apply-only \
    --video-dir /root/autodl-tmp/data/video-200/water_mark_out \
    --caption-dir /root/autodl-tmp/outputs/hybrid_iter_v4/captions_iter0 \
    --output-dir /root/autodl-tmp/outputs/vmad_v4_l2_a008 \
    --sample-ids 7 17 21 31 32 33 34 43 46 47 \
    --alpha 0.008 --no-blend --no-token_decode \
    --content SELF --seed 42 -v
```

| 配置 | CLIP | XCLIP | 命令关键参数 |

## 无效/有害方法

### Layer 1 — Token Decode（motion_text 拼接）

| 项目 | 内容 |
|------|------|
| 状态 | ❌ 有害 |
| 影响 | CLIP -0.016 |
| 根因 | Token Decoder 在错误空间（输入 embedding）做最近邻，解码出乱码 |
| 关闭方式 | `--no-token_decode` |
| 修复方向 | 改用 `_build_output_codebook()` 在输出空间搜索（代码已有但未调用） |

### Layer 3 — Noise Prior Blending

| 项目 | 内容 |
|------|------|
| 状态 | ❌ 严重有害 |
| 影响 | CLIP -0.10~-0.15 |
| 根因 | 外部 `torch.randn` 和 pipeline 内部 `randn_tensor` 路径产生不同随机数 |
| 关闭方式 | `--no-blend` |
| 修复方向 | 让 pipeline 先正常生成噪声，再在外部 blend eta_motion（见下方修复方案） |
