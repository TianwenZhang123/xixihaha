- Apply：三层全开（motion_text 拼接 + Δe hook + noise prior blend），alpha=0.001

**输出路径**：`/root/autodl-tmp/outputs/vmad_phase3a_full/`

## Bug 修复与验证实验

**时间**：2025-05-31

### 发现的 Bug

| # | Bug | 位置 | 影响 | 修复 |
|---|-----|------|------|------|
| 1 | `apply_noise_prior` 无条件调用 | `pipeline.py` L657 | generator 状态被额外 `torch.randn` 消耗，后续 diffusion 采样的随机序列完全偏移，生成视频与 P-Flow 完全不同 | 加 `if cfg.use_blend:` 条件保护 |
| 2 | `motion_text` 无条件拼接到 prompt | `pipeline.py` L630 | 即使传 `--no-token_decode`，apply 阶段仍读取 asset 中已保存的垃圾 token（如 `ayarak guerre polítics 熙 カラー`）并拼接到 prompt，污染 text encoding | 加 `and cfg.use_token_decode` 条件 |

**根因分析**：

- Bug 1：extract 阶段的 `use_blend` 检查在 L746 是正确的（`if eta_motion is not None and cfg.use_blend:`），但 apply 阶段的 L657 漏掉了条件检查，导致 `--no-blend` flag 在 apply 中无效。
- Bug 2：`--no-token_decode` 只控制 extract 阶段是否执行 token decoding，但 apply 阶段从已保存的 asset 中读取 motion_text 时未检查该 flag。

**Token Decode 产生乱码的原因**：Token Decoder 在 T5 encoder 的 **输入 embedding 空间** 做最近邻搜索，但 Δe 实际存在于 T5 encoder 的 **输出 hidden states 空间**（经过 12 层 Transformer 变换后）。两个空间的几何结构完全不同，导致最近邻结果无语义意义。代码中虽有 `_build_output_codebook()` 方法可在正确空间搜索，但未被调用。

### 验证实验（单样本 #7）

以下实验逐步隔离 bug 影响，确认修复效果：

| # | 实验配置 | orig_gen_clip | orig_gen_xclip | 说明 |
|---|----------|--------------|----------------|------|
| 0 | P-Flow baseline（参考值） | **0.9159** | **0.7108** | P-Flow 原始结果 |
| 1 | P-Flow reseed（同 caption/seed/steps） | 0.9159 | 0.7108 | ✅ 确认 P-Flow 可复现 |
| 2 | VMAD 原版（修复前，三层全开） | 0.8007 | — | Bug 1+2 同时生效 |
| 3 | VMAD --no-velocity（修复前） | 0.8031 | — | 仍受 Bug 1 影响 |
| 4 | VMAD --no-blend（修复前） | 0.8031 | — | --no-blend 无效 (Bug 1) |
| 5 | VMAD --no-blend 修复后，仍有 motion_text | 0.8986 | 0.6384 | Bug 1 修复，Bug 2 残留 |
| **6** | **VMAD 纯净基线（三层全关，两 bug 均修复）** | **0.9150** | **0.7077** | **✅ 与 P-Flow 一致** |

**关键结论**：

- Bug 1（noise_prior 消耗 generator）造成 **-0.12 CLIP** 的巨大偏差（0.9150 → 0.80）
- Bug 2（motion_text 污染 prompt）造成 **-0.016 CLIP** 的轻微偏差（0.9150 → 0.8986）
- 两个 bug 修复后，VMAD pipeline 在三层全关时与 P-Flow 完全等价（CLIP 差异 < 0.001）

## Layer 2 (Δe) 效果验证 — Strength 扫描

**时间**：2025-05-31

**目的**：在修复 bug 后，隔离测试 Layer 2（Position-Aware Velocity Field Matching 优化的 Δe）的实际效果。通过调整 `alpha` 参数控制注入强度（`effective_strength = strength × alpha`），寻找最优注入点。

**实验设计**：
- 关闭 Layer 1 motion_text（`--no-token_decode`）：排除垃圾 token 干扰
- 关闭 Layer 3 noise prior（`--no-blend`）：排除 generator 状态偏移
- 保留 Layer 2 velocity（不加 `--no-velocity`）：仅 Δe hook 作为变量
- Asset 来源：vmad_phase3a_full 已提取的 assets（‖Δe‖=36.25）

**命令模板**：
```bash
cd /root/autodl-tmp/videofake/VMAD
mkdir -p /root/autodl-tmp/outputs/vmad_l2_a{XXX}
ln -s /root/autodl-tmp/outputs/vmad_phase3a_full/assets /root/autodl-tmp/outputs/vmad_l2_a{XXX}/assets

python run_batch_extract.py \
    --apply-only \
    --video-dir /root/autodl-tmp/data/video-200/water_mark_out \
    --caption-dir /root/autodl-tmp/outputs/baseline_captions \
    --output-dir /root/autodl-tmp/outputs/vmad_l2_a{XXX} \
    --content SELF --sample-ids 7 \
    --no-blend --no-token_decode \
    --alpha {ALPHA} --seed 42

python evaluation/run_reproduction_eval.py \
    --orig-dir /root/autodl-tmp/data/video-200/water_mark_out \
    --gen-dir /root/autodl-tmp/outputs/vmad_l2_a{XXX}/generated \
    --caption-dir /root/autodl-tmp/outputs/baseline_captions \
    --output-dir /root/autodl-tmp/outputs/vmad_l2_a{XXX}/eval \
    --limit 1
```

### 结果（单样本 #7）

| Alpha | Effective Strength | 实际注入量 (‖Δe‖×α) | orig_gen_clip | orig_gen_xclip | Δ CLIP vs 基线 |
|-------|-------------------|---------------------|--------------|----------------|---------------|
| 0 (基线) | 0 | 0 | **0.9150** | **0.7077** | — |
| 0.001 | 0.001 | 0.036 | 0.9086 | 0.6857 | -0.0064 |
| **0.01** | **0.01** | **0.36** | **0.9192** | **0.6987** | **+0.0042** |
| 0.05 | 0.05 | 1.81 | 0.8999 | 0.5925 | -0.0151 |
| 0.1 | 0.1 | 3.63 | 0.9016 | 0.5670 | -0.0134 |

### 分析

1. **存在最优注入强度**：alpha=0.01 时 CLIP 超过基线（+0.0042），达到 0.9192（甚至超过 P-Flow 的 0.9159）。说明 Δe 确实编码了有用的运动信息。

2. **强度曲线呈倒 U 形**：
   - alpha=0.001：注入量太小（0.036），信噪比不足，只是微小噪声扰动 → 轻微下降
   - alpha=0.01：注入量适中（0.36），Δe 的运动方向开始主导 → 超过基线
   - alpha=0.05~0.1：注入量过大（1.8~3.6），Δe 开始主导 embedding → 偏离正常生成路径，质量下降

3. **CLIP vs XCLIP 的分离**：alpha=0.01 时 CLIP 提升但 XCLIP 略降。CLIP 测帧级外观相似度，XCLIP 测时序连贯性。这可能说明 Δe 主要改善了视觉外观匹配，但对时序运动连贯性的捕获还不够。

4. **最优区间**：effective_strength 在 0.005~0.02 之间可能存在更精细的最优点。

## V4 Caption + Layer 2 消融验证（Caption 质量 × Δe 叠加效果）

**时间**：2025-06-01

**目的**：验证 P-Flow V4 自动化改写 caption（融合策略：Subject-First + Temporal Action Chain + Preserve Visual Vocabulary）与 VMAD Layer 2 Δe 注入是否可以互补叠加。核心问题：当 caption 已经很好时，Δe 还有多少增量？

**实验设计**：

2×2 消融矩阵，以 sample #7 做单样本快速验证：

| | 三层全关（纯 T2V） | Layer 2 α=0.01 |
|---|---|---|
| **Baseline caption** | 实验 A（已有） | 实验 B（已有） |
| **V4 caption** | 实验 C（新跑） | 实验 D（新跑） |

**关键设计决策**：

- 实验 C/D 必须**重新 extract**（不能复用之前 baseline caption 的 assets），因为 Δe 是相对于 e₀ = T5_encode(caption) 优化的残差。caption 变了 → e₀ 变了 → 最优 Δe 方向也变了。
- V4 caption 来源：`/root/autodl-tmp/outputs/hybrid_iter_v4/captions_iter0/`（P-Flow V4 自动化改写 iter1 的 prompt）

**V4 Caption 内容（sample #7）**：

> Two small sailboats floating on a cup of coffee. The first boat, positioned towards the left side of the frame, is larger and more detailed, with a white sail that has a black symbol on it. The second boat, slightly smaller and to the right, also features a white sail with a distinct black symbol. Both boats have dark brown hulls and appear to be intricately designed.
>
> As the video progresses, the two boats initially remain motionless, then gradually begin to drift slowly around the cup of coffee. The larger boat moves clockwise while the smaller one moves counterclockwise, creating a sense of dynamic movement within the still setting. The contrast between the dark coffee and the light wooden boats creates a striking visual effect. The camera remains steady throughout the video, allowing viewers to fully absorb the intricate details of the boats as they navigate through the dark coffee surface in gentle circular motion.

**命令**：

```bash
cd /root/autodl-tmp/videofake/VMAD

# 实验 C: V4 caption + 三层全关
python run_batch_extract.py \
    --video-dir /root/autodl-tmp/data/video-200/water_mark_out \
    --caption-dir /root/autodl-tmp/outputs/hybrid_iter_v4/captions_iter0 \
    --output-dir /root/autodl-tmp/outputs/vmad_v4caption_pure \
    --sample-ids 7 \
    --alpha 0.01 --num_opt_steps 200 \
    --no-svd --no-disentangle --no-blend --no-token_decode --no-velocity \
    --content SELF --resume -v

# 实验 D: V4 caption + Layer 2 α=0.01
python run_batch_extract.py \
    --video-dir /root/autodl-tmp/data/video-200/water_mark_out \
    --caption-dir /root/autodl-tmp/outputs/hybrid_iter_v4/captions_iter0 \
    --output-dir /root/autodl-tmp/outputs/vmad_v4caption_l2 \
    --sample-ids 7 \
    --alpha 0.01 --num_opt_steps 200 \
    --no-svd --no-disentangle --no-blend --no-token_decode \
    --content SELF --resume -v
```

### 结果（sample #7）

| 配置 | orig_gen_clip | orig_gen_xclip | Δ CLIP vs A | Δ XCLIP vs A |
|------|--------------|----------------|-------------|--------------|
| A) Baseline caption + 三层全关 | 0.9150 | 0.7077 | — | — |
| B) Baseline caption + Layer 2 α=0.01 | 0.9192 | 0.6987 | +0.0042 | -0.0090 |
| **C) V4 caption + 三层全关** | **0.9397** | **0.7260** | **+0.0247** | **+0.0183** |
| **D) V4 caption + Layer 2 α=0.01** | **0.9394** | **0.7486** | **+0.0244** | **+0.0409** |

### 增量分析

| 对比 | Δ CLIP | Δ XCLIP | 含义 |
|------|--------|---------|------|
| C - A（V4 caption 的纯贡献） | +0.0247 | +0.0183 | caption 质量提升带来的基线提升 |
| B - A（Δe 在 baseline caption 上的贡献） | +0.0042 | -0.0090 | Δe 对 CLIP 有增量，对 XCLIP 轻微负面 |
| **D - C（Δe 在 V4 caption 上的贡献）** | **-0.0003** | **+0.0226** | **CLIP 饱和，但 XCLIP 显著提升** |
| D - B（V4 caption + Δe vs Baseline + Δe） | +0.0202 | +0.0499 | 双指标大幅领先 |

### 核心结论

1. **V4 caption 和 Layer 2 Δe 是互补叠加关系，但各自贡献维度不同**：
   - V4 caption 主攻 **CLIP（帧级外观）**：+0.0247，将外观相似度推到 0.94 高位
   - Δe 主攻 **XCLIP（时序运动）**：在 V4 基础上再 +0.0226，将运动连贯性从 0.7260 提升到 0.7486

2. **CLIP 维度的优化空间被 V4 caption 挤压**：Δe 在 baseline 上能贡献 +0.0042 CLIP，但在 V4 上仅 -0.0003（持平）。说明 V4 caption 已经把 T5 embedding 推到了帧级外观的最优方向附近，Δe 无法再改善。

3. **Δe 的独特价值在时序信息**：V4 caption 虽然加了时序动作链（"initially remain motionless, then gradually begin to drift..."），但这些文字描述无法精确编码运动的节奏、加速度、相位等连续信息。Δe 在 embedding 空间的修正恰好补上了这块。

4. **最佳组合（D）达到了所有实验中的最高 XCLIP**：0.7486 超过了 P-Flow 所有实验的单样本结果。

5. **有趣的反转**：在 baseline caption 上，Δe 提升 CLIP 但降低 XCLIP（B vs A）；在 V4 caption 上，Δe 对 CLIP 无效但大幅提升 XCLIP（D vs C）。这说明 Δe 的优化方向与 caption 质量有交互——当 caption 已经给出好的外观引导时，Δe 的残差修正被"释放"到时序维度。

## Layer 3 (Noise Prior) 消融验证

**时间**：2025-06-01

**目的**：验证 Layer 3（Flow Matching Inversion → SVD Spectral Decomposition → Noise Prior Blending）在修复后的 VMAD pipeline 中是否有正向贡献。此前 P-Flow 框架中 noise prior 验证有效（+0.0077 CLIP），但未在 VMAD bug 修复后独立测试。

**实验设计**：

在已有 A-D 基础上补充 E-H 四个实验，形成完整 2×4 矩阵：

| | 三层全关 | L2 only | L3 only | L2+L3 |
|---|---|---|---|---|
| Baseline caption | A | B | E | F |
| V4 caption | C | D | G | H |

**命令**（apply-only，复用已有 assets）：

```bash
cd /root/autodl-tmp/videofake/VMAD
OUTBASE=/root/autodl-tmp/outputs111

# E) Baseline + L3 only (alpha=0.001)
python run_batch_extract.py --apply-only --video-dir ... --output-dir ${OUTBASE}/vmad_l3_only \
    --no-velocity --no-token_decode --alpha 0.001 --seed 42

# F) Baseline + L2+L3 (alpha=0.01)
python run_batch_extract.py --apply-only --video-dir ... --output-dir ${OUTBASE}/vmad_l2l3 \
    --no-token_decode --alpha 0.01 --seed 42

# G) V4 + L3 only (alpha=0.001)
python run_batch_extract.py --apply-only --video-dir ... --output-dir ${OUTBASE}/vmad_v4caption_l3 \
    --no-velocity --no-token_decode --alpha 0.001 --seed 42

# H) V4 + L2+L3 (alpha=0.01)
python run_batch_extract.py --apply-only --video-dir ... --output-dir ${OUTBASE}/vmad_v4caption_l2l3 \
    --no-token_decode --alpha 0.01 --seed 42
```

### 结果（sample #7）

| 编号 | Caption | 层配置 | orig_gen_clip | orig_gen_xclip | Δ CLIP vs 基线 |
|------|---------|--------|--------------|----------------|----------------|
| A | Baseline | 三层全关 | 0.9150 | 0.7077 | — |
| B | Baseline | L2 only (α=0.01) | 0.9192 | 0.6987 | +0.0042 |
| **E** | **Baseline** | **L3 only (α=0.001)** | **0.8097** | **0.6407** | **-0.1053** |
| **F** | **Baseline** | **L2+L3 (α=0.01)** | **0.7952** | **0.7025** | **-0.1198** |
| C | V4 | 三层全关 | 0.9397 | 0.7260 | — |
| D | V4 | L2 only (α=0.01) | 0.9394 | 0.7486 | -0.0003 |
| **G** | **V4** | **L3 only (α=0.001)** | **0.7993** | **0.6782** | **-0.1404** |
| **H** | **V4** | **L2+L3 (α=0.01)** | **0.7875** | **0.6952** | **-0.1522** |

### 核心结论：Layer 3 在 VMAD 中有害

1. **Layer 3 造成 CLIP 暴跌 -0.10 ~ -0.15**：无论 caption 质量如何，只要开 blend，CLIP 从 0.91~0.94 崩到 0.79~0.81。

2. **根因：generator 状态偏移**。`apply_noise_prior()` 调用 `torch.randn(generator=generator)` 生成 η_random 进行混合，这改变了 generator 内部状态。后续 diffusion 30 步采样使用同一个 generator，随机序列完全偏移，生成结果与基线大相径庭。这本质上和 Bug 1 是同一个问题的不同表现。

3. **与 P-Flow 结果的矛盾解释**：P-Flow 当时的 `+noise_prior` 实验可能使用了**独立的 generator** 或者在 noise 采样前重置了 seed。VMAD pipeline 共享单一 generator 导致冲突。

4. **Layer 3 的理论价值仍在，但工程实现需要修复**：解决方案是将 noise blending 用独立的 generator（如 `torch.Generator().manual_seed(seed+1)`），不干扰主 diffusion generator。但当前阶段不值得投入修复——Layer 2 已经足够好。

5. **决策：放弃 Layer 3，专注 Layer 2 + V4 caption**。

## Alpha 精扫（V4 Caption + Layer 2 Only）

**时间**：2025-06-01

**目的**：在确认 Layer 3 无效后，在最优组合（V4 caption + Layer 2 only, 不开 blend）上精细搜索 alpha 最优值。之前 alpha=0.01 是从 baseline caption 实验中选出的，V4 caption 下最优点可能不同。

**实验设计**：固定 V4 caption + `--no-blend --no-token_decode`，扫描 alpha ∈ {0.005, 0.008, 0.015, 0.02}，与已有的 alpha=0.01 (D) 和 alpha=0 (C) 对比。

**命令**：

```bash
for alpha in 0.005 0.008 0.015 0.02; do
    python run_batch_extract.py --apply-only \
        --caption-dir /root/autodl-tmp/outputs/hybrid_iter_v4/captions_iter0 \
        --output-dir ${OUTBASE}/vmad_v4_alpha_${alpha} \
        --sample-ids 7 --alpha ${alpha} --no-blend --no-token_decode \
        --content SELF --seed 42 -v
done
```

### 结果（sample #7, V4 caption + L2 only）

| Alpha | Effective Strength | orig_gen_clip | orig_gen_xclip | Δ CLIP vs C | Δ XCLIP vs C |
|-------|-------------------|--------------|----------------|-------------|--------------|
| 0 (C 基线) | 0 | 0.9397 | 0.7260 | — | — |
| **0.005** | **0.005** | **0.9446 ★** | **0.7541** | **+0.0049** | **+0.0281** |
| **0.008** | **0.008** | **0.9395** | **0.7581 ★** | **-0.0002** | **+0.0321** |
| 0.01 (D) | 0.01 | 0.9394 | 0.7486 | -0.0003 | +0.0226 |
| 0.015 | 0.015 | 0.9343 | 0.7519 | -0.0054 | +0.0259 |
| 0.02 | 0.02 | 0.9381 | 0.7299 | -0.0016 | +0.0039 |

### 分析

1. **最优 alpha = 0.005（双指标均衡）或 0.008（XCLIP 极值）**：
   - α=0.005：**唯一同时提升 CLIP (+0.0049) 和 XCLIP (+0.0281) 的点**，帧级外观和时序运动都改善
   - α=0.008：CLIP 持平但 **XCLIP 达到所有实验最高值 0.7581**

2. **强度曲线更清晰**：
   - α < 0.005：注入量不足（从之前 α=0.001 的负结果推断）
   - α ∈ [0.005, 0.008]：最优区间，Δe 编码的运动信息与 V4 caption 互补
   - α > 0.01：注入过量，CLIP 开始下降，XCLIP 也在 0.02 时回落
   - 相比 baseline caption 的最优点 α=0.01，V4 caption 的最优点左移到 0.005~0.008

3. **为什么 V4 caption 下最优 alpha 更小**：V4 caption 已经把 T5 embedding e₀ 推到更接近目标的方向，Δe 只需要更小的修正量就能补齐时序信息。过大的 Δe 反而会把已经很好的 e₀ 推偏。

4. **当前全局最优配置**：**V4 caption + Layer 2 (α=0.005), no blend**
   - CLIP = 0.9446（所有实验最高）
   - XCLIP = 0.7541（次高，仅低于 α=0.008 的 0.7581）
   - 双指标综合最优

### VMAD Apply 流程（修复后）

```
content_prompt
     │
     ▼
[Layer 1] if use_token_decode: prompt += motion_text  ← 当前产生乱码，建议关闭
     │
     ▼
[Layer 2] if use_velocity: hook(encoder_output += effective_strength × Δe)
     │                     effective_strength = strength × alpha
     ▼
[Layer 3] if use_blend: latents = √(α)·η_motion + √(1-α)·η_random
     │                  ⚠️ 会消耗 generator 状态！导致后续 diffusion 偏移！
     │                  ⚠️ 已证实有害（-0.10~-0.15 CLIP），不应使用！
     ▼
[Generate] pipe(prompt=full_prompt, latents=latents, generator=generator)
```

### Layer 3 失败的根因备忘

Layer 3 noise prior blending 需要调用 `torch.randn(shape, generator=generator)` 来生成 η_random。这一次调用会推进 generator 的内部状态（MT19937 序列前进一步）。随后 diffusion 采样的 30 步中每步也调用同一个 generator 生成噪声。由于 generator 状态被 noise blending 额外消耗了一次，后续 30 步的噪声序列与不开 blend 时完全不同，导致生成结果严重偏离。

修复方案（未实施）：
```python
# 用独立 generator 做 noise blending，不影响主 diffusion generator
blend_gen = torch.Generator(device=device).manual_seed(seed + 1000)
eta_random = torch.randn(eta_motion.shape, generator=blend_gen, ...)
latents = sqrt(alpha) * eta_motion + sqrt(1-alpha) * eta_random
# 主 generator 状态不变，后续 diffusion 采样正常
```
