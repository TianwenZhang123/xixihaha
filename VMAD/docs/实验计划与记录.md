# VMAD 实验计划与记录

> **定位**：Video Prompt Inversion — 视频的逆问题。给定参考视频 V_ref 和冻结参数的 T2V 模型 v_θ，找到最优输入条件 (text, Δe, η_inv) 使模型忠实再现参考视频。
>
> **架构**：VMAD = P-Flow (Layer 1) + Velocity Field Matching (Layer 2) + Flow Matching Inversion (Layer 3)
>
> **评测指标**：`orig_gen_clip` = 原始视频与生成视频的 CLIP 帧级相似度；`orig_gen_xclip` = X-CLIP 时序语义相似度；`SSIM` / `LPIPS` 用于像素级评估。
>
> **实验环境**：AutoDL A800 80GB，Wan2.1-T2V-1.3B，生成参数 steps=30, guidance=5.0, seed=42, 480×832, 81帧。

---

## 总体方案：三层渐进编码

### 信息论视角

| 层级 | 编码内容 | 信息量 | 可迁移性 | 方法 |
|------|----------|--------|----------|------|
| Layer 1 (Text) | 主体、场景、动作类别 | ~100 bits | 最高（跨模型通用） | P-Flow V4 LLM 改写 |
| Layer 2 (Δe) | 节奏、加速度、相位、风格细节 | ~100K bits | 中（同模型通用） | Position-Aware Velocity Field Matching |
| Layer 3 (η_inv) | 精确空间结构、像素级布局 | ~1M bits | 最低（同种子绑定） | Flow Matching Inversion |

每一层编码的是前一层的**残差信息**，三层互不冗余。

### 核心公式

**Layer 2 — Velocity Field Matching：**
```
L_vel = E_{t~U[0,T_m]} [ || v_θ(x_t, t, e₀+Δe) - v* ||² ]
其中:
  v* = z₀ - η_inv  (理想速度场，rectified flow 线性插值性质)
  x_t = (1-t)·η_inv + t·z₀  (ground-truth ODE 轨迹)
  e₀ = TextEnc(VLM(V_ref))  (初始文本 embedding)
```

**Layer 3 — Flow Matching Inversion：**
```
η_inv = z₀ - ∫₀¹ v_θ(x_t, t, e₀) dt  (Euler discretization, N=50 steps)
```

**Position-Aware Gradient Scaling：**
```
∂L/∂Δe[:,j,:] ← w_j · ∂L/∂Δe[:,j,:]
w_j ∝ attention_mass(j)  (DiT position 0 权重为中间位置的 10-15×)
```

### 与 P-Flow 的关系

P-Flow 已验证的结论（VMAD 直接继承，不重复验证）：

| P-Flow 结论 | 数据 | VMAD 如何使用 |
|-------------|------|---------------|
| Subject-First 开头优于 "The video shows..." | CLIP +0.014 | Layer 1 改写策略直接复用 V4 |
| 时序动作链提升 XCLIP | XCLIP +0.013 | 改写时注入 initially/then/gradually |
| Position 0 注意力权重 10-15× | 纯位置实验证实 | Layer 2 的 gradient scaling 依据 |
| 迭代 1 轮最优，多轮退化 | V4 iter1 最佳 | 迭代轮数默认 --iter 1 |
| V4 prompt 主体识别 100% | 消融实验 | 复用完整 REWRITE_SYSTEM |

### Phase 1：Layer 3 — Flow Matching Inversion 验证

**目的**：验证 η_inv 的质量，以及混入不同比例 α 时对生成的影响。

**核心假设**：η_inv 编码了视频的完整空间结构，即使 α 极小（0.001），也能在不破坏生成自然度的前提下提升保真度。

**实验设计**：

| 子实验 | α 值 | 说明 |
|--------|------|------|
| 1a | 0.0（纯随机噪声） | baseline，等同于纯 Layer 1 |
| 1b | 0.001 | 千分之一混入，最低风险 |
| 1c | 0.01 | 百分之一 |
| 1d | 0.1 | 十分之一 |
| 1e | 1.0 | 纯 η_inv（完美重建上限） |

**运行命令**：

### Phase 2：Layer 3 升级 — RF-Solver 高阶反演

**目的**：用 2 阶 Taylor 展开（RF-Solver-2）替换 Euler inversion，降低 η_inv 的离散化误差。

**理论依据**：Taming Rectified Flow (ICML 2025) 证明 RF-Solver-2 可将重建误差降低 50%+。

**改动范围**：`src/pipeline.py` 中 `_inversion()` 方法，约 50 行代码。

**消融设计**：

| 方法 | 步数 | 预期精度 |
|------|------|---------|
| Euler (当前) | 50 | baseline |
| RF-Solver-2 | 50 | +0.5~1% CLIP |
| RF-Solver-2 | 25 | 验证是否 25 步即可达到 Euler 50 步水平 |

**运行命令**：
```bash

    --video-dir /root/autodl-tmp/data/video-200/water_mark_out \

    --output-dir /root/autodl-tmp/outputs/vmad_phase2_rfsolver \
```

### Phase 4：Layer 2 应用 — Motion Transfer

**目的**：验证 VMAD 的独特能力——将提取的运动资产迁移到新内容上。

**核心假设**：当 T_m<1.0 时，Δe 只编码运动信息（不编码具体外观），因此可以应用到不同主体上。

**实验设计**：

从参考视频提取运动资产，分别应用到 5 种不同内容：
- "a white cat" (基础测试)
- "a robot" (跨类别)
- "a teddy bear" (布料物理)
- "an astronaut" (人形但不同纹理)
- "a goldfish" (极端跨类别)

### Phase 5：Iterative Refinement — VLM 反馈闭环

**目的**：在完整三层架构上验证 VLM 迭代反馈的增益。由于 Layer 2 已提供运动保底，VLM 反馈可专注于内容/风格差异。

**关键改进**（相对 P-Flow 的迭代策略）：
- Layer 2 (Δe) 提供连续运动保底 → VLM 只需关注 CONTENT/STYLE 差异
- 迭代轮数更少（1-2 轮 vs P-Flow 的 3+）
- 当 VLM 反馈全部为 "matches" 时自动终止（避免 P-Flow 的过度修改退化问题）

### Phase 6 (P1)：高级优化方向

以下实验在 Phase 1-5 完成后按优先级进行：

**6a. η_inv 频域分解**（来源：FreqPrior, ICLR 2025）
```
η_inv = η_low + η_high
η_init = η_low_inv + η_high_random  (低频来自参考，高频随机)
```
避免 trivial solution 质疑，同时保留运动结构。

**6b. 动态 Position Weight**（来源：Attention Sinks in DiT, ICML 2026）
每个 timestep 提取实际 attention map 作为梯度权重，替代固定 U-shape。

**6c. Δe 分频段优化**（来源：ConsisID, CVPR 2025）
- 前 100 步优化 Δe 低频分量（对 t∈[0,0.5] 计算 loss）
- 后 100 步优化 Δe 高频分量（对 t∈[0.5,1] 计算 loss）

**6d. Inversion-Free Velocity Matching**（来源：FlowEdit, ICCV 2025）
```python
# 当前方案（依赖 η_inv）
v* = z₀ - η_inv
L_current = E_t [ || v_θ(x_t, t, e₀+Δe) - v* ||² ]

# 新方案（无反演，直接分布间 ODE）
L_new = E_t [ || v_θ(x_t, t, e₀+Δe) - (z₀ - x_t)/(1-t) ||² ]
# 其中 x_t = (1-t)·η_rand + t·z₀
# 不需要 η_inv！完全绕过 inversion 误差累积
```
